## 日本語ルール
執筆にあたっては以下のルールを厳守する．
- JLPT（日本語能力試験）N1相当の流暢で適切な日本語を用いること
- ですます調の敬体をベースに，一定の砕けて親しみやすい文体を用いること
- 句読点は「、。」ではなく「，．」を用いること
- 学術的かつ中立的な立場で議論を展開すること
- 一人称は「私」を用いること

### 記事テンプレート

記事作成のプロンプトにおいて特に指定がない限りは以下のテンプレートに従って執筆する．

```markdown
# {タイトル}
<!-- 読者の興味を引く簡潔なタイトル。「〜したい！」「〜してみた」「〜入門」など -->
<!-- ユーザからの指定があればそれにそのまま従う -->

## はじめに
<!-- 記事の背景・モチベーション・解決したい課題を簡潔に述べる -->
<!-- 読者が「自分に関係ある」と思えるフックを入れる -->

## 前提条件
<!-- 必要に応じて記載。不要な場合は省略可 -->
<!-- - 動作環境（OS、言語バージョン、ライブラリ等）-->
<!-- - 読者に求める前提知識 -->
<!-- - 本記事で扱わない範囲の明示 -->

## {本論セクション1}
<!-- 記事の核となる内容。複数セクションに分割してよい -->
<!-- 課題 → 解決策 → 実装例 の流れが基本 -->

## {本論セクション2}
<!-- 必要に応じて追加 -->

## まとめ
<!-- 記事の要点を箇条書きで整理 -->
<!-- 読者が持ち帰るべきポイントを明確に -->

## おわりに
<!-- 今後の展望、関連リンク、謝辞など -->
<!-- 参照元の実験レポートがある場合はここで言及 -->
```

#### テンプレート適用時の注意

| 項目 | ガイドライン |
|------|-------------|
| タイトル | 具体的かつ検索されやすいキーワードを含める |
| はじめに | 3〜5文程度で課題設定を完了させる |
| 前提条件 | 環境依存が少ない場合は省略可 |
| 本論 | 内容に応じてセクション名を適切に命名する |
| コード例 | 実際に動作するコードを掲載し、言語指定を付ける |
| まとめ | 3〜5項目の箇条書きで要点を整理 |
| おわりに | 冗長にならないよう簡潔に |


## 命名規則

| 要素 | 形式 | 例 |
|------|------|-----|
| ディレクトリ名 | `art{NNN}_{記事名}` | `art001_genesis_rl_walking` |
| 主原稿 | `art{NNN}_{記事名}.md` | `art001_genesis_rl_walking.md` |
| 補足ファイル | `art{NNN}_{記事名}_{補足}.{拡張子}` | `art001_genesis_rl_walking_architecture.drawio.svg` |

- `{NNN}`: 3桁の通し番号（001, 002, ...）
- `{記事名}`: snake_caseで簡潔に（長すぎる場合は略称を使用）
- `{補足}`: 図や画像の内容を示すsuffix（`_architecture`, `_sequence`, `_result_01` 等）

## ルールの意図

1. **自己完結性**: ファイル単体で記事番号と内容が判別できる（移動・共有に強い）
2. **検索性**: `art002`で検索すれば関連ファイルがすべてヒット
3. **整理性**: 各記事の関連ファイルがディレクトリ内にカプセル化される

## 補足ファイルの扱い

- 補足ファイル（図・画像等）は必ず主原稿（`.md`）から参照すること
- 参照されていない孤立ファイルは作成しない
- 参照例: `![システムアーキテクチャ](./art001_genesis_rl_walking_architecture.drawio.svg)`

## 実験レポートとの関係

本リポジトリの実験レポート（`doc/experiments/exp{NNN}_*/`）から得られた知見を記事として再構成する場合がある．その際は以下を遵守すること:

- 記事内で参照元の実験レポートを明記する（例: `本記事はexp007の成果に基づく`）
- 実験レポートの内容をそのままコピーせず、公開向けに再構成・加筆する
- 実験固有の詳細（失敗談、試行錯誤の過程等）は実験レポートに残し、記事には知見のエッセンスを抽出する
