# V6: action_rate scale増加（ジタバタ挙動抑制）

## 概要

V4で観測されたジタバタ挙動（hip_yaw 3.62Hz振動、L_hip_rollアクション変化率+32%、R_hip_roll速度RMS 1.52倍）の抑制を目指す。V4レポートの「V6以降の候補」第1案に基づき、action_rate scaleを-0.005→-0.01（2倍増）に変更する。action_rateペナルティはアクション間の変化量の二乗に比例するため、高周波関節振動に対して直接コストを課す。

## 前バージョンからの改善事項

V4レポートの「次バージョンへの提案」に記載された「V6以降の候補」第1案に基づく変更。

| パラメータ | V5値 | V6値 | 変更理由 |
|-----------|------|------|---------|
| action_rate scale | -0.005 | -0.01 | V4で観測されたhip_yaw 3.62Hz振動の抑制。-0.005はexp008の前進歩行用設計で全方向歩行には不十分。保守的に2倍増で試行 |

## 設計詳細

### action_rate penaltyの役割

action_rateペナルティは `Σ(action_t - action_{t-1})²` に比例する。高周波振動はステップ間のアクション差分が大きいため、このペナルティで直接抑制される。

V4の分析で判明した問題:
- hip_yaw両側の主周波数が3.62 Hzに遷移（V3: 0.12-1.25 Hz）
- 高周波成分比率: L_hip_yaw 62.5%（1.44x）、R_hip_yaw 72.1%（1.68x）
- L_hip_rollアクション変化率が+32%突出
- action_rateペナルティは+24.6%悪化（-0.0313→-0.0390）

### scale選択の根拠

V4レポートは-0.01 or -0.02を候補としている。overkillリスク（exp008 V5の教訓: penalty/positive ratio 0.186でoverkill zone）を考慮し、保守的に-0.01（2倍増）を選択。V4のratio 0.077は健全域にあるため、action_rate単独の2倍増ではoverkillに至りにくいと判断。

### 報酬構成

16項目（V5と同一構成、action_rate scaleのみ変更）。

## 考察と改善案

### 目視評価所見

目視評価において以下が観察された:
- グネグネした動きは多少マシになった
- 後退するとき転ぶ問題が新たに出ている

### 対話による多角的分析

以下、データ分析専門エージェント（DA）とロボティクス・物理学俯瞰エージェント（Lead）の対話形式で考察を行う。

---

**DA**: V6で最も顕著な変化5項目を定量報告します。(1) **hip_yaw振動の劇的抑制**: hip_yaw vel RMS L 3.743→0.771 (0.21x), R 2.535→0.694 (0.27x)。>2Hz成分比率 L 85.6%→10.5% (0.12x)。アクション変化率RMS全体平均35.2→18.1 /s (0.51x)。(2) **hip_pitch交互歩行の崩壊**: 相関-0.226→+0.036（正領域=交互歩行消失）。L_hip_pitch主周波数が0.12Hz→1.12Hzに回復したがR_hip_pitchとの位相関係が崩壊。ローリング相関std 0.252→0.539に増大（歩行パターンの不安定化）。(3) **hip_roll action RMSの急減と非対称化**: L 30.01→3.65 (0.12x), R 30.98→11.47 (0.37x)。R/L比 V5 1.03→V6 3.14。(4) **横方向揺れ大幅悪化**: detrended std 62.2→107.9mm (+73.5%)、残差std +68.3%。(5) **Yawドリフト大幅改善**: +28.27°→-4.92°（絶対値83%減）。

「グネグネ多少マシ」はhip_yaw >2Hz 0.12-0.23x + アクション変化率0.51xで定量的に裏付けられます。「後退で転ぶ」はhip_pitch相関崩壊 + L/R非対称度+12.8%による交互歩行不成立と対応します。

**Lead**: DAの分析で最も重要な構造的知見は、**action_rateペナルティがhip_yaw振動だけでなく、hip_rollのバランス制御機構まで抑制した**ことです。

因果連鎖を整理します:
1. action_rate scale 2x → 二乗ペナルティにより**大振幅の低周波制御**も強く罰される
2. V5のhip_roll action RMS 30.0/31.0は歩行帯域（~1Hz）の大振幅バランス制御
3. action_rateペナルティは周波数×振幅の二乗に比例 → 大振幅の低周波制御のコストが高い
4. hip_rollバランス制御の抑制 → 交互歩行パターンを維持できない
5. hip_pitch位相崩壊 → 不安定な後退歩行 → 転倒

hip_rollの高周波成分比率が2.6x増大した点が裏付けです。低周波大振幅が抑制され、高周波小振幅が相対的に増加しました。これはexp008 V18（outward_limit too tight → hip_roll自由度喪失 → Yawドリフト-85.84°）と同じ「制約がバランス機構を破壊する」パターンです。ただしV6の場合、Yawは改善している点が異なります。hip_yaw振動の抑制がYaw改善に直結したためです。

ratio 0.074は健全域であり、overkillではありません。問題は**action_rateペナルティの選択性の欠如**です。高周波振動と低周波バランス制御を区別できない。

**DA**: Leadの仮説を補足データで検証します。hip_rollフェーズ別分析: V6のstance hip_roll L -9.2°/R +11.8°はV5の-12.2°/+12.2°から変化。特にLの内向き減少（-12.2°→-9.2°）が顕著で、バランスストロークもL 7.5→6.7° (-11.7%)に減少。一方、V6ではL外向き最大が+15.5°（V5: -2.1°）まで使用されており、**外向き修正が活性化**しています。V5はhip_rollを一方向（内向き）にしか使わなかったのに対し、V6は両方向に使い始めましたが、その振幅制御が不安定です。

接触遷移レートは8.27→6.52/s (-21.2%)に改善。Lマイクロスイングは36.8%→7.7%に劇的改善。ただしRマイクロスイングは7.1%→15.4%に悪化しており、L/R非対称性がここにも現れています。

**Lead**: 統合考察をまとめます。

**V6の本質**: action_rate 2xはhip_yaw振動抑制には**完全に成功**（主目的達成）したが、副作用としてhip_rollのバランス制御を抑制し、交互歩行パターンが崩壊した。これは**action_rateの構造的限界**（周波数選択性がない）に起因する。V7のKp/Kd変更は物理層の改善であり、報酬構造の問題とは独立に作用するため、V8では報酬パラメータの調整が必要。

V5レポートの未反映改善案への見解:
1. **action_rate -0.01→-0.02**: **不採用推奨**。V6で既にバランス制御が抑制されており、さらなる増加は悪化必至
2. **gait_frequency引き上げ**: **最有力候補**。exp008 V20で実証済み。高周波化 → 片足バランス時間短縮 → hip_rollバランス負荷軽減 → 交互歩行安定化の因果連鎖が期待できる
3. **カリキュラム学習**: 長期的には有効だが、実装コストが大きい。現状の報酬構造の問題を先に解決すべき

---

### 成功点

- **hip_yaw振動の劇的抑制（主目的達成）**: >2Hz成分比率 L 85.6%→10.5% (0.12x)、R 71.1%→16.2% (0.23x)。主周波数が2.38Hz→歩行帯域に回復
- **アクション変化率の半減**: 全体平均35.2→18.1 /s (0.51x)。全10アクションで改善
- **Yawドリフト大幅改善**: +28.27°→-4.92°（絶対値83%減）。hip_yaw非対称振動の消失が直接原因
- **ペナルティトレンドの反転**: ratio 0.091→0.074。V3→V4→V5の単調増加が反転し健全域に回復
- **Value function loss大幅改善**: 0.005→0.0015（-71%）。Criticの推定精度が回復
- **X速度改善**: 0.069→0.088 m/s (+28.3%)

### 課題

1. **hip_pitch交互歩行の崩壊**: 相関-0.226→+0.036。ローリング相関std 0.252→0.539に増大。安定した交互歩行パターンが消失
2. **横方向揺れの大幅悪化**: detrended std 62.2→107.9mm (+73.5%)。R²は0.110で依然低く、kinematic chainによる揺れが支配的
3. **Roll std悪化**: 3.10→4.05° (+30.7%)
4. **hip_roll L/R非対称の拡大**: R_hip_roll action RMS 11.47（L: 3.65、比率3.14x）。系統的バイアスが残存
5. **hip_roll高周波成分の増大**: L 10.4%→27.0% (2.61x)、R 9.6%→13.9% (1.46x)。action_rateが低周波バランス制御を抑制した副作用
6. **後退時の転倒**: 人間の目視で確認。hip_pitch相関崩壊と対応

### 次バージョンへの提案

**V7は関節グループ別Kp/Kd（膝Kp=50.0/Kd=3.0、足首Kp=20.0/Kd=5.0）の導入であり、先行して実施済み（未訓練）。報酬構成はV6と同一。**

**V8推奨: gait_frequency引き上げ 1.2 → 1.5 Hz**

根拠:
- exp008 V20で単一パラメータ変更による多指標同時改善が実証済み（X vel +22.4%, hip_pitch corr ALL-TIME BEST, lateral std -52.9%, Yaw≈0°, ratio -57.8%）
- V6のhip_pitch交互歩行崩壊は「片足バランス時間が長すぎる」ことが一因。高周波化により片足支持時間が短縮され、hip_rollバランス負荷が軽減される
- exp008では0.9→1.2 Hzで成功。exp009の全方向歩行ではさらに高い周波数（1.2→1.5 Hz）が必要な可能性がある（横歩き時のバランス確保のため）
- 1変更1検証原則を遵守。V7のKp/Kd変更がベースになるため、gait_frequencyのみを変更

期待効果:
- hip_pitch交互歩行パターンの回復（片足支持時間短縮→位相ロック促進）
- 横方向揺れの低減（バランスストローク短縮）
- hip_roll高周波成分の低減（低周波バランス需要の減少）

リスク:
- 1.5 Hzが高すぎる場合、feet_swing_heightペナルティ増大（exp008 V20で+17.5%の前例あり）
- 全方向歩行では前進歩行より制約が多く、高周波化の効果が限定的な可能性

**V5レポート未反映改善案の処理**:

| V5提案 | V8での採否 | 理由 |
|--------|-----------|------|
| action_rate -0.01→-0.02 | **不採用** | V6でバランス制御抑制が発生。さらなる増加は悪化必至 |
| gait_frequency引き上げ | **採用（推奨案）** | exp008 V20実証済み。V6の交互歩行崩壊に対する直接的解決策 |

## まとめ

exp009 V6はaction_rate scaleを-0.005→-0.01（2倍増）に変更し、V4以来のジタバタ挙動（hip_yaw高周波振動）の抑制を図った。

**主目的は完全に達成された**:
1. hip_yaw >2Hz成分比率: L 85.6%→10.5% (0.12x)、R 71.1%→16.2% (0.23x)
2. アクション変化率RMS全体平均: 35.2→18.1 /s (0.51x)
3. Yawドリフト: +28.27°→-4.92°（絶対値83%減）
4. penalty/positive ratio: 0.091→0.074（健全域に回復）

**しかし副作用として交互歩行パターンが崩壊した**:
1. hip_pitch相関: -0.226→+0.036（交互歩行消失）
2. 横方向detrended std: 62.2→107.9mm (+73.5%)
3. Roll std: 3.10→4.05° (+30.7%)

根本原因は**action_rateペナルティの周波数選択性の欠如**である。高周波振動（hip_yaw 2.38Hz）だけでなく、歩行帯域（~1Hz）のhip_rollバランス制御も二乗ペナルティにより強く罰された。hip_roll action RMSが30→3.7-11.5に急減した結果、交互歩行を維持するバランス機構が損なわれた。

V7（関節グループ別Kp/Kd）は先行して実施済み（未訓練）。V8ではgait_frequency引き上げ（1.2→1.5 Hz）を推奨する。exp008 V20で実証された「動的パラメータによる多指標同時改善」効果により、V6の交互歩行崩壊を解決しうる。

- 本バージョンの変更はaction_rate scaleのみ（-0.005→-0.01）。他の報酬構成・環境設定・コマンド設定はV5と完全同一
- V4レポートの「V6以降の候補」第1案に基づく変更
- 1変更1検証原則に基づき、action_rate scaleのみを変更
