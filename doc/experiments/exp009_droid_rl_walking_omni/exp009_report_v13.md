# V13: Mirror Augmentationによる左右対称性の構造的保証

## 概要

V11をベースに、Mirror Augmentation（L↔R反転データ拡張）を導入する。V10/V11/V12で一貫して観測された左右非対称歩容（hip_pitch range非対称+28.5%、Yawドリフト）に対し、報酬チューニングではなく訓練データの構造的対称化により解決を目指す。

## 前バージョンからの改善事項

V12レポートの「次バージョンへの提案」（Round 5-6追加分析）に基づき、Mirror Augmentationを導入。

V12で試行したsymmetry_vel_ema（報酬ベースの対称化）はtrivial solutionを許容して構造的に失敗し、右足過剰挙上の主指標（hip_pitch range非対称）はV11 +28.5% → V12 +28.6%で不変だった。追加分析により、非対称性の根本原因はPPO最適化の自発的対称性の破れであり、報酬関数の追加では原理的に解決不能と判明。データ自体を対称化するMirror Augmentationが唯一の構造的解法として選定された。

| パラメータ | V11値 | V13値 | 変更理由 |
|-----------|-------|-------|---------|
| mirror_augmentation | False（未使用） | **True** | PPO最適化の自発的対称性の破れを構造的に防止 |

### V13でMirror Augmentationを選定した理由

1. **V12のsymmetry_vel_emaが構造的に失敗**: trivial solutionを許容し、右側活動量の選択的削減でrange非対称を隠蔽。hip_pitch range非対称+28.5%は不変
2. **報酬ベース対称化の原理的限界**: 対称な報酬関数をいくら追加しても、PPOの確率的最適化が対称性を自発的に破る。V11提案のhip_yaw対称性報酬も同じ構造的限界に直面する
3. **非対称がコマンド方向依存**: ランダムコマンドでR>L +28.5%、FWD固定でL>R -27.0%。特定の脚を標的とする報酬設計は原理的に不適切
4. **Mirror Augmentationはtrivial solution不可**: データ自体が対称なため、非対称なポリシーは学習データに適合不能
5. **1変更1検証原則の厳密な遵守**: 報酬構成・パラメータは一切変更なし。mirror_augmentation=True のみ

### Mirror Augmentationの設計

#### 実装方式: 永続ミラー環境

4096環境のうち後半2048環境を永続的にミラー環境とする。ミラー環境では:
- **観測**: L↔Rスワップ + 矢状面鏡像変換（vy, roll, yaw符号反転）
- **アクション**: ポリシー出力をデミラー（L↔Rスワップ + yaw/roll符号反転）してから物理シミュレーションに適用
- **報酬**: 実状態で計算（ミラーなし）→ 物理的に正しい報酬

#### ミラー変換の詳細

**観測ベクトル (50次元)**:

| 成分 | インデックス | ミラー操作 |
|------|------------|-----------|
| base_lin_vel | 0-2 | vy (idx 1) 符号反転 |
| base_ang_vel | 3-5 | roll (idx 3), yaw (idx 5) 符号反転 |
| projected_gravity | 6-8 | gy (idx 7) 符号反転 |
| commands | 9-11 | cmd_vy (idx 10), cmd_vyaw (idx 11) 符号反転 |
| dof_pos | 12-21 | L↔Rスワップ + yaw/roll符号反転 |
| dof_vel | 22-31 | 同上 |
| actions | 32-41 | 同上 |
| gait_phase | 42-43 | 変更なし |
| leg_phase | 44-45 | L↔Rスワップ |
| feet_pos_z | 46-47 | L↔Rスワップ |
| contact_state | 48-49 | L↔Rスワップ |

**DOFミラー規則**:

| 関節 | 操作 | 根拠 |
|------|------|------|
| hip_yaw | L↔Rスワップ + 符号反転 | 矢状面鏡像でyaw回転方向が反転 |
| hip_roll | L↔Rスワップ + 符号反転 | 矢状面鏡像でroll回転方向が反転 |
| hip_pitch | L↔Rスワップ | pitch方向は鏡像で不変 |
| knee_pitch | L↔Rスワップ | 同上 |
| ankle_pitch | L↔Rスワップ | 同上 |

#### 横方向移動への安全性

「左に0.15m/sで歩く」の物理的鏡像は「右に0.15m/sで歩く」。ミラー時にcmd_vyの符号を反転するため、この物理的対応関係がそのまま学習される。横方向移動に悪影響を与えない。

### リスク

- vyおよびvyawの符号反転の実装ミスは致命的（左右逆転を学習）
- mirror時のobservation/action対応テーブルの正確な定義が不可欠
- Mirror Augmentationにより全ポリシーが構造的に対称になるため、非対称な最適解が存在する場合はパフォーマンスが低下する可能性（ただし対称URDFでは非対称が最適になる物理的理由はない）
- 実効的な学習データの多様性が半減する可能性（2048通常 + 2048ミラー ≒ 2048の独立データ）

## 考察と改善案

### 目視評価所見

目視評価において以下が観察された:
- 評価を起動した瞬間に転倒して終了した
- ヘッドレス評価では転倒せず完走したが、FWD固定コマンドで20秒間にYaw +92.4°の壊滅的旋回が発生

### 対話による多角的分析

以下、データ分析専門エージェント（DA）とロボティクス・物理学俯瞰エージェント（Lead）の対話形式で考察を行う。

---

**DA（Round 1: 初期定量分析）**:

- **Yawドリフト 7.3倍悪化（FWD固定）**: 12.7° → 92.4°（20秒間）。ドリフト速度 1.42°/s → 4.77°/s。直進安定性の致命的劣化。
- **横方向ドリフト 5.7倍悪化（FWD固定）**: y=0.574m → y=3.292m。Yawドリフトと連動した軌道逸脱。
- **L足接地偏重の顕著化**: L=51.7%→58.0%（+6.3pp）、Lマイクロスタンス 0%→21.4%が新規発生。L/R接地時間比 1.00→1.14。
- **base_height低下とペナルティ急増**: base_height +55%、orientation +67%、contact_no_vel +75%。姿勢維持能力の劣化。
- **特定関節の高周波振動増大**: L_hip_pitch 18.6%→30.3%（1.63x）、R_hip_yaw 16.6%→29.3%（1.76x）。
- **対称性改善 vs 安定性劣化のトレードオフ**: hip_pitch非対称度+28.5%→+9.8%と改善した一方、安定性指標は全面悪化。
- 報酬の正コンポーネント一律+27〜39%増とペナルティ+14〜75%増が相殺し総報酬-0.6%。「制御不能域」と判定。

**Lead（Round 2: 物理解釈・仮説構築）**:

- **主因果連鎖仮説**: Mirror Augmentation の obs/action 変換不整合 → 2048ミラー環境で矛盾勾配 → Yaw軸（最も繊細な制御軸）から破綻 → カスケード悪化。
- Yawドリフト4.77°/s（左回転）→ R側偏位 → L足代償的接地延長（51.7%→58.0%）の力学的整合を確認。
- exp008 V24型パターン: 「総報酬不変だが品質壊滅」。訓練環境内総報酬は品質指標として信頼できない。
- R_hip_yaw vel RMS -54%改善とR_hip_yaw高周波1.76x増大の矛盾は、ミラー変換でhip_yawの動きが抑制されつつ不安定化する兆候。
- 追加データ要求: ミラー/通常環境の報酬分離、訓練初期パターン、Yaw回転方向、ミラーインデックステーブル。

**DA（Round 3: 仮説検証データ）**:

- **gait_phase ミラーバグを確定的に発見**: `gait_phase` (obs [42:43]) は `sin(L脚位相 * 2π)`, `cos(L脚位相 * 2π)` だが、ミラー環境で「変更なし」。正しくは R脚位相に変換すべき: `sin((phase+0.5)*2π) = -sin(phase*2π)` → **符号反転が必要**。現状2048環境で位相180°ズレの勾配が生成されている。
- **Yaw方向の確認**: +92.4° = 左回転。L足接地偏重(58%)と物理的に整合。
- **hip_yawバイアスの矛盾がバグ仮説を支持**: V13のhip_yaw L-R差5.70°はV11の8.28°より小さいにもかかわらずYawドリフト9.2倍。hip_yaw非対称だけでは説明不能 → gait_phaseバグが主因。
- **訓練初期**: 急落→回復パターンなし。位相矛盾があっても粗い歩行獲得は可能だが、微細制御（Yaw）は破壊される。
- **ミラーインデックステーブル**: DOF系のL↔Rスワップ+符号反転は全て正しい。バグはgait_phase_sin/cosの未処理のみ。

**Lead（Round 4: 統合考察・改善案）**:

**悪化メカニズムの確定因果関係**:
1. gait_phase (obs[42:43]) のミラー変換欠落（根本原因）
2. → 2048ミラー環境で「左脚位相信号のまま」のgait_phaseが入力される（leg_phaseはL↔Rスワップ済みなのにgait_phaseは未処理 → 矛盾）
3. → ポリシーが「どの脚をいつ接地/遊脚させるか」の位相制御精度を喪失
4. → L足接地偏重(58%)・マイクロスタンス(21.4%)・Yawドリフト(4.77°/s) → カスケード悪化

**成功点**: Mirror Augmentationの本来目的（対称性の構造的保証）は機能。hip_pitch range非対称度 +28.5%→+9.8%（random）、-27.0%→+2.0%（FWD）と大幅改善。gait_phaseバグ修正後もこの効果は保持される見込み。

**改善案（推奨: 案A）**: `_mirror_observations()` 内で `obs[m, 42] *= -1; obs[m, 43] *= -1` の2行を追加するのみ。確定的なバグ修正であり、1変更1検証原則に適合。

### 成功点
- hip_pitch range非対称度の大幅改善: +28.5%→+9.8%（random）、-27.0%→+2.0%（FWD）。V10/V11/V12で3バージョン連続解決できなかった非対称問題に対し、Mirror Augmentationが構造的に有効であることを実証
- hip_yaw L-R差の改善: 5.57°→4.06°（random, -27.1%）、8.33°→5.78°（FWD, -30.6%）
- gait_phaseバグの発見により、Mirror Augmentationの正しい実装への道筋が明確化

### 課題
- gait_phase (obs[42:43]) のミラー変換欠落により、位相制御が崩壊しYawドリフト7.3倍悪化（FWD固定: 12.7°→92.4°/20s）
- L足接地偏重（51.7%→58.0%）とマイクロスタンス新規発生（21.4%）
- base_heightペナルティ+55%、orientationペナルティ+67%と全面的な姿勢安定性劣化
- 訓練環境内の総報酬（-0.6%）が品質劣化を検出できないexp008 V24型問題が再確認された

### 次バージョンへの提案
- `_mirror_observations()` 内で gait_phase_sin/cos (obs[42], obs[43]) の符号反転を追加する（2行のバグ修正のみ）。これにより位相制御の正常化とMirror Augmentationによる対称性保証を両立できる見込み
- 検証の優先指標: (1) Yawドリフト(FWD 20s) < 25.6°（V11水準）、(2) L/R接地時間比 < 1.05、(3) マイクロスタンス < 5%、(4) hip_pitch非対称度 < 10%維持

## まとめ

V13はV11ベースにMirror Augmentation（L↔R反転データ拡張）のみを追加した実験。対称性の構造的保証という本来目的は達成（hip_pitch非対称度 +28.5%→+9.8%）したが、`_mirror_observations()`内でgait_phase (obs[42:43]) のミラー変換が欠落していたため、2048ミラー環境で位相180°ズレの矛盾勾配が生成され、位相制御が崩壊。結果としてYawドリフト7.3倍悪化（FWD固定: 12.7°→92.4°/20s）、L足接地偏重、マイクロスタンス新規発生という壊滅的結果となった。バグ修正（gait_phase_sin/cosの符号反転2行追加）により、対称性改善と位相制御正常化の両立が期待できる。
