# V5: 訓練イテレーション数増加（収束不足の解消）

## 概要

V4のis_moving修正により報酬ランドスケープが構造的に変化したが、2000 iterationsでは収束不十分（Last 50 std 0.376、V3比+64.9%）であった。V5ではmax_iterationsを4000に増加し、現在の報酬構造での十分な収束を目指す。ジタバタ挙動（hip_yaw 3.62Hz振動）が未収束ポリシーの過渡的挙動であるかを検証する。

**目視評価結果**: グネグネした動きも不安定さも何も解消されなかった。V4と同様のジタバタ挙動が継続。

## 前バージョンからの改善事項

V4レポートの「次バージョンへの提案」に基づき、max_iterationsのみを変更する。

| パラメータ | V4値 | V5値 | 変更理由 |
|-----------|------|------|---------|
| max_iterations | 2000 | 4000 | is_moving修正による報酬ランドスケープの構造的変更に対し、2000 iterationsでは収束不十分（Last 50 std 0.376） |

## 設計詳細

### 変更の背景

V4のis_moving修正は「1パラメータのバグ修正」であるが、実効的にはコマンド空間の約50%（横方向寄りのコマンド）でsingle_foot_contact報酬が新たに有効化される**報酬ランドスケープの構造的変更**に相当する。

V4の学習曲線分析:
- Q3→Q4増分が-0.1（V3: +0.6）と鈍化
- Last 50 std 0.376（V3: 0.228、+64.9%）
- 報酬ランドスケープ変更後の新しい均衡点に未到達の可能性

### 1変更1検証原則

報酬パラメータ（action_rate scale増加等）の調整は、十分な収束を確認した後に実施すべきである。未収束状態でパラメータを変更すると、「パラメータの効果」と「収束の効果」を分離できない。

### 報酬構成

V4と同一の16項目。変更なし。

## 考察と改善案

### 目視評価所見

目視評価において以下が観察された:
- V4と同様のグネグネ・ジタバタした動き
- 不安定な歩行が継続
- 4000 iterationsでの追加学習による改善は認められず

### 対話による多角的分析

以下、データ分析専門エージェント（DA）とロボティクス・物理学俯瞰エージェント（Lead）の対話形式で考察を行う。

---

**DA**: まず、収束不足仮説を定量的に評価します。V5の学習曲線は、step 1342でピーク報酬59.09に達した後、残り2658 stepsで単調下降し最終57.12に着地しました。V4の最終報酬58.93を**下回っています**（-3.1%）。Last 50 stdはV4の0.376からさらに悪化し0.412（+9.5%）。Value function lossは5倍に増大（0.001→0.005）。これらは**学習が収束していないのではなく、報酬ランドスケープ上の安定均衡点が存在しない**ことを示唆します。

**Lead**: 非常に重要な知見です。通常、PPOで学習が収束不足の場合、追加イテレーションで報酬は少なくとも横ばいか微増します。V5ではピーク後に**下降**しているのは、ポリシーが**不安定な局所解の周辺を振動している**ことを示します。Value function lossの5倍増大は、状態-価値関数の推定が困難になっていることの直接証拠です。つまりポリシーの行動パターンが頻繁に変動しており、Criticがそれを追跡できていない。

**DA**: 報酬コンポーネントの詳細を見ると、正報酬合計はV4比+0.8%と微増していますが、ペナルティ合計は+15.0%悪化しています。特に3つのペナルティが突出しています:
1. **foot_lateral_velocity**: -0.0211→-0.0292（+38.4%）— V3→V4→V5で3バージョン連続悪化
2. **hip_yaw_pos**: -0.0251→-0.0330（+31.5%）— hip_yaw振動の直接反映
3. **action_rate**: -0.0390→-0.0479（+22.8%）— 高頻度制御の直接反映

penalty/positive ratioはV3: 0.071→V4: 0.077→V5: 0.088と単調増加しています。exp008のoverkill境界（0.186）にはまだ余裕がありますが、**ペナルティの増加トレンド自体が問題**です。

**Lead**: この3つのペナルティの増大パターンは、exp008 V18のカスケード悪化と同じ構造です。メカニズムを整理しましょう:

1. **is_moving修正**により、横方向コマンド時にもsingle_foot_contact報酬が有効化
2. **MLPが全方向で片足接地を最適化**しようとするが、横歩きでの片足バランスは前進より困難
3. **hip_yawで方向微調整**を行うことでバランスを補償（hip_yaw 3.62Hz→2.25-2.38Hz振動）
4. **hip_rollの高頻度制御**でバランスを維持（action RMS 2.6x増加）
5. 結果として**action_rate、hip_yaw_pos、foot_lateral_velocity全てが悪化**

つまり、ジタバタ挙動は「収束不足」ではなく「**全方向での片足バランス維持のためのMLP戦略**」です。追加イテレーションでは解消しません——むしろ、ポリシーはこの戦略を**より洗練させる方向に進化**し、ペナルティが増大します。

**DA**: その仮説を方向別データで検証します。興味深い非対称パターンがあります:

- **RGTが唯一改善**: 追従率19.7%→35.0%（+15.3pp）、Yaw -70.2°→+21.5°（大幅改善）、hip_pitch相関が正（同相=崩壊）→負（逆相=正常）に回復
- **BWDが最大悪化**: 追従率76.0%→51.0%（-25.0pp）、hip_pitch相関-0.299→-0.118（-60.5%）、一部seedでリセット発生
- **FWD微減**: 追従率71.8%→71.3%（-0.5pp）だがYaw -35.5°→+53.0°（符号反転+悪化）
- **LFT大幅悪化**: 追従率40.5%→26.0%（-14.5pp）

4方向平均の追従率は52.0%→45.8%（-6.2pp）と低下。しかしLFT/RGT差は20.8pp→9.0ppに縮小しており、**横方向の左右バランスは改善**しています。

**Lead**: RGT改善とBWD悪化の非対称パターンは、**RL stochasticityによるポリシーの再配分**として説明できます。V4のRGTは追従率19.7%で壊滅的でしたが、これはMLPの左右非対称性に起因していました。V5の追加イテレーションで、ポリシーが偶然的にRGT方向のバランス戦略を獲得した——その代償としてBWDの安定性が犠牲になった。

全方向追従の難しさは、**16項目の報酬構造が前進歩行用に設計されている**ことに起因します。action_rate scale -0.005はexp008の前進歩行で十分でしたが、全方向歩行では:
- 横歩き時のhip_yaw補償動作を抑制できない
- hip_rollの高頻度バランス制御にコストが不十分

**DA**: 振動分析で注目すべき変化があります。hip_yawの主周波数が3.62 Hz→2.25-2.38 Hzに下降しました。しかし、高周波成分比率（>2Hz）はL_hip_yaw 62.5%→85.6%（+37%）、R_hip_yaw 72.1%→79.3%（+10%）と**増大**しています。つまり、**周波数は下がったがエネルギーの大部分が依然として高周波帯に集中**している状態です。

hip_roll action RMSはさらに深刻で、V3→V4で約2.6x増加した後、V4→V5でさらに約2.6x増加しています。V3→V5での累積増加は約6.7x。これは**追加学習がhip_rollの高頻度制御戦略をさらに強化した**ことを示しています。

**Lead**: hip_yawの周波数シフト（3.62→2.25-2.38 Hz）は、追加学習によってポリシーが**より効率的な振動パターン**に遷移したことを示します。3.62 Hzは歩行周波数（1.25 Hz）の約3倍で3次高調波でしたが、2.25-2.38 Hzは約2倍の2次高調波に近い。これは歩行サイクルとのカップリングがより強い、つまり**構造的に組み込まれた振動**に進化しています。

hip_roll action RMSの6.7x累積増加は、ポリシーがhip_rollを**能動的なバランサーとして使用する度合いが指数的に増加**していることを意味します。これはaction_rate scale -0.005では制御不能なレベルです。

**DA**: 横方向揺れのR²が0.05前後で極めて低い点も重要です。exp008では0.7-0.9が典型的でした。これは横方向揺れがRoll角ではなく、**運動学的連鎖（hip_roll→knee→foot position）を通じたkinematic sway**で発生していることを示します。exp008 V17で観察されたR²崩壊（0.038）と同じパターンです。

**Lead**: 全体像をまとめます。V5は「収束不足仮説」を**定量的に棄却**しました:

1. **学習曲線の下降**: ピーク後に下降しており、「まだ収束していない」のではなく「安定均衡が存在しない」
2. **ペナルティの単調増加**: V3→V4→V5で3バージョン連続でratio増加。追加学習がペナルティ行動を強化
3. **振動の構造的進化**: hip_yawが3次→2次高調波に遷移、hip_roll action RMSが6.7x累積増加

根本原因は**action_rate scale -0.005の過小**です。この値はexp008の前進歩行で設計されたもので、全方向歩行で生じるhip_yaw/hip_rollの高頻度制御に対して桁違いに弱い。ポリシーは追加学習を重ねるほどこの高頻度戦略を洗練させ、振動を構造的に組み込んでいきます。

---

### 成功点

- **収束不足仮説の明確な棄却**: 4000 iterations（V4の2倍）でも改善せず、ピーク後に下降。ジタバタ挙動が構造的問題であることを定量的に確定
- **RGT追従率の改善**: 19.7%→35.0%（+15.3pp）。hip_pitch相関も正→負に回復
- **LFT/RGTバランス改善**: 追従率差20.8pp→9.0pp。横方向の左右バランスは改善

### 課題

1. **ジタバタ挙動の継続・悪化**: hip_yaw >2Hz成分比率がL 85.6%（V4: 62.5%）に増大。hip_roll action RMSは累積6.7x増加。目視で改善なし
2. **全方向追従率の低下**: 4方向平均45.8%（V4: 52.0%、-6.2pp）
3. **BWDの大幅悪化**: 追従率-25.0pp、hip_pitch相関-60.5%。一部seedでリセット発生
4. **Yawドリフトの全方向悪化**: ランダム評価+28.27°（V4: +3.35°）、FWD符号反転+53.0°
5. **報酬の逆転減少**: 最終報酬57.12（V4: 58.93）。追加学習が報酬を悪化
6. **ペナルティのトレンド悪化**: ratio 0.071→0.077→0.088。action_rate、hip_yaw_pos、foot_lateral_velocityが3バージョン連続増加

### 次バージョンへの提案

**推奨（V6）: action_rate scale増加 -0.005 → -0.01（2倍）**

根拠:
- V4-V5のジタバタ挙動の直接原因はhip_yaw/hip_rollの高頻度制御であり、action_rate penaltyがこれを抑制できていない
- V5のaction_rate penalty -0.0479はV4から+22.8%増大しており、高頻度制御のコスト不足が明確
- action_rate scale -0.005はexp008の前進歩行で設計された値であり、全方向歩行には不十分
- 保守的に2倍（-0.005→-0.01）で試行し、exp008 V5-V18で確認されたoverkillリスクを回避
- 1変更1検証原則に基づき、action_rate scaleのみを変更

期待効果:
- hip_yaw >2Hz振動の抑制
- hip_roll action RMSの低減
- 全体的なジタバタ挙動の緩和

リスク:
- ペナルティ増加によるoverkill（ただしratio 0.088はoverkill境界0.186の半分以下であり、余裕あり）
- 滑らかな制御を強制することで追従率が低下する可能性

**V7以降の候補**:
- **action_rate scale -0.01→-0.02**: V6で不十分な場合の追加増加
- **gait_frequency引き上げ**: exp008 V20で実証された多指標同時改善効果。全方向歩行でも適用可能
- **カリキュラム学習**: 前進→前後→全方向の段階的コマンド範囲拡大

## まとめ

exp009 V5はmax_iterationsを2000→4000に増加し、V4のジタバタ挙動が収束不足に起因するかを検証した。結果、**収束不足仮説は定量的に棄却**された:

1. **報酬がピーク後に下降**（step 1342でピーク59.09、最終57.12）。V4の最終報酬58.93を-3.1%下回る
2. **Last 50 stdが+9.5%悪化**（0.376→0.412）。収束していないのではなく、安定均衡が存在しない
3. **ペナルティが+15.0%増大**。action_rate +22.8%、hip_yaw_pos +31.5%、foot_lateral_velocity +38.4%
4. **hip_yaw >2Hz成分比率が増大**（L 62.5%→85.6%）。振動が3次→2次高調波に構造的に進化
5. **hip_roll action RMSが約2.6x増加**（V3→V5で累積6.7x）。高頻度バランス戦略の強化

方向別評価ではRGTが唯一改善（追従率+15.3pp）したがBWDが-25.0pp、LFTが-14.5ppと悪化。全4方向平均追従率は45.8%（V4: 52.0%、-6.2pp）。

根本原因は**action_rate scale -0.005がexp008前進歩行用の設計であり、全方向歩行の高頻度振動を抑制できない**ことにある。V6ではaction_rate scaleの2倍増（-0.005→-0.01）を推奨する。

## 備考

- 本バージョンの変更はmax_iterationsのみ。報酬構成・環境設定・コマンド設定はV4と完全同一
- V4レポートの推奨（max_iterations 4000）に基づく変更
- 1変更1検証原則に基づき、iteration増加のみを変更
- 収束不足仮説の棄却により、今後のバージョンでは**報酬パラメータの調整**に焦点を移す
