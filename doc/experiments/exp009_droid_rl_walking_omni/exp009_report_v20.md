# V20: V19ベース + hip_roll外向きPDクランプ導入

## 概要

V19をベースに、hip_roll外向きPDターゲットクランプ（`hip_roll_outward_limit = 0.349 rad`、20°）を導入する。V19でYaw追従能力を獲得したが、R側バランス代償動作が過剰拡大（R_hip_roll下限-26.8°、V18比-44%拡大）し、横方向増幅率が0.67x→1.78xに上昇、散発的な全身傾き崩しイベントが確認された。hip_roll外向き制限で崩しイベントの物理的上限を設定し、Yaw追従能力を維持しつつR側代償動作の過剰拡大を抑制することを目指す。

## 前バージョンからの改善事項

V19レポートの「次バージョンへの提案」に基づく改善:

> V19をベースに、R_hip_roll range制限（[-20°, 20°]）を導入し、崩しイベントの物理的上限を設定する。Yaw追従能力は維持しつつ、R側バランス代償動作の過剰拡大を抑制。効果確認後、num_envs増加（4096→6144）による学習量増加を第2段階として検討する。

| パラメータ | V19値 | V20値 | 変更理由 |
|-----------|-------|-------|---------|
| hip_roll_outward_limit | 未設定（制限なし） | **0.349 rad（20°）** | V19でR_hip_roll下限-26.8°まで拡大し崩しイベントが発生。20°で物理的上限を設定し代償動作の過剰拡大を抑制 |

## 設計詳細

### hip_roll外向きPDクランプの設計

- **クランプ値**: `hip_roll_outward_limit = 0.349 rad`（20° = 20 × π/180）
- **適用方式**: 全位相クランプ（`hip_roll_clamp_stance_only = False`、デフォルト）
  - 接地相・遊脚相の両方でPDターゲットを制限
  - 崩しイベントは位相に依存せず発生するため、全位相で制限が適切
- **適用方向**: 外向き方向のみ制限
  - 左脚: PDターゲット max = 0.349 rad（20°外向き）
  - 右脚: PDターゲット min = -0.349 rad（-20° = 20°外向き）
- **内向き制限**: `hip_roll_inward_limit = -0.05 rad`（V19から継承、変更なし）

### 20°の根拠

- V19のR_hip_roll下限: -26.8°（崩しイベントの痕跡）
- V18のR_hip_roll下限: -18.6°（崩しイベントなし）
- 20°は両者の中間値であり、V18水準の安定性を保ちつつYaw制御の自由度を確保する妥協点

### リスク

- hip_roll外向き制限によりバランス代償動作の自由度が低下し、追従率が悪化する可能性
- 制限が厳しすぎる場合、特にYaw旋回時に必要なhip_roll動作が制約され、Yaw追従能力が低下する可能性
- 全位相クランプのため、遊脚相の脚振り出し動作にも影響する可能性

## 考察と改善案

### 目視評価所見

目視評価において以下が観察された:
- 内股の症状が再発している
- 連動して安定性も大きく損なわれた
- PDクランプは不適切かもしれない
- 先にyaw角の操縦を実装した方が良い可能性がある

### 対話による多角的分析

以下、データ分析専門エージェント（DA）とロボティクス・物理学俯瞰エージェント（Lead）の対話形式で考察を行う。

---

**DA（Round 1: 初期定量分析）**:

- **バランスストローク壊滅**: L脚 6.5°→1.3°（-80.8%）、R脚 7.6°→0.5°（-93.5%）。hip_rollが内股方向に約3倍拡大（L: -8.1°→-22.9°、R: +7.8°→+24.3°）し、動的バランス余裕がほぼ消失。
- **後退追従率の大幅劣化**: BWD追従率 73.6%→37.5%（-49.1%pt）。接地比L/R=6.14と極端な左右非対称が発生。方向間バランスstdが13.3%→27.7%に悪化。
- **歩行パターンの崩壊**: L_hip_pitch主周波数が1.50Hz→0.12Hzへ低下し、周期的歩行から逸脱。knee_pitch相関が-0.321→0.006となり膝連動が消失。step_length報酬が-55.9%。
- **Pitch安定性の大幅改善**: Pitch std 2.20°→0.96°（-56.4%）。orientation/ang_vel_xyペナルティも半減以下。体幹前後揺れは明確に抑制。
- **Y方向揺れ抑制**: Y速度std 0.2209→0.1017（-54.0%）。増幅率1.78x→0.46xに低下。ただし関節可動域の固着による見かけ上の改善の可能性あり。
- **姿勢安定 vs 移動能力のトレードオフ**: 総報酬+5.4%だがtracking_lin_vel -19.2%、step_length -55.9%。PDクランプによる関節拘束が姿勢ペナルティを減らす一方、歩行自由度を喪失。報酬増はペナルティ減による見かけ上の改善。
- **Yaw指標（固定コマンド評価のみ）**: FWD -3.5°→-13.7°（悪化）、RGT +15.3°→-111.2°±30.5°（壊滅的悪化、旋回に近い挙動）。全方向で悪化。

**Lead（Round 2: 物理解釈・仮説構築）**:

**主因果連鎖**: PDクランプ(20°外向き制限) → hip_roll可動域の強制収縮 → 内股固着(L:-22.9°, R:+24.3°) → バランスストローク消失 → 動的バランス余裕の喪失 → 歩行パターン崩壊 → 移動能力の壊滅的劣化。

二脚歩行においてhip_rollのバランスストロークは動的バランスの根幹である。V20のPDクランプは外向き20°に制限したが、V19でのR_hip_roll実動域が-26.8°に達していたことから、制限値がバランス動作の必要域を侵食した。exp007 V7-V10のカスケード悪化パターン（ペナルティ過剰→静止ポリシー）と同型。

**仮説**: PDクランプ(20°)がV19のバランス動作必要域(~26.8°)を6.8°分削り、ポリシーがバランス不能と判断して歩行戦略を放棄した。penalty/positive ratioはoverkill域に該当。

**DA（Round 3: 仮説検証データ）**:

1. **PDクランプ飽和頻度**: ポリシーが外向き修正を学習段階で放棄（L外向き最大-9.4°、R外向き最大+15.6°で20°クランプに到達すらしていない）。クランプ値でなく、学習段階で歩行戦略自体が変質。
2. **体高低下**: V19=0.2985m → V20=0.2598m（-3.87cm、-13.0%）。変動1.55倍増。内股固着により重心が下がり、腰折れ姿勢で歩行。
3. **膝過屈曲**: L_knee=-56.6°→-88.7°（-32.1°深化）、R_knee=-64.3°→-82.1°（-17.8°深化）。hip_roll制限→内股固着→重心低下→膝過屈曲の因果連鎖を裏付け。
4. **総括**: 4項目すべてがLead仮説を裏付け。

**Lead（Round 4: 統合考察・改善案）**:

#### 悪化メカニズムの因果関係

PDクランプ（20°外向き制限）がV19で獲得された歩行戦略の前提条件を学習初期段階で破壊した。核心はPDクランプが「ハードウォール」ではなく「勾配情報の遮断」として機能した点にある。PPOの勾配更新において外向き方向のアクションに対するリターンの勾配が切断され、ポリシーはクランプ境界に到達する前の段階で外向き動作自体を探索対象から除外した。

```
PDクランプ(20°) → hip_roll外向き修正放棄（L最大-9.4°, R最大+15.6°: 20°に到達せず）
  → 内股固着（L:-22.9°, R:+24.3°） → バランスストローク消失（L -80.8%, R -93.5%）
  → 重心低下（-13.0%）+ 膝過屈曲（L:-32.1°, R:-17.8°深化）
  → 歩行パターン崩壊（主周波数1.50Hz→0.12Hz）
  → BWD追従率半減（73.6%→37.5%）, Yaw壊滅（RGT:-111.2°）
```

#### 成功点の評価

Pitch安定性改善（-56.4%）およびY方向揺れ抑制（-54.0%）は、歩行能力の犠牲による見かけ上の改善であり、実質的な改善とは見なせない。総報酬+5.4%はペナルティ回避型の報酬ハッキングの一種。

#### 改善案

- **案A（PDクランプ撤廃）**: V19ベースへの純粋ロールバック。確実にV19の歩行能力を復元。リスクゼロに近い。
- **案B（Yaw追従強化）**: V19ベースでtracking_ang_velスケールを引き上げ。tracking_lin_velとの報酬競合リスクあり。
- **案C（ソフト制約）**: PDクランプの代わりに報酬ペナルティでhip_roll外向き制約。勾配情報が保持される。
- **案D（Yaw操縦機能の先行実装）**: 人間の所感に基づくYaw角速度コマンドの外部入力インターフェース実装。

**推奨**: 案A（V19ベースへのロールバック）を第1段階として実施。V19の歩行能力を確実に復元した上で、第2段階として案C（ソフト制約）を検証する。Yaw操縦のインターフェース実装（案D）はポリシー変更を必要としないため、ロールバックと並行して進行可能。

---

### 成功点
- 総報酬+5.4%（ただしペナルティ回避型の見かけ上の改善）
- Pitch std -56.4%、Y速度std -54.0%（歩行能力犠牲の副産物）
- PDクランプの不適切性が実証され、「PDターゲットクランプはhip_rollバランス制御に対して過剰に侵襲的」という重要な知見を獲得

### 課題
- 内股固着の再発（hip_roll L:-22.9°、R:+24.3°）とバランスストロークの壊滅（L -80.8%、R -93.5%）
- BWD追従率の大幅劣化（73.6%→37.5%）、方向間バランスの悪化（std 13.3%→27.7%）
- 歩行パターンの崩壊（L_hip_pitch主周波数 1.50Hz→0.12Hz）
- Yaw制御の壊滅的悪化（RGT方向 -111.2°）
- 体高低下（-13.0%）と膝過屈曲（L:-32.1°、R:-17.8°深化）

### 次バージョンへの提案

V19をベースにPDクランプを撤廃（= V19完全復帰）し、V19の歩行能力を復元する。V20の結果から得られた知見「PDターゲットクランプはhip_rollに対して不適切な介入手法であり、勾配情報の遮断を通じて歩行戦略自体を変質させる」に基づく。R側バランス代償動作の過剰拡大には、第2段階としてソフト制約（報酬ペナルティ）での対処を検討する。

なお、Yaw操縦のインターフェース（ゲームパッド右スティック→`ang_vel_yaw`、`--command VX VY VYAW`による固定コマンド指定）は既に実装済みであり、新規実装は不要である。改善が必要なのはポリシー側のYaw追従精度であり、V19ロールバック後の次ステップとして以下を検討する:
- `tracking_ang_vel`スケールの引き上げ（現行0.5 → 0.8-1.0）によるYaw追従学習の強化
- Yaw旋回専用の固定コマンド評価（`--command 0 0 0.3`等）を方向別評価プロトコルに追加し、Yaw追従精度を定量的に追跡する

## まとめ

V20ではV19をベースにhip_roll外向きPDクランプ（20°）を導入したが、期待とは逆にポリシーの歩行戦略を根本から変質させた。PDクランプは物理的上限としてではなく学習時の勾配情報遮断として機能し、ポリシーが外向きバランス動作の探索を放棄。内股固着→バランスストローク消失→体高低下→膝過屈曲→歩行パターン崩壊というカスケード悪化が発生した。総報酬+5.4%はペナルティ回避型の見かけ上の改善に過ぎず、実質的な歩行能力は大幅に劣化した。「PDターゲットクランプはhip_rollバランス制御に対して過剰に侵襲的である」という重要な知見が得られた。次バージョンではV19ベースにロールバックし、ソフト制約での代替アプローチを検討する。
