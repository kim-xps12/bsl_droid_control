# V3: 訓練イテレーション数増加

## 概要

V2で完了したvelocity_deficit scale修正の上に、訓練イテレーション数を500から2000に4倍増加する。V2ではLast 50 std=0.398と収束が未完了であり、追加学習によるポリシーの成熟を目指す。

## 前バージョンからの改善事項

V2レポートの「次バージョンへの提案」に基づき、max_iterationsの増加のみを実施する。

| パラメータ | V2値 | V3値 | 変更理由 |
|-----------|------|------|---------|
| max_iterations | 500 | 2000 | V2のLast 50 std=0.398（閾値0.1超過）。Q3→Q4で+7.43の増分が継続しており学習は進行中 |

報酬設定・コマンド設定・環境設定は**全てV2と同一**。

## 設計詳細

変更はmax_iterationsのみのため、報酬構成に変更なし。

- 報酬項目数: 16（V2と同じ）
- velocity_deficit scale: -0.1（V2で修正済み）
- コマンド範囲: X [-0.3, 0.3], Y [-0.3, 0.3], Yaw [0, 0]
- gait_frequency: 1.2 Hz
- hip_roll_inward_limit: -0.05 rad（PDクランプ維持）

1変更1検証原則に基づき、iteration増加の純粋な効果を検証する。

## 実行コマンド

```bash
cd rl_ws
uv run python biped_walking/train/droid_train_omni_v3.py --max_iterations 2000
```

## 評価コマンド

```bash
cd rl_ws
# ランダムコマンド評価（従来）
uv run python biped_walking/biped_eval.py -e droid-walking-omni-v3 --no-viewer --duration 10
# 方向別評価（Forward例、4方向x3試行の詳細はexp009_commands.md Section 6参照）
uv run python biped_walking/biped_eval.py -e droid-walking-omni-v3 --no-viewer --duration 20 --csv --command 0.3 0.0 0.0 --seed 1
```

## 考察と改善案

### 成功点

- **報酬改善**: 59.89（V2: 56.21, +6.5%）。4倍のiteration増加が報酬改善に寄与
- **収束改善**: Last 50 std 0.228（V2: 0.398, -42.7%）。訓練安定性が向上
- **FWD/BWD追従**: 前後方向では78%の追従率を達成。速度追従そのものは良好
- **ratio健全域維持**: 0.071（V2: 0.080）。ペナルティバランスは健全
- **正報酬全項目改善**: 6項目全てが改善。tracking_lin_vel +9.0%、tracking_ang_vel +10.7%
- **ペナルティ7/11改善**: ang_vel_xy -27%、orientation -35.1%、swing_contact_penalty -64.6%
- **L/R接地比改善**: 45.2%/49.0%（V2: 41.5%/51.5%）。対称性が向上

### 課題

1. **学習飽和**: Q3→Q4の増分が+0.6（V2のQ3→Q4: +7.43）。2000 iterationsで学習がほぼ飽和。追加iteration単体での改善は限定的

2. **base_height +88.9%**: 最大の悪化項目。base_height 0.282m（V2: 0.262m, +7.7%）。ポリシーが重心を上げる戦略を採用

3. **Action RMS +216%**: 1.809→5.718。関節動作の振幅が激増。DOF vel RMS +37.4%、Total DOF range +40.8%も伴う

4. **R²崩壊の持続**: 0.042（V2: 0.147, -71.7%）。横方向揺れのわずか4.2%がRollで説明可能。lateral std 84.3mm（V2: 72.7mm, +16.0%）

5. **LFT/RGT追従非対称**: 41.5%/27.0%（差14.5%）。右歩きの追従率が全方向中最低

6. **方向別Yawドリフト**: Forward -57.2°、Left +101.7°。特定方向で深刻な方向逸脱が発生。ランダム評価（+10.60°）では方向依存の問題が隠れている

7. **Backward転倒**: 3試行全てで転倒発生。後退歩行の安定性が不足

8. **hip_pitch相関の劣化**: -0.312（V2: -0.389, -19.8%）。特にLeft方向では+0.055（交互歩行パターン崩壊）

### ポリシー戦略変化のメカニズム分析

V3ポリシーは「重心引き上げ戦略」を採用した。因果チェーンは以下の通り:

1. **2000 iterationsの学習で発見**: base_heightを上げると、orientation（-35.1%）・ang_vel_xy（-27%）・lin_vel_z（-21.9%）が改善する
2. **base_height scale -5.0が不十分**: 重心引き上げのコスト（base_height penalty +88.9%）よりも、orientation系列の改善が大きい
3. **高重心→大きなaction必要**: 不安定な姿勢維持のため、Action RMS +216%、DOF vel RMS +37.4%が発生
4. **大きなaction→kinematic chain効果**: 関節可動域拡大（+40.8%）により、hip_roll以外の関節が横方向変位を生成→R²崩壊
5. **外向き修正の放棄**: V3ではL/R両方とも外向き最大値が±2°（V2: L=+11.4°）。バランス制御戦略が変化

base_height_target 0.20mに対して実際の高さが0.282mであり、0.082mの乖離が存在する。この乖離がペナルティ構造全体を歪めている。

### 方向別性能の非対称性

- **FWD/BWD対称性良好（78%/78%）**: 前後方向はhip_pitchの対称運動で実現できるため、L/R非対称性の影響を受けにくい
- **LFT > RGT（41.5% vs 27.0%）**: MLPは対称解を保証しない（exp008 V8の教訓）。Left方向の学習がRight方向より進行した可能性
- **Left方向のhip_pitch corr ≈ 0**: 横歩きではhip_pitchの交互パターンが不要なため構造的にcorrが低い
- **FWDのYawドリフト（-57.2°）はR²崩壊と関連**: kinematic chain効果で横方向への逸脱が制御不能

### サーベイ知見との照合

- **Section 7.3.1（カリキュラム学習）**: Q3→Q4の鈍化（+0.6）は、単純なiteration追加が全方向タスクの複雑度に対して限界に達していることを示す。段階的コマンド範囲拡大が有効な可能性がある
- **Section 2.2.2（Scale Tuning）**: base_height +88.9%、hip_yaw_pos +77.6%の悪化は新たなペナルティボトルネック。scale調整（base_height_targetの引き上げ）で対応可能な可能性がある。ただしscale強化はoverkillリスク（V5/V18パターン）がある
- **Section 2.1.1（Gaussian Reward）**: tracking_lin_vel +9.0%の改善にもかかわらず、方向別ではLeft 41.5%、Right 27.0%の追従率。Gaussian報酬は「平均的に近い」ことを報酬化するが「全方向で均一」を保証しない
- **Section 8.6（静止ポリシー）**: 静止ポリシー問題は発生していない。velocity_deficit -0.1 + tracking_lin_vel 1.5の組み合わせが有効に機能

### 次バージョンへの提案

**推奨（V4）: max_iterations 2000 → 4000**

根拠:
- Last 50 std=0.228はexp008の収束水準（0.07-0.12）に対して明確に未収束。2000 iterationsでは全方向タスクの複雑度に対して不十分
- Q3→Q4の増分+0.6は鈍化しているが、std=0.228は依然として大きな変動を示しており、収束余地がある
- V3で観察されたbase_height +88.9%、Action RMS +216%等の問題は未収束ポリシーの過渡的挙動である可能性がある。報酬パラメータを変更する前に、まず現在の設定で十分に収束させるべき
- 1変更1検証原則に基づき、iteration増加のみを変更。収束後に残存する問題を正しく同定する

**V5以降の候補**（V4収束後の残存問題に応じて選択）:
- base_height scale -5.0 → -10.0（exp008水準に復帰）: 収束後もbase_heightが高い場合
- action_rate scale増加（-0.005→-0.01）: Action RMS過大が持続する場合
- カリキュラム学習（前進のみ→前後→全方向の段階的拡大）: 方向別追従率の非対称が持続する場合

## まとめ

exp009 V3はV2のmax_iterationsを500→2000に4倍増加した。訓練報酬は59.89（V2: 56.21, +6.5%）、Last 50 stdは0.228（V2: 0.398, -42.7%）に改善したが、目標0.1は未達。Q3→Q4の増分が+0.6に鈍化しており、2000 iterationsで学習が飽和に近い。

正報酬は全6項目が改善し、ペナルティ11項目中7項目が改善。ratio 0.071は健全域。一方、ポリシーは「重心引き上げ戦略」を採用し、base_height +88.9%（0.262→0.282m）、Action RMS +216%（1.809→5.718）が発生。R²は0.042に崩壊が進行。

方向別評価では、FWD/BWD追従率は78%で対称的だが、LFT 41.5%/RGT 27.0%と横歩きの追従が不十分。Forward方向のYawドリフト-57.2°、Left方向+101.7°が深刻。Backwardでは全試行で転倒が発生した。

2000 iterationsでは全方向歩行タスクの収束に不十分であり、V3で観察された問題（base_height上昇、Action RMS増大等）は未収束ポリシーの過渡的挙動の可能性がある。V4ではmax_iterationsを4000に増加し、現在の報酬構造での十分な収束を目指す。

## 備考

- 評価時のコマンドはランダムサンプリングのため、各評価実行で指標値が変動する
- V2とV3はmax_iterations以外同一設定のため、差分は純粋に追加学習の効果を反映する
- V2の評価値は本レポート作成時に再実行した値を使用（V2レポート記載値とは異なる評価セッション）
- 方向別評価のseed 1とseed 2は決定論的シミュレーションにより同一結果を返した。実効的に2ユニークシード（seed 1/2共通、seed 3）での評価となっている
