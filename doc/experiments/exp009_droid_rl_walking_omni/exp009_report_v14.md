# V14: gait_phaseミラー変換バグ修正

## 概要

V13をベースに、`_mirror_observations()`内のgait_phase (obs[42:44]) ミラー変換バグを修正する。V13ではMirror Augmentationにより対称性の構造的保証を実証したが（hip_pitch非対称度+28.5%→+9.8%）、gait_phaseのミラー変換が欠落しており、2048ミラー環境で位相180°ズレの矛盾勾配が生成され、位相制御が崩壊した。本バージョンではこの確定的バグを2行の修正で解消し、対称性改善と位相制御正常化の両立を目指す。

## 前バージョンからの改善事項

V13レポートの「次バージョンへの提案」に基づく改善:

> `_mirror_observations()` 内で gait_phase_sin/cos (obs[42], obs[43]) の符号反転を追加する（2行のバグ修正のみ）。これにより位相制御の正常化とMirror Augmentationによる対称性保証を両立できる見込み

| パラメータ | V13値 | V14値 | 変更理由 |
|-----------|-------|-------|---------|
| gait_phaseミラー変換 | 変更なし（バグ） | **sin/cos符号反転** | gait_phase = sin(L脚位相×2π), cos(L脚位相×2π)。ミラー時はR脚位相 = L脚位相+0.5 → sin((φ+0.5)×2π) = -sin(φ×2π)、cos同様。2048ミラー環境で位相180°ズレの矛盾勾配を解消 |

### バグ修正の詳細

`droid_env_unitree.py` の `_mirror_observations()` に以下の2行を追加:

```python
# gait_phase: sin/cos符号反転（V14バグ修正）
# gait_phase = sin(L脚位相*2π), cos(L脚位相*2π)
# ミラー時はR脚位相 = L脚位相+0.5 → sin((φ+0.5)*2π) = -sin(φ*2π), cos同様
obs[m, 42] *= -1
obs[m, 43] *= -1
```

## 考察と改善案

### 目視評価所見

目視評価において以下が観察された:
- かなり安定してきた（全体的な姿勢安定性の向上を確認）
- 前進中に右足だけが大きく持ち上がる動作が依然として存在
- 徐々にyaw driftが発生している
- 左右の並進がぎこちない

### 対話による多角的分析

以下、データ分析専門エージェント（DA）とロボティクス・物理学俯瞰エージェント（Lead）の対話形式で考察を行う。

---

**DA（Round 1: 初期定量分析）**:

- **hip_pitch非対称度の方向反転と増大**: +9.8% → -57.7%（ランダム評価）。V13ではほぼ対称だったものが、V14ではR側が大幅に優位に。R_hip_pitch可動域1.106 rad vs L 0.611 rad（R/L = 1.81倍）、FFT振幅もR=49.27 vs L=22.79（2.16倍）。
- **姿勢安定性の大幅改善**: Pitch std 2.32° → 1.38°（-40.5%）、Roll std 4.60° → 3.48°（-24.3%）。マイクロスタンス21.4% → 0%で完全解消。
- **FWD追従率の改善とLFT/RGTの低下**: FWD 81.7% → 93.2%（+11.5pp）、BWD 89.1% → 83.8%（-5.3pp）。LFT 39.8%→31.1%、RGT 50.5%→41.4%と低下。
- **横方向増幅率の反転**: 0.44x → 1.46x。Roll自体は減少したが横方向detrended stdが74.5mm → 104.4mmへ増大。
- **hip_yaw振動特性の変化**: L_hip_yaw vel RMSが0.532→0.860（+61.7%）に悪化。L/R比=1.994で左側に偏移。
- **所感との対応**: 「右足だけが大きく持ち上がる」はR_hip_pitch可動域がLの1.81倍で裏付け。「yaw drift」はFWD固定評価で+29.5°で裏付け。「左右並進がぎこちない」はLFT/RGT追従率31.1%/41.4%と横方向増幅率1.46xで裏付け。
- **Yaw信頼性チェック**: ランダムコマンド評価のYaw値はコマンド切替の影響で信頼性低。固定コマンド評価ではFWD+29.5°/LFT+31.0°で悪化、BWD-0.4°/RGT+0.8°で改善と方向依存性が強い。

**Lead（Round 2: 物理解釈・仮説構築）**:

**仮説**: gait_phaseミラー修正が位相制御を正常化した結果、Mirror Augmentationが左右脚の役割分化を促進し、「R脚=推進担当（大振幅swing）、L脚=支持担当（小振幅swing）」という非対称最適解に収束した。

因果連鎖: gait_phase符号反転修正 → ミラー環境で位相制御が正常化 → ポリシーが位相依存スイング制御を獲得（hip_pitch相関 -0.194→-0.502で実証） → 位相制御の改善が特定脚への役割固定を加速 → R_hip_pitch可動域がLの1.81倍に拡大。

マイクロスタンス完全解消・姿勢安定性改善は位相制御正常化の直接的恩恵であり、非対称性悪化との独立した改善と判断。penalty/positive ratioは健全域。

**DA（Round 3: 仮説検証データ）**:

1. **振幅差 vs オフセット差**: stance mean R=0.674 vs L=0.685（差0.6°）でオフセットはほぼゼロ。swing range R=0.867 vs L=0.589（R/L=1.47倍）。非対称の主因は**振幅差**であり、Lead仮説を裏付け。
2. **hip_yaw相別分析**: L hip_yawがswing時+0.141 rad（8.1°外旋）、stance時-0.030 rad（相間差9.8°）。R hip_yawは相間差1.8°で安定。**Yawドリフトの駆動源はL脚swing時の外旋**。
3. **横方向FFT**: 0.06Hz成分（mag=48.0）が歩行周波数1.50Hz成分（mag=10.2）の4.7倍。横方向変位は**低周波ドリフトが支配的**（CoM制御の問題）。
4. **BWD非対称方向依存性**: BWDでもR>L（V14: -64.8%, V13: -52.2%）で同方向。**R脚優位は方向依存ではなくポリシー固有の構造的偏り**。V14は既存の偏りを13pp増幅したのみ。

**Lead（Round 4: 統合考察・改善案）**:

**悪化メカニズムの因果連鎖**:
gait_phaseバグ修正 → 位相制御の正常化 + Mirror Augmentation制約 + 報酬構造がR脚大振幅を容認 → R脚推進/L脚支持への機能分化（swing range R/L=1.47倍） → L脚swing時のhip_yaw外旋（8.1°、相間差9.8°）→ 低周波横方向ドリフト（0.06Hz支配） → FWD方向Yaw+29.5°、横方向増幅率1.46x。

**位置づけ**: V14は「位相制御の崩壊が隠していた次のレベルの課題」が露出したものであり、開発プロセスとして健全。

**改善案の比較**:
- 案A: swing振幅非対称ペナルティ追加（直接的だが報酬複雑化リスク）
- 案B: foot_lateral_velocityスケール増大（LFT/RGT追従率さらに低下のリスク）
- 案C: **横方向コマンド速度範囲の拡大**（lin_vel_y_range [-0.20,0.20]→[-0.30,0.30]）（推奨）
- 案D: hip_yaw_posスケール増大（相に非依存な介入で精度低）

**推奨: 案C** — LFT/RGT追従率の低さが最大の課題であり、コマンド範囲がFWD/BWDの2/3に制限されていることが横方向移動の訓練データ不足の構造的原因。報酬構成に手を加えずコマンド範囲のみの変更で、V14の成功基盤（姿勢安定性・FWD追従率）を活かしつつ最も弱い横方向を強化する戦略。

---

### 成功点
- **姿勢安定性の大幅改善**: Roll std -24.3%、Pitch std -40.5%、マイクロスタンス完全解消（21.4%→0%）
- **位相制御の正常化**: hip_pitch相関が-0.194→-0.502に大幅改善（V11の-0.288をも上回る）
- **FWD追従率の改善**: 81.7%→93.2%（+11.5pp、V11の86.5%をも上回る）
- **L/R接地時間比の正常化**: 1.14→0.99（ほぼ完全対称）
- **BWD/RGT方向のYawドリフト解消**: BWD -0.4°、RGT +0.8°とほぼゼロに
- **hip_yaw L-R差の改善**: 4.06°→3.18°（-21.7%）

### 課題
- **hip_pitch非対称度の方向反転と増大**: +9.8%→-57.7%（R脚swing振幅がLの1.47倍）
- **FWD方向のYawドリフト**: +29.5°（V13の92.4°からは68%改善だがV11の12.7°の2.3倍）
- **LFT/RGT追従率の低さ**: LFT 31.1%、RGT 41.4%（方向間std 26.1%）
- **横方向並進のぎこちなさ**: 横方向detrended std 104.4mm、低周波ドリフト支配（0.06Hz成分）
- **L脚swing時hip_yaw外旋**: 8.1°の外旋がYawドリフトの駆動源

### 次バージョンへの提案
- `lin_vel_y_range` を [-0.20, 0.20] から [-0.30, 0.30] に拡大し、横方向コマンド速度範囲をFWD/BWDと統一する。これにより訓練時の横方向移動経験を増やし、LFT/RGT追従率の改善と横方向CoM制御の向上を期待する。報酬構成は変更しない。

## まとめ

V14はgait_phaseミラー変換バグの2行修正により、位相制御の正常化を達成した。姿勢安定性（Roll/Pitch std）、マイクロスタンス完全解消、hip_pitch相関、FWD追従率、L/R接地時間比で大幅な改善を実現し、gait_phaseバグ修正の有効性を実証した。一方、正常化された位相制御の下で「R脚=大振幅推進、L脚=小振幅支持」という非対称パターンが顕在化し、FWD方向のYawドリフト（+29.5°）と横方向追従率の低さ（LFT 31.1%/RGT 41.4%）が次の課題として浮上した。これは「位相制御の崩壊が隠していた次レベルの課題」の露出であり、開発プロセスとして健全な進行である。次バージョンでは横方向コマンド速度範囲の拡大による横方向移動能力の底上げを提案する。
