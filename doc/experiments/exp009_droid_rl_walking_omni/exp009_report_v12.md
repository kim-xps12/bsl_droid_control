# V12: 速度二乗EMA対称性報酬の追加（symmetry_vel_ema）

## 概要

V10をベースに、新規報酬関数`symmetry_vel_ema`（速度二乗EMAによる左右運動エネルギー均等性報酬）を追加する。V10で顕在化した左右非対称性（hip_pitch非対称度-15.1%、右脚過剰挙上）に対し、交互歩行の位相差を許容しつつ振幅対称性を促進することを狙う。V11（案A: vy範囲±0.20）とは独立した並行検証である。

## 前バージョンからの改善事項

V10レポートの「次バージョンへの提案」改善案Bに基づき、対称化報酬を追加。

ただし、既存の`_reward_symmetry`（瞬時関節位置対称性）は交互歩行で左右の関節角度が異なる瞬間に同値を要求するため、同位相同期（うさぎジャンプ）を誘導するリスクがある。そこで瞬時位置ではなく、**時間平均運動エネルギーの左右均等性**を評価する新報酬関数`_reward_symmetry_vel_ema`を設計・実装した。

| パラメータ | V10値 | V12値 | 変更理由 |
|-----------|-------|-------|---------|
| symmetry_vel_ema (reward_scale) | 0（未使用） | **0.3** | V10で顕在化した左右非対称性を位相不問で改善 |
| symmetry_ema_alpha | — | **0.03** | EMA時定数≈1歩行周期（50Hz制御、1.5Hz歩容で約33ステップ） |
| symmetry_ema_sigma | — | **2.0** | ガウシアン報酬の幅パラメータ |

### V12で速度二乗EMA対称性報酬を選定した理由

1. **V10の最重要課題への直接対応**: 左右非対称性がV10で顕在化（hip_pitch非対称度-15.1%、R hip_pitch range=1.222 vs L=0.902 rad）
2. **既存symmetry報酬の根本的欠陥の回避**: 瞬時位置対称性は交互歩行と矛盾し、うさぎジャンプを誘導（exp007 V9でscale=1.0→hip_pitch同期誘発の実績あり）
3. **物理学的に正しい対称性定義**: 対称な交互歩行では1歩行周期の時間平均で ∫dq_L²dt = ∫dq_R²dt が成立。EMAでこの時間平均を近似
4. **位相不問性**: 速度二乗のEMAは位相情報を平滑化するため、交互歩行の位相差を自然に許容
5. **1変更1検証原則の遵守**: コマンド範囲・PD制御は変更なし

### 設計の理論的背景

#### なぜ瞬時対称性報酬が二脚歩行で破綻するか

対称な交互歩行では q_L(t) = q_R(t + T/2) が成立する。つまり各瞬間で q_L(t) ≠ q_R(t) が正常。既存の`_reward_symmetry`は:

- **knee/ankle**: 遊脚（高速屈曲中）と支持脚（ほぼ静止）で角度が異なるのに等値を要求 → 同位相同期（うさぎジャンプ）を誘導
- **roll成分**: L_roll ≈ -R_roll は鏡面対称で歩行中も妥当（唯一安全な成分）

#### 速度二乗EMAが位相不問である理由

対称歩行で q_L(t) = A·f(ωt)、q_R(t) = A·f(ωt+π) とすると:

$$\frac{1}{T}\int_0^T \dot{q}_L^2\,dt = \frac{1}{T}\int_0^T A^2\omega^2 f'(\omega t)^2\,dt$$

変数置換 s = ωt + π により:

$$\frac{1}{T}\int_0^T \dot{q}_R^2\,dt = \frac{1}{T}\int_0^T A^2\omega^2 f'(s)^2\,ds = \frac{1}{T}\int_0^T \dot{q}_L^2\,dt$$

よって1歩行周期の時間平均運動エネルギーは左右で等しい。EMAはこの時間平均の指数重み付き近似である。

#### 正規化の意義

$$\text{error}_j = \left(\frac{\text{EMA}_L^{(j)} - \text{EMA}_R^{(j)}}{0.5(\text{EMA}_L^{(j)} + \text{EMA}_R^{(j)}) + \epsilon}\right)^2$$

絶対差ではなく相対差を用いることで:
- 低速歩行（速度二乗が小さい）でも高速歩行でも一貫した対称性評価が可能
- 特定の関節速度スケールへの依存を排除

### リスク

- EMAの収束に約33ステップ（0.66秒）のウォームアップが必要（エピソード20秒に対し3%）
- 対称性制約が横方向移動コマンドと部分的に矛盾する可能性（横方向移動時はhip_rollの左右差が自然に生じる）
- 報酬項目数17は推奨上限付近（相互作用リスクに注意）
- symmetry_vel_ema報酬が正報酬として既存ペナルティとのバランスを変化させる可能性

### 接地時間非対称（びっこ歩き）への効果予測

V10の目視評価で「前進・後退時に左右の接地時間が不均等で、片足に重心が長く乗っている」現象が観察されている。以下にsymmetry_vel_emaによる改善見込みを分析する。

#### 接地時間非対称のメカニズム

接地時間の左右差は以下の因果連鎖で生じると考えられる:

1. **関節振幅非対称**（V10実測: R hip_pitch range 1.222 > L 0.902 rad）
2. → 片脚（R）が大きく振り出され、遊脚時間が長い（もしくは短い激しい動き）
3. → 反対脚（L）が相対的に長く接地（重心が片足に偏る）
4. → 接地時間の左右差 → びっこ歩きとして視認

#### symmetry_vel_emaの改善メカニズムと限界

**改善が期待できる部分（間接的効果）:**

- 速度二乗EMAは各関節の「活動量」の左右差を直接ペナルティ化する
- R側の過剰な活動（R knee vel RMS 5.554 vs L 4.492）が抑制されれば、遊脚の振幅が均等化に向かう
- 振幅均等化 → 遊脚時間の均等化 → 接地時間の均等化、という間接的な改善経路が存在する

**改善が困難な部分（構造的限界）:**

- symmetry_vel_emaはエピソード全体にわたるEMAであり、接地相/遊脚相を区別しない。関節速度の時間平均が左右等しくても、その速度が発現するタイミング（接地相vs遊脚相）の分布は制約されない
- 例: 右脚が「短い遊脚で高速に振り、長い接地でゆっくり動く」場合と「長い遊脚で低速に振り、短い接地で速く動く」場合は同じEMA値を取りうる
- 接地時間そのものは`swing_duration`報酬と`contact`報酬が制約しているが、これらも左右の区別を持たない

**結論: 部分的改善は期待できるが、完全解消の保証はない**

symmetry_vel_emaは接地時間非対称の主要な上流原因（関節振幅非対称）を攻撃するため、間接的に接地時間の左右差を軽減する効果が期待できる。しかし接地タイミングそのものを左右で等しくする直接的メカニズムは持たない。V12の結果で接地時間非対称が残存する場合、以下の追加策を検討する:

1. **左右接地時間差ペナルティの導入**: `feet_air_time`のL/R差を直接ペナルティ化する報酬関数の新規実装
2. **Mirror augmentation**: 左右反転データ拡張によりポリシーの構造的対称性を強制
3. **位相条件付き対称性報酬**: 接地相・遊脚相それぞれで左右の関節挙動を比較する報酬

## 考察と改善案

### 目視評価所見

目視評価において以下が観察された:
- 前後移動についてはびっこを引いた歩きが軽減されたが、まだ非対称で右足が大きく持ち上がりすぎることが多い
- 左右移動は速度が遅くなりすぎて動きが悪くなった
- 膝が伸びすぎて胴体が高くあがっている。膝の伸びすぎが極めて不自然な見た目
- 膝の伸びすぎと動作全体の安定性を最優先で解消したい

### 対話による多角的分析

以下、データ分析専門エージェント（DA）とロボティクス・物理学俯瞰エージェント（Lead）の対話形式で考察を行う。

---

**DA（Round 1: 初期定量分析）**:

V11→V12で最も顕著な変化5項目:

- **振動の大幅抑制**: 高周波成分比率 14.7%→7.8% (-47%)、knee vel RMS 37-41%減。V12は明らかに滑らかな動作を獲得。
- **横方向揺れの抑制**: Detrended Y std 78.0mm→46.8mm (-40%)。ただし外向き修正戦略を放棄し、受動的安定に移行。
- **R_hip_pitch周波数異常**: 主周波数が1.50Hz→0.12Hzに崩壊（L側は正常維持）。右脚の歩行周期に低周波変調が発生。上位に1.50Hzが残存するため歩行自体は継続しているが、振幅や位相が大きな周期で変動している。
- **knee非対称度の悪化**: +5.7%→+18.6% (L>R)。L knee range=1.788 vs R range=1.153。右膝が十分に屈曲しない。
- **step_length報酬の低下**: -13.3%。歩幅が縮小。

人間所感との対応:
- 「びっこ軽減」: 接地比率 L/R: V11 51.8/44.2→V12 48.2/50.8 でほぼ対称化。hip_pitch逆相関も -0.289→-0.406 に改善。**裏付け**。
- 「右足が大きく持ち上がりすぎ」: knee非対称度 +18.6% (L>R)は右膝rangeが小さい=右膝が伸びたまま振り出すことを示し、「右足が棒のように上がる」と整合。**裏付け**。
- 「左右の速度が遅い」: step_length報酬 -13.3%低下。vy範囲はV11 ±0.20に対しV12は±0.15のまま。**裏付け**。
- 「膝が伸びすぎ、胴体が高い」: base_height V12=0.290m vs V11=0.294mで数値上はV12がむしろ低い。ただしL knee min=-0.266（ほぼ直立）。**部分的矛盾**—平均高さは矛盾するが、左膝の伸展傾向は所感と整合。

Yaw信頼性チェック: ランダムコマンド評価のYaw指標は信頼性が低い。固定コマンド評価データのみによる比較: V12 FWD 20s drift rate ≈-1.68°/s、V11 FWD drift rate = 1.424°/s。符号が反転（V11右偏向→V12左偏向）しており、Yawバイアスは改善しておらず方向が反転しただけと評価。

**Lead（Round 2: 物理解釈・仮説構築）**:

主因果連鎖: **symmetry_vel_ema → 「活動量均等化」を最小抵抗経路で達成 → 右膝抑制 → カスケード悪化**

1. symmetry_vel_ema(scale 0.3)は4関節ペアの速度二乗EMAの左右差を最小化する正報酬
2. V10で右側が過活動だったため、policyは**右側の活動量を下げる**ことで対称性報酬を獲得する最小抵抗経路を発見
3. R knee range -35%、R hip_pitch主周波数 1.50→0.12Hz崩壊 = 右脚を「サボらせる」戦略

**Lead（Round 6: 非対称性の根本原因と構造的解決策）**:

#### 根本原因: 自発的対称性の破れ（Spontaneous Symmetry Breaking）

URDFは完全に左右対称（初期姿勢 L/R hip_pitch = 1.047 rad、L/R knee_pitch = -1.745 rad）。報酬関数も左右を区別しない対称設計。にもかかわらず、V10/V11/V12のすべてでポリシーが左右非対称な歩容を獲得する原因は:

1. **NNの初期重みの非対称性**: ランダム初期化で左右の出力にμVオーダーの差が発生
2. **PPOの勾配ノイズ**: 確率的最適化が対称な報酬ランドスケープ上で対称性を自発的に破る
3. **正のフィードバックループ**: 微小な非対称 → Yawドリフト → 補償的な非対称アクション → 非対称の固定化

これは**報酬設計の問題ではなくオプティマイザの構造的問題**。対称な報酬関数をいくら積み重ねても、最適化過程の対称性の破れは防げない。symmetry_vel_emaが失敗した根本原因であり、hip_yaw対称性報酬（V11提案の案A）やknee制約報酬もまた同じ構造的限界に直面する。

#### 正しい解決策: Mirror Augmentation（データ拡張による構造的対称性強制）

報酬チューニングではなく、**訓練データ自体を対称化する**:

```
訓練の各ステップで50%の確率で:
  obs_mirror  = swap_LR(obs)     ← dof_pos/vel の L↔R 入れ替え
  cmd_mirror  = (vx, -vy, -vyaw) ← 横方向・旋回コマンドの符号反転
  action_mirror = swap_LR(action)
として学習データに含める
```

Mirror Augmentationにより:

- ポリシーは `π(mirror(s), mirror(cmd)) = mirror(π(s, cmd))` を**構造的に保証**される
- 報酬関数の追加不要（報酬項目数16のまま）
- trivial solutionの余地なし（データ自体が対称なため、非対称ポリシーは学習データに適合不能）
- コマンド方向依存の非対称にも対応（「左移動は右移動の鏡像」が自動保証）

**横方向移動への影響**: 「左に0.15m/sで歩く」の物理的鏡像は「右に0.15m/sで歩く」であり、vyの符号反転と脚の役割入れ替えで正確に対応する。Mirror Augmentationはこの物理的に正しい対応関係をそのまま学習させるため、横方向移動に悪影響を与えない。

**実装上の注意**: vy・vyawの符号反転を忘れると「左に行けと言われたら右に行く」を学習するため破滅的に壊れる。この1点のみ確認が必要。

---

### 成功点

- **振動の大幅抑制**: 高周波成分比率 14.7%→7.8%（-47%）、knee vel RMS 37-41%減。symmetry_vel_emaによる全体的な関節速度振幅の抑制効果
- **びっこの軽減**: 接地比が L/R 51.8/44.2→48.2/50.8 でほぼ対称化。hip_pitch逆相関も -0.289→-0.406 に改善
- **横方向揺れの抑制**: Detrended Y std 78.0→46.8mm（-40%）、Roll-横方向増幅率 1.51→1.34倍
- **正報酬全般の向上**: tracking_lin_vel +7.6%、contact +7.4%、single_foot_contact +8.1%

### 課題

- **膝の伸びすぎ（V12固有）**: L knee min=-0.266 rad（ほぼ直立）、R knee swingで伸びたまま振り出し。symmetry_vel_emaが右側活動量を選択的に削減した結果。V11では膝過伸展は0.2%以下であり、symmetry_vel_ema除去で解消見込み
- **symmetry_vel_emaの構造的欠陥**: 速度二乗EMAは活動量削減のtrivial solutionを許容。右側vel RMSの選択的抑制（hip_pitch R -49.4%、knee R -40.8%）でrange非対称を隠蔽。非対称の方向が反転しただけで根本的対称化に失敗
- **右足過剰挙上が未解決**: hip_pitch range非対称は V11 +28.5% → V12 +28.6% で不変。symmetry_vel_emaは本課題に対して無力だった
- **非対称性の根本原因は報酬設計外**: コマンド方向でL/R非対称の符号が反転（random: R>L +28.5%、FWD固定: L>R -27.0%）。対称URDFに対するPPO最適化の自発的対称性の破れが原因であり、報酬関数の追加では原理的に解決不能
- **R hip_pitch歩行パターンの崩壊**: 主周波数が1.50→0.12Hz（8秒周期の超低周波変調）に崩壊
- **左右速度の遅さ**: vy範囲±0.15のままかつstep_length -13.3%低下
- **Yawドリフト**: FWD固定 drift rate ≈-1.68°/s（V11: 1.424°/s）。符号反転のみで改善なし

### 次バージョンへの提案

**推奨案: V11ベース + Mirror Augmentation**

- **変更内容**: V11をベースとし（symmetry_vel_emaは除去）、訓練時に50%確率でL↔R反転データ拡張（Mirror Augmentation）を適用。報酬関数・パラメータは一切変更なし
- **期待効果**: (1) ポリシーの左右対称性を構造的に保証（報酬チューニング不要）。(2) 右足過剰挙上（hip_pitch range非対称+28.5%）の根本解消。(3) コマンド方向依存の非対称も自動的に解消（左移動=右移動の鏡像が保証）。(4) Yawドリフトの改善（hip_yaw非対称がドリフトの主因であるため）。(5) V11の成功点（tracking_lin_vel回復、hip_roll安定化）の維持
- **リスク**: vyおよびvyawの符号反転の実装ミスは致命的（左右逆転を学習）。mirror時のobservation/action対応テーブルの正確な定義が必要
- **根拠**: V10→V12で試行した報酬ベースの対称化はtrivial solutionを許容して構造的に失敗。非対称性の根本原因はPPO最適化の自発的対称性の破れであり、報酬関数の追加では原理的に解決不能。データ自体を対称化するMirror Augmentationが唯一の構造的解法。報酬項目数16維持、1変更1検証原則に合致

**却下案: base_height_target引き下げ（0.20m → 0.18m）**

当初案CとしてV11ベース回帰と組み合わせてbase_height_target引き下げを推奨していたが、Round 5の追加分析により却下:

1. **ターゲットが非現実的**: 実測base_height 0.29mに対しtarget 0.20mは既に47%乖離。0.18mにしても追加ペナルティは0.021（tracking_lin_vel報酬の4%）で報酬ランドスケープ上無視可能
2. **勾配変化が微小**: 勾配増加21.4%（-0.935→-1.136）では、現行勾配が既に無視されている状況で行動変化を誘起しない
3. **解決すべき問題が存在しない**: 膝の伸びすぎはV12固有（symmetry_vel_emaの副作用）でありV11では0.2%以下。V11ベースに戻すだけで解消
4. **因果の逆転**: corr(height, sum_knee)=+0.524。高さは膝伸展の結果であり原因ではない

## まとめ

V12（symmetry_vel_ema追加）は、振動抑制(-47%)・接地対称化・横方向揺れ抑制(-40%)という副次的な成功を達成したが、核心目標である左右対称化には構造的に失敗した。symmetry_vel_emaは速度二乗EMAの活動量削減というtrivial solutionを許容し、policyは右側の活動量を選択的に抑制（hip_pitch R -49.4%、knee R -40.8%）することで報酬を獲得。その結果、R hip_pitchの歩行パターンが崩壊（主周波数0.12Hz）し、左膝が補償的に過伸展（min=-0.266 rad ≒ 直立）するカスケード悪化が発生した（exp008 V5「ペナルティ達成のためのgait degradation」と構造的に同一）。

追加分析により、以下の重要な知見が得られた:

1. **symmetry_vel_emaは右足過剰挙上を解決していない**: hip_pitch range非対称は V11 +28.5% → V12 +28.6% で不変。本課題に対して無力
2. **非対称性はコマンド方向依存で符号反転する**: random R>L +28.5%、FWD固定 L>R -27.0%。特定の脚を標的とする報酬設計は原理的に不適切
3. **根本原因は自発的対称性の破れ**: 対称URDFに対するPPOの確率的最適化が対称性を自発的に破る。報酬関数の追加では解決不能
4. **base_height_target引き下げは無効**: 実測0.29mに対しtarget 0.20mが既に47%乖離。0.18mにしても追加ペナルティ0.021（勾配増21%）で行動変化は誘起されない。膝の伸びすぎもV12固有でありV11ベース回帰で解消

次バージョンではV11ベースに回帰しsymmetry_vel_emaを除去した上で、Mirror Augmentation（50%確率のL↔R反転データ拡張）を導入し、ポリシーの左右対称性を構造的に保証することを推奨する。
