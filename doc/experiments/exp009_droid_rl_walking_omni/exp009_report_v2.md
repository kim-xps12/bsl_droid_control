# V2: velocity_deficit scale縮小

## 概要

V1で発覚したvelocity_deficitの構造的不適合（+2650%爆発）を修正するため、scaleを-0.5から-0.1に5倍縮小する。ペナルティ構造の安定化により、ポリシーが歩行品質を維持しながら多方向コマンドを学習できるようになることを目指す。

## 前バージョンからの改善事項

V1でvelocity_deficitが因果チェーンの起点となり、hip_pitch相関崩壊（-0.254）、Yawドリフト（+39.24°）、action_rate爆発（+231.5%）が発生。V1レポートの「次バージョンへの提案」に基づき対応。

| パラメータ | V1 | V2 | 変更理由 |
|-----------|-----|-----|---------|
| velocity_deficit scale | -0.5 | -0.1 | +2650%爆発の構造的不適合を修正。tracking_lin_velが速度追従の主要インセンティブとして十分機能 |

## 設計詳細

### velocity_deficit scale縮小の根拠

1. **構造的不適合**: velocity_deficitは前進のみで最適化されたscale(-0.5)。多方向コマンドでは常時発火し、-0.0008→-0.0220（+2650%）に急増
2. **tracking_lin_velとの重複**: tracking_lin_vel（scale 1.5, Gaussian）がXY両方向の速度追従インセンティブを提供。velocity_deficitは補助的役割
3. **scale tuning原則**: exp008 V23で実証された「項目追加/削除よりscale調整が効果的」に基づく段階的対応（サーベイ Section 2.2.2）
4. **完全削除ではなく5倍縮小**: 効果検証のため-0.1を維持。tracking_lin_velのみでは速度不足が生じる可能性への安全弁

### 報酬構成

有効報酬項目数: 16（V1と同一）

## 考察と改善案

### 成功点

- **velocity_deficit主目的達成**: -0.0220 → -0.0042（-80.9%）。V1レポートの予測値（~-0.0044、80%減）とほぼ一致
- **ratio健全域回復**: 0.102 → 0.080。exp008 V13-V25の健全域（0.07-0.08）に復帰
- **ペナルティ全面改善**: 全11ペナルティ項目のうち10項目が改善。velocity_deficit圧力低下がカスケード改善を誘発
- **action_rate -31.7%**: ポリシーの過剰応答が大幅に緩和。多方向コマンドへの対応が落ち着いた
- **L/R歩行周波数同期化**: V1のL=1.25/R=1.12Hz → V2では両方1.25Hzに同期。hip_pitch相関改善（+52.8%）に寄与
- **収束改善**: Last 50 std が1.020 → 0.398（-61%）。訓練が安定化
- **Roll std改善**: 4.22° → 3.66°（-13.3%）

### 課題

1. **Yawドリフト不変（+39.24° → +40.60°）**: velocity_deficit縮小ではYawドリフトは改善されなかった。Yawドリフトの主因はvelocity_deficitペナルティ圧力ではなく、多方向歩行タスク自体に起因するL/R非対称性にある

2. **hip_pitch相関依然低い（-0.389）**: V25基準（-0.851）と比較して依然54%の劣化。多方向コマンドでは前進・後退・横移動を切り替えるため、定常的なanti-phase hip_pitchパターンが成立しにくい構造的要因がある

3. **横方向 detrended std +20.1% 悪化（36.5 → 43.8mm）**: V1よりも横方向揺れが増加

4. **R² 崩壊（0.513 → 0.122）**: V1では横方向揺れの51%がRollで説明できたが、V2ではわずか12%。残差std が25.4→41.0mmに増加し、非Roll成分の横方向揺れが支配的に。原因はL hip_roll range の爆発（0.203→0.489 rad、2.4x）。L脚が外向きに+13.5°まで振れることで、運動学的チェーンによる横方向変位がRoll角度とは独立に発生

5. **L/R接地比非対称化（47.2%/45.8% → 40.0%/54.5%）**: R脚のスタンス時間がL脚の1.36倍。knee_pitch非対称度もV1の+22.0%から-12.6%へ反転。RL stochasticityによるL/R非対称パターンの変化

6. **収束未完了**: Last 50 std=0.398はV1の1.020から大幅改善だが、0.1の閾値を超過。Q3→Q4で+7.43の増分が継続しており、追加学習で更なる改善が見込める

### velocity_deficit縮小がR²崩壊を引き起こしたメカニズム

V1ではvelocity_deficit圧力がポリシーの自由度を強く制約し、hip_rollの活動範囲を狭めていた（L range=0.203 rad）。V2でこの圧力を80%低減したことで、ポリシーはhip_rollをバランス制御に積極活用する自由を獲得。結果としてL hip_rollが0.489 radまで拡大し、運動学的チェーンによる横方向変位がRollとは独立に発生するようになった。

これは「ペナルティ低減は常に改善をもたらすわけではない」という重要な教訓を示す。velocity_deficit圧力は歩行品質を劣化させる面と、hip_roll活動範囲を暗黙的に制約する面の二面性を持っていた。

### サーベイ知見との照合

- **Section 2.2.2（scale tuning）**: velocity_deficit scale 5倍縮小でペナルティ81%減。scale tuningの有効性が多方向タスクでも確認された
- **Section 7.3.1（カリキュラム学習）**: 多方向タスクの収束不足（Last 50 std=0.398）は、タスク複雑度に対する学習量不足を示唆。段階的コマンド範囲拡大やiteration増加が有効な可能性
- **Section 3.x（安定性）**: R²崩壊は、ペナルティ構造変更に伴うポリシーの自由度変化が予期しない動力学変化を引き起こす例。サーベイの「報酬変更の二次効果」に該当

### 次バージョンへの提案

**推奨（V3）: max_iterations 500 → 2000**

根拠:
- V2のLast 50 std=0.398（V1の1.020から61%改善だが、0.1閾値を超過）
- Q3→Q4で+7.43の増分が継続。学習はまだ進行中
- V1レポートで「V2でscale修正→V3でiteration増加」の順序が提案されており、V2でscaleは修正完了
- 1変更1検証原則に基づき、iteration増加のみを変更
- velocity_deficit scale修正とiteration増加を同時に行っていないため、追加学習の純粋な効果を検証可能
- 多方向タスクはexp008（前進のみ）より本質的に複雑。500 stepsでは収束に不十分であり、十分なマージンを確保する

期待効果:
- Last 50 stdの0.1以下への収束
- hip_pitch相関の追加改善（学習途上で非対称パターンが解消される可能性）
- L/R接地比の対称化
- R²の回復（ポリシーが成熟することでhip_roll活用が最適化される可能性）

**V4以降の候補**:
- コマンド範囲段階的拡大（カリキュラム学習: [-0.2, 0.2]から開始→徐々に[-0.3, 0.3]へ）
- symmetry_range報酬の再有効化（L/R非対称の直接的制御）
- ang_vel_range非ゼロ化（Yaw旋回コマンドの追加）

## まとめ

exp009 V2 はV1のvelocity_deficit構造的不適合（+2650%爆発）に対し、scale を -0.5 → -0.1 に5倍縮小した。主目的であるvelocity_deficitペナルティの低減は達成（-80.9%、予測値-80%と一致）し、ratio は0.102→0.080の健全域に回復した。全11ペナルティ項目のうち10項目が改善し、action_rate -31.7%、L/R歩行周波数の同期化（両方1.25Hz）、hip_pitch相関 +52.8%（-0.254→-0.389）など、ペナルティ圧力低下による連鎖的改善が確認された。

一方、Yawドリフトは+40.60°で不変であり、velocity_deficit圧力が原因ではなく多方向タスク固有のL/R非対称性に起因することが判明した。また、ペナルティ低減によりL hip_roll range が2.4倍に拡大し、R²が0.513→0.122に崩壊するという予期しない二次効果が発生。収束もLast 50 std=0.398で未完了であるため、V3ではmax_iterationsを500→2000に増加して追加学習の効果を検証する。

## 備考

- 評価時のコマンドはランダムサンプリングのため、各評価実行で指標値が変動する。特にX速度・Y速度・Yawドリフトは発行コマンドに強く依存
- V1とV2の評価は異なるランダムシードのコマンドシーケンスで実行されているため、Yawドリフトの+3.5%変化は統計的に有意ではない可能性がある
- L hip_roll range拡大（+13.5°外向き）はV17の「カタパルトパターン」（±26-27°）と同方向だが、規模は大幅に小さい
