# V12: 歩行品質改善（内股・爪先立ち・小刻み歩行対策）

## 概要

V11は静止ポリシー回避に成功し、X速度0.212 m/sを達成した。しかし、報酬項目の削減（22→15項目）により、以下の歩行品質問題が顕在化した：

1. **内股**（hip_roll + hip_yaw偏向）
2. **爪先立ち**（右ankle_pitch逸脱）
3. **小刻み歩行**（hip_pitch可動域不足・同期傾向）

V12では、V11レポートの「次バージョンへの提案」に基づき、`hip_pos`と`foot_flat`報酬を復活させて歩行品質を改善する。

**方針**:
- 報酬項目数: 15 → 17（hip_pos, foot_flat復活）
- V11の歩行速度を維持しながら品質を改善
- symmetryは不採用（hip_pitch同期リスク）

## 前バージョンからの改善事項

V11レポート（exp007_report_v11.md）の「次バージョンへの提案」セクションに基づく改善。

### V11の課題（データに基づく）

#### 1. 内股問題（hip_roll + hip_yaw）

**hip_roll（内股の直接原因）**:
| 脚 | 可動域 (rad) | 問題点 |
|---|-------------|--------|
| 左 | [-0.442, 0.0] | 常に負 = 内転方向のみ |
| 右 | [0.0, 0.332] | 常に正 = 内転方向のみ |

- デフォルト(0)を跨がず、**両脚とも常に内側に偏向**

**hip_yaw（追加の非対称性）**:
| 脚 | 可動域 (rad) | 問題点 |
|---|-------------|--------|
| 左 | [-0.444, 0.284] | 両方向に動く（正常） |
| 右 | [-0.542, 0.000] | **常に負方向のみ** |

- 右脚の回旋が制限され、歩行パターンの非対称性に寄与

**原因**: V11で`hip_pos`報酬を削除したため、hip_roll/hip_yawの過度な変位を抑制する機構がなくなった。

#### 2. 爪先立ち問題（ankle_pitch）

| 指標 | 左ankle_pitch | 右ankle_pitch | 差異 |
|------|--------------|---------------|------|
| デフォルト | 0.785 rad (45°) | 0.785 rad (45°) | - |
| max | 0.819 rad | **1.269 rad** | 右が+0.45 rad |
| デフォルトからの最大逸脱 | +2° | **+28°** | 右が14倍 |
| 可動域 | 0.577 rad | **1.027 rad** | 右が1.78倍 |

**原因**: V11で`foot_flat`報酬を削除したため、ankle_pitchの逸脱を抑制する機構がなくなった。

#### 3. 小刻み歩行問題（hip_pitch）

| 指標 | 値 | 問題点 |
|------|-----|--------|
| 左hip_pitch range | 0.218 rad (12.5°) | **極めて小さい** |
| 右hip_pitch range | 0.643 rad (36.8°) | 左の約3倍 |
| hip_pitch相関 | +0.245 | 同期方向（理想は-1.0に近い値） |

**原因**: 左脚がほぼ動いておらず、右脚主導の非対称歩行。

### V12: パラメータ変更一覧

| パラメータ | V11値 | V12値 | 変更理由 |
|-----------|-------|-------|---------|
| hip_pos | なし | **-0.3** | 内股対策（hip_roll/hip_yaw偏向是正） |
| foot_flat | なし | **-0.3** | 爪先立ち対策（ankle_pitch逸脱抑制） |
| single_foot_contact | 0.8 | **1.0** | 交互歩行促進（左脚の動作不足改善） |
| air_time_offset | 0.25 | **0.20** | 小刻み歩行対策（歩行サイクル促進） |
| swing_height_target | 0.03 | **0.04** | 足上げ高さ増加（引きずり防止） |

**報酬項目数**: 15 → 17（+2: hip_pos, foot_flat復活）

### 2. 復活報酬の設計根拠

#### 2.1 hip_pos報酬（内股対策）

**既存実装（droid_env_unitree.py:629）**:
```python
def _reward_hip_pos(self):
    """股関節位置ペナルティ（V1追加）

    股関節（hip_yaw, hip_roll）の過度な変位をペナルティ。
    目的: 脚が大きく開くことを防止
    """
```

**設定値**: -0.3（V10の-0.5より緩和）

**根拠**:
- V10で-0.5を使用した際、静止ポリシーを誘発
- V11で削除した結果、内股問題が顕在化
- -0.3は控えめな値で、静止誘発リスクを軽減しつつ偏向を是正

#### 2.2 foot_flat報酬（爪先立ち対策）

**既存実装（droid_env_unitree.py:772）**:
```python
def _reward_foot_flat(self):
    """足裏水平報酬（V7追加）

    足首（ankle_pitch）が極端に傾かないように制御。
    爪先立ちや踵立ちを防止し、足裏全体での接地を促進。
    """
```

**設定値**: -0.3

**根拠**:
- V11で削除した結果、右ankle_pitchがデフォルトから+28°逸脱
- ankle_pitchのデフォルトからの乖離を直接ペナルティ化
- symmetryを使用しない理由: V9でhip_pitch相関が+0.605に悪化

### 3. その他パラメータの調整根拠

#### 3.1 single_foot_contact強化（0.8 → 1.0）

- 片足接地状態をより強く報酬化
- 左脚の動作不足（hip_pitch range 12.5°）を改善
- 交互歩行を誘導

#### 3.2 air_time_offset調整（0.25 → 0.20）

- より短い遊脚時間でも報酬を獲得可能に
- 歩行サイクルを促進
- ストライド拡大を誘導

#### 3.3 swing_height_target微増（0.03 → 0.04）

- 足上げ高さを増加
- 引きずりを防止
- V10の0.05は過剰だったため、0.04で試行

### 4. symmetryを不採用とする理由

symmetry報酬は一見、左右非対称性の解決に有効に見えるが、以下の理由で採用しない：

1. **V9での悪影響**: symmetry=1.0でhip_pitch相関が-0.382 → +0.605に悪化（同期方向へ）
2. **hip_pitch同期リスク**: 両脚の関節角度を揃えようとすると、交互歩行ではなく同期歩行を誘発
3. **問題の切り分け**: 爪先立ち問題はankle_pitchに限定されており、全関節の対称性を報酬化する必要はない

代わりに、問題の根本原因を直接ペナルティ化する`hip_pos`と`foot_flat`報酬を採用する。

## 考察

### V12の成果と限界

V12では、hip_pos（-0.3）とfoot_flat（-0.3）を復活させ、single_foot_contactを強化（0.8→1.0）した。

**成果**:
- hip_pitch相関が+0.245→-0.208と改善（交互方向へ）
- 左hip_pitch rangeが0.218→0.617radと約3倍に拡大

**限界**:
- **内股問題は改善されず**: hip_rollは依然としてデフォルト(0)を跨がない
- **爪先立ち問題は改善されず**: 右ankle_pitchの逸脱は+28°→+29°と微増
- **目視で確認される問題は継続**

### 根本原因分析：V1との比較から見える問題

**V1からV12への変更の蓄積が問題を引き起こしている**。

V1は以下の点で健全な歩行をしていた：
1. **hip_rollがデフォルト(0)を跨いでいた**（内股ではなかった）
2. **ankle_pitchが安定**（爪先立ちではなかった）
3. **動きが適度**（DOF range sum: 2.290 rad）

V2-V12で加えた変更の副作用で、これらが失われた。

#### 1. action_rate削除の影響

V1では`action_rate=-0.01`があり、連続するアクション間の急激な変化をペナルティ化していた。

```python
# V1: action_rate=-0.01
# V12: action_rateなし（V11で削除）
```

**影響**: action_rate削除により、関節が急激に動くようになった。
- ankle_pitch velocity std: V1=2.0-2.6 → V12=3.5-3.6（約1.5倍）
- knee_pitch velocity std: V1=0.9-1.1 → V12=1.9-2.4（約2倍）

#### 2. hip_posの弱体化

V1では`hip_pos=-0.5`で強めにhip_roll/hip_yaw偏向を抑制していた。V12では-0.3と弱めに設定。

```python
# V1: hip_pos=-0.5
# V12: hip_pos=-0.3（V11で削除後、復活時に緩和）
```

**影響**: hip_rollの可動域が拡大し、内股傾向が強まった。
- V1 hip_roll L: [-0.172, 0.006]（デフォルト跨ぐ）
- V12 hip_roll L: [-0.422, -0.000]（デフォルト跨がない）

#### 3. tracking_sigmaの厳格化

V1では`tracking_sigma=0.25`と緩やかだったが、V6以降で0.10に厳格化。

```python
# V1: tracking_sigma=0.25
# V12: tracking_sigma=0.10
```

**影響**: 速度追従がより敏感になり、過剰反応を誘発した可能性。

### 教訓：「改善」の累積が悪化を招いた

V1→V12の間で、各バージョンで「改善」として加えた変更が累積し、**当初あった健全な歩行パターンを破壊**した。

1. **action_rate削除**: 静止ポリシー回避のため → 急激な動きを抑制する機構を失う
2. **hip_pos削減**: ペナルティ累積回避のため → 内股を抑制する機構が弱体化
3. **tracking_sigma厳格化**: 静止回避のため → 過剰反応を誘発
4. **その他多数の変更**: 各問題への対処が新たな問題を生む

**結論**: 「余計なこと」をしすぎた。V1に近い設定に戻すべき。

## 次バージョン（V13）への提案

### 方針：V1回帰

V12までの「改良」を大幅に取り消し、**V1に近い設定に戻す**。

### 具体的変更案

| パラメータ | V12値 | V13提案値 | 変更理由 |
|-----------|-------|----------|---------|
| **action_rate** | なし | **-0.01** | 【復活】急激な動きを抑制、V1と同じ |
| **hip_pos** | -0.3 | **-0.5** | 【復活】内股抑制を強化、V1と同じ |
| **tracking_sigma** | 0.10 | **0.25** | 【復活】緩やかな追従、V1と同じ |
| **gait_frequency** | 1.0 Hz | **0.8 Hz** | 【調整】さらにゆっくり歩行（周期1.25秒） |
| single_foot_contact | 1.0 | **0.5** | やや緩和、過剰な動作を抑制 |
| lin_vel_x_range | [0.15, 0.25] | **[0.12, 0.18]** | 速度を下げて安定性優先 |
| foot_flat | -0.3 | **-0.5** | 爪先立ち抑制を強化 |

### gait_frequency検討

**歩行の基本関係**: `速度 ≈ 歩幅 × 周波数`

| バージョン | 周波数 | 周期 | 目標速度 | 想定歩幅 |
|-----------|--------|------|---------|---------|
| V1 | 1.5 Hz | 0.67秒 | 0.2-0.3 m/s | 0.13-0.20 m/step |
| V12 | 1.0 Hz | 1.0秒 | 0.15-0.25 m/s | 0.15-0.25 m/step |
| **V13案** | **0.8 Hz** | **1.25秒** | 0.12-0.18 m/s | **0.15-0.23 m/step** |

**「ゆっくり大股」の実現**:
- 速度を下げる: 0.15-0.25 → 0.12-0.18 m/s
- 歩幅を維持: 約0.15-0.20 m/step
- 周波数を下げる: 1.0 → 0.8 Hz

**計算例**:
- V13目標速度 0.15 m/s ÷ 0.8 Hz = 歩幅 0.1875 m/step
- これはV1の0.175 m/stepより若干大きい「大股」

**注意点**:
- 周波数を下げすぎると、1歩の時間が長くなり不安定になるリスク
- 0.8 Hz（周期1.25秒）は実験的な値、効果を確認して調整

### 報酬項目数

17項目 → 18項目（action_rate復活）

|------|-------|---------|---------|
| X速度 | 0.213 m/s | > 0.15 m/s | 維持（多少低下はOK） |
| hip_roll L min | -0.422 | > -0.20 | 内股解消 |
| hip_roll L max | -0.000 | > 0.00 | デフォルト跨ぐ |
| ankle_pitch R max | 1.301 | < 0.90 | 爪先立ち改善 |
| DOF range sum | 6.851 rad | 2-4 rad | 動きの適正化 |
| 目視評価 | 悪い | 改善 | ユーザー確認 |

### 設計思想の転換

**「静止ポリシー回避」から「歩行品質優先」へ**

V2-V12では静止ポリシー回避を最優先し、「動くこと」に報酬を与え続けた。その結果、過剰に動くポリシーが学習され、内股・爪先立ち・急激な動作が発生した。

V13では、**V1が達成していた安定した歩行パターンを再現**することを目標とする。速度は多少犠牲にしても、「まともな歩行」を優先する

## まとめ

V12は、hip_pos（-0.3）とfoot_flat（-0.3）を復活させて歩行品質改善を試みた。

**数値上の改善**:
- hip_pitch相関: +0.245 → -0.208（交互方向へ）
- 左hip_pitch range: 0.218 → 0.617 rad（約3倍）

**改善されなかった問題**:
- **内股**: hip_rollは依然としてデフォルト(0)を跨がない
- **爪先立ち**: 右ankle_pitchの逸脱は+28°→+29°と微増
- **目視での評価は「悪い」のまま**

**根本原因**:
V1→V12の間で、各バージョンで加えた「改善」が累積し、V1にあった健全な歩行パターンを破壊した。特に:
1. **action_rate削除**: 急激な動きの抑制機構を失った
2. **hip_pos弱体化**: 内股抑制が不十分に
3. **tracking_sigma厳格化**: 過剰反応を誘発

**V13への提案**:
「余計なこと」をやめ、**V1に近い設定に戻す** + **さらにゆっくり大股へ調整**。
- action_rate=-0.01を復活
- hip_pos=-0.5に戻す
- tracking_sigma=0.25に戻す
- gait_frequency=0.8 Hzに下げる（周期1.25秒でゆっくり）
- lin_vel_x_range=[0.12, 0.18]に下げる（速度抑制）
