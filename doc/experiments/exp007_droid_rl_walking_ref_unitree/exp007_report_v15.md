# V15: V3ベース + ゆったり大股歩行

## 概要

V3は0.246 m/sを達成し、静止ポリシー回避に成功した実績がある。V15では、**V3の静止ポリシー回避機構を維持しつつ「ゆったり大股」を目指す**。

**方針**:
- V3の成功要因（single_foot_contact, velocity_deficit, alive削除）を維持
- gait_frequencyを下げて「ゆっくり」を実現
- feet_air_timeを強化して「大股」を誘導
- 報酬項目数: 16項目（V3と同じ）

**V13との違い**:
- V13は「V1ベース + 静止回避機構なし」で失敗した
- V15は「V3ベース + 静止回避機構あり」で成功を期待

## 前バージョンからの改善事項

V3レポート（exp007_report_v3.md）の成功要因を維持しつつ、「ゆったり大股」を実現するための調整を行う。

### V3の成功要因（維持）

| 要素 | 説明 | V15での扱い |
|------|------|-------------|
| single_foot_contact: 0.3 | 片足接地報酬 | **維持** |
| velocity_deficit: -0.5 | 速度未達ペナルティ | **維持** |
| alive: 削除 | 静止の報酬価値を除去 | **維持** |
| tracking_lin_vel: 1.5 | 動く動機を強化 | **維持** |

### V15での変更点（3点）

| パラメータ | V3値 | V15値 | 変更理由 |
|-----------|------|-------|---------|
| **gait_frequency** | 1.5 Hz | **1.0 Hz** | ゆっくり歩行（周期0.67秒→1.0秒） |
| **lin_vel_x_range** | [0.2, 0.3] | **[0.15, 0.25]** | やや低速の目標 |
| **feet_air_time** | 1.0 | **1.5** | 滞空時間報酬を強化して大股誘導 |

### V13との設計比較

| 要素 | V13（失敗） | V15（本バージョン） |
|------|------------|-------------------|
| ベース | V1（静止回避機構なし） | V3（静止回避機構あり） |
| single_foot_contact | なし | **0.3** |
| velocity_deficit | なし | **-0.5** |
| alive | あり（0.03相当） | **なし** |
| gait_frequency | 0.8 Hz | **1.0 Hz** |
| 報酬項目数 | 18 | **16** |

V13は静止回避機構なしで周波数を下げたため「動かない」問題が発生した。V15はV3の静止回避機構を維持することで、この問題を回避することを期待。

## 考察

### V15の成果

1. **静止ポリシー回避に成功**: V14（V1ベース）は0.003 m/sで静止したが、V15（V3ベース）は0.198 m/sで歩行
2. **hip_pitch相関の改善**: V3の+0.886（同期）→ V15の-0.165（交互傾向）
3. **目標速度への追従**: 0.198 m/s（目標0.15-0.25 m/s内）

### 「小刻み」問題の原因分析

**データからの観察**:
- DOF range sum: 1.834 rad（V3の2.43 radより小さい）
- knee_pitch可動域: 0.099 rad（V3の0.305 radの約1/3）
- hip_pitch可動域: 0.246 rad（V3の0.135 radの約1.8倍）

**仮説**: hip_pitchは動いているが、**膝がほとんど動いていない**ため、足全体の振り幅が小さく「小刻み」に見える。

### サーベイ（exp007_unitree_rl_gym_survey.md）からの知見

#### 7.3 歩幅拡大に関する知見

1. **feet_air_timeオフセットの調整**（セクション7.3.3）:
   - 現在のオフセット0.5秒はUnitree G1/H1（大型ロボット）向け
   - BSL-Droidの歩行周波数1.0Hz（周期1.0秒）では遊脚期は約0.45秒
   - **オフセットを0.5→0.3秒に変更**することで、より小さい滞空時間でも報酬を獲得可能

2. **action_rate緩和**（セクション7.3.1）:
   - action_rate=-0.01は急激な動作を抑制するが、大きな関節動作も抑制してしまう
   - **action_rate=-0.005に緩和**することで、大きな関節動作を許容

3. **swing_height_target増加**:
   - 現在0.03mだが、足上げが不十分な可能性
   - **0.04mに増加**して足上げを誘導

### V3との比較分析

| 要素 | V3 | V15 | 影響 |
|------|-----|-----|------|
| gait_frequency | 1.5 Hz | 1.0 Hz | 周期が長くなったが、1歩あたりの移動距離が増えていない |
| feet_air_time | 1.0 | 1.5 | 強化したが、オフセット0.5秒では報酬を獲得しにくい |
| action_rate | -0.01 | -0.01 | 大きな関節動作を抑制している可能性 |
| knee_pitch可動域 | 0.305 rad | 0.099 rad | **大幅に減少** |

### 教訓

1. **feet_air_time報酬のスケールを上げても、オフセットが大きすぎると効果がない**
2. **action_rateペナルティが大きな動作を抑制している可能性**
3. **膝の動きが不十分なため、足全体の振り幅が小さい**

## 次バージョン（V16）への提案

### 方針：「大股」を実現するための調整

V15は歩行に成功したが、「小刻み」問題が残った。V16では以下の調整で「大股」を実現する。

### V16の変更案（3点）

| パラメータ | V15値 | V16案 | 変更理由 |
|-----------|-------|-------|---------|
| **action_rate** | -0.01 | **-0.005** | 大きな関節動作を許容 |
| **swing_height_target** | 0.03 m | **0.04 m** | 足上げを誘導 |
| **feet_air_time offset** | 0.5秒 | **0.3秒** | 小さい滞空時間でも報酬獲得可能に |

### V16の報酬設計（V15から微調整）

```python
reward_cfg = {
    "swing_height_target": 0.04,  # ★変更: 0.03 → 0.04 m
    "feet_air_time_offset": 0.3,  # ★新規: オフセットを0.5 → 0.3秒に変更
    # 他はV15と同じ

    "reward_scales": {
        # ...
        "action_rate": -0.005,    # ★変更: -0.01 → -0.005
        # 他はV15と同じ
    },
}
```

## まとめ

V15は、**V3の静止ポリシー回避機構を維持しつつ歩行に成功した**。

**成功点**:
- 静止ポリシー回避: V14（V1ベース）は0.003 m/sで静止、V15（V3ベース）は0.198 m/sで歩行
- hip_pitch相関改善: +0.886 → -0.165（交互傾向に改善）
- 目標速度達成: 0.198 m/s（目標0.15-0.25 m/s内）

**残課題**:
- 「小刻み」歩行: DOF range sum 1.834 rad（V3の2.43 radより小さい）
- 膝の動きが不十分: knee_pitch可動域 0.099 rad（V3の0.305 radの約1/3）

**原因分析**:
- feet_air_timeのオフセット0.5秒がBSL-Droidには大きすぎる
- action_rate=-0.01が大きな関節動作を抑制している

**V16への提案**:
1. action_rate: -0.01 → -0.005（大きな関節動作を許容）
2. swing_height_target: 0.03 → 0.04 m（足上げ誘導）
3. feet_air_time offset: 0.5 → 0.3秒（小さい滞空時間でも報酬獲得）
