# V32: swing_contact_penaltyをV30の値に復元

## 概要

V32では、V31で悪化した**Yawドリフト**と**タップダンス問題**に対処するため、`swing_contact_penalty`をV30の値（-0.5）に戻す。

V31の成功点（`single_foot_contact`強化によるhip_pitch相関改善）は維持しつつ、V31の失敗点（`swing_contact_penalty`緩和による副作用）を修正する。

**1変更1検証の原則**に従い、今回はswing_contact_penaltyの復元のみを行う。

## 前バージョンからの改善事項

### V31の評価結果

| 指標 | V30値 | V31値 | 変化 | 問題点 |
|------|-------|-------|------|--------|
| X速度 | 0.170 m/s | 0.166 m/s | -2% | - |
| 片足接地率 | 95.4% | 93.0% | -2.4% | タップダンス再発 |
| hip_pitch相関 | -0.571 | -0.660 | +16%改善 | 目標-0.8未達 |
| Yawドリフト | +11.76° | **+30.37°** | **+158%悪化** | **深刻な問題** |
| DOF range sum | 5.493 rad | 6.290 rad | +14%増加 | 動きが大きい |

### V31で観測された課題

1. **Yawドリフトの大幅悪化**: +11.76° → +30.37°
   - 原因: `swing_contact_penalty`緩和により足踏み動作が増加、非対称な動作がYawを乱した

2. **タップダンス問題の再発**:
   - 片足接地率: 95.4% → 93.0%
   - ユーザー観察: 「大きな動きで足踏みを挟みながら歩いている」
   - 原因: `swing_contact_penalty`を-0.5→-0.3に緩和したことで、スイング中の接地抑制が弱まった

3. **「大きな動き」の発生**:
   - DOF range sum: 5.493 → 6.290 rad（+14%）
   - 全関節の可動範囲が増加

### 改善策（V32）

V31レポートの「次バージョンへの提案」セクションの**提案A**を採用。

| パラメータ | V31値 | V32値 | 変更理由 |
|-----------|-------|-------|---------|
| swing_contact_penalty | -0.3 | **-0.5** | V30の値に戻す（タップダンス抑制回復） |
| single_foot_contact | 0.5 | **0.5** | 維持（hip_pitch相関改善に寄与） |
| 報酬項目数 | 20 | **20** | 変更なし |

## 設計詳細

### swing_contact_penalty復元の根拠

**問題分析**:
- V31で`swing_contact_penalty`を-0.5→-0.3に緩和した結果:
  - Yawドリフト: +11.76° → +30.37°（+158%悪化）
  - 片足接地率: 95.4% → 93.0%（タップダンス再発）
  - DOF range sum: +14%増加（動きが大きくなった）

- V30では`swing_contact_penalty=-0.5`で以下を達成:
  - 片足接地率: **95.4%**
  - タップダンス: 「複数回の足踏み」→「1回タップして着地」

**対策**:
- swing_contact_penalty: -0.3 → -0.5（V30の値に戻す）
- スイング位相中の接地を強くペナルティ化し、タップダンスを抑制

### single_foot_contact維持の根拠

**V31での効果**:
- hip_pitch相関: -0.571 → -0.660（+16%改善）
- single_foot_contact報酬: 0.2196 → 0.4116（+87%増加）

**結論**:
- `single_foot_contact=0.5`は交互歩行パターン改善に寄与
- この設定を維持し、V31の成功点を保持

## 考察と改善案

### 成功点

1. **Yawドリフトの大幅改善**: +30.37° → -2.00°
   - swing_contact_penalty=-0.5への復元が効果を発揮
   - スイング中の足踏み抑制により、非対称動作によるYaw乱れが減少

2. **タップダンス抑制の部分的回復**: 片足接地率 93.0% → 94.4%
   - swing_contact_penaltyの発動量が22%増加（-0.0303 → -0.0370）
   - swing_durationが12%改善（0.1176 → 0.1320）

3. **歩行安定性の向上**: DOF range sum 6.290 → 5.651 rad（-10%）
   - 「大きな動き」が抑制され、安定した動作に

### 課題

1. **左右非対称問題が残存**
   - hip_pitch velocity std: 左=2.161、右=1.187（左が82%大きい）
   - ユーザー観察: 右足は改善したが、左足だけタップダンスが残る
   - 根本原因: leg_phase初期値の偏り（左=0、右=0.5で固定）、MLPの対称性非保証

2. **hip_pitch相関の悪化**: -0.660 → -0.544
   - 交互歩行パターンがやや悪化
   - swing_contact_penalty強化により、片脚の動きが抑制されすぎた可能性

3. **成功基準の一部未達**
   - hip_pitch相関: 目標-0.65に対し-0.544（未達）
   - 片足接地率: 目標95%に対し94.4%（ほぼ達成だがわずかに未達）

### 次バージョンへの提案

**採用案: symmetry_range報酬の有効化**

左右非対称問題に直接対処するため、`symmetry_range`報酬を有効化する。

```python
"symmetry_range": 0.3  # V32では0（無効）→ V33で0.3に設定
```

**根拠**:
1. 左右のhip_pitch振幅の非対称性を直接ペナルティ化
2. 「片足だけタップダンス」問題に直接対処
3. 既存の実装を再利用（新規コード不要）
4. 1変更1検証の原則に適合

**代替案（symmetry_rangeが効果なしの場合）**:
- 案A: swing_contact_penaltyをさらに強化（-0.5 → -0.7）
- 案C: hip_pitch_antiphase_v2の有効化（0.3）
- 案D-1: leg_phase初期値のランダム化（環境クラス修正）
