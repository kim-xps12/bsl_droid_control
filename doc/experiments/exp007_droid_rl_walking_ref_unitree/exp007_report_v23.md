# V23: B案（air_time_offset短縮）によるfeet_air_time報酬の正への転換を狙う

## 概要

V22で発生した「タップダンス歩容」（歩容一周期の間に片足で断続的に2回地面を叩く動作）を解消するため、B案（air_time_offset短縮）を実施する。

**タップダンス歩容の主因**: `air_time_offset=0.3s`が実際の遊脚時間より長いため、`feet_air_time`報酬が負のまま（V22: -0.0202）であり、足を長く上げておく動機が不足している。

**B案の変更**: `air_time_offset`を0.3sから**0.2s**に短縮し、`feet_air_time`報酬の正への転換を狙う。

## 設計詳細

### 1. B案: air_time_offset短縮

**設計原理**:

V22で発生したタップダンス歩容の原因分析:
- feet_air_time報酬が負（-0.0202）
- `air_time_offset=0.3s`が実際の遊脚時間より長い
- 足を長く上げておく動機が不足
- 結果: 一度上げた足がすぐに接地し、再度上げるパターン（タップダンス）が発生

**歩行周期と滞空時間の関係**:
- 歩行周波数: 1.2Hz
- 1周期: 0.833s（= 1/1.2）
- 片足の遊脚期: 約0.42s（周期の半分）
- V22の`air_time_offset=0.3s`では、遊脚期の71%（0.3/0.42）以上を滞空しないと報酬が正にならない
- V23の`air_time_offset=0.2s`では、遊脚期の48%（0.2/0.42）で報酬が正になる

**期待される効果**:
- feet_air_time報酬が正に転じることを期待
- 足上げ動機が維持され、タップダンス歩容が軽減
- より自然な歩行サイクルの獲得

### 2. A案の維持

V22で導入したA案（ankle_pitch_rangeペナルティ）は維持する。

```python
reward_cfg = {
    "ankle_pitch_limit": 0.3,  # ankle_pitchの許容範囲（rad）
    "reward_scales": {
        "ankle_pitch_range": -0.3,  # 遊脚時のankle_pitch角度制限ペナルティ
    },
}
```

**維持理由**:
- V22でペナルティ値は-0.0006と小さかったが、悪影響も確認されなかった
- A案の本格的な見直しはV24以降で検討
