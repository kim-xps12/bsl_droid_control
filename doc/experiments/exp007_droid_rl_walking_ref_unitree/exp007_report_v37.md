# V37: hip_pos強化による横揺れ抑制（-0.8 → -1.0）

## 概要

V37では、V36をベースにhip_posスケールを-0.8から-1.0に強化し、hip_roll運動の抑制による横揺れ軽減を目指す。V36で報酬項目数を16項目（推奨範囲15-17）に到達済みであり、V37では報酬項目数を維持したまま既存パラメータの調整のみを行う。

**1変更1検証の原則**に従い、今回はhip_posスケールの変更のみを行う。

## 前バージョンからの改善事項

### V36で観測された課題

1. **hip_pitch相関の大幅悪化**: -0.669→-0.435。交互歩行パターンの明確さが大きく低下
2. **hip_roll R rangeの増加**: 0.416→0.439 rad。右脚の横方向運動が増加し横揺れに寄与
3. **スイング持続時間の短縮**: 0.49s→0.31s。V35で獲得した「長時間空中保持」戦略が弱化
4. **ユーザー目視**: タップダンス再発、横揺れ継続

### 改善策（V37）

V36レポートの「次バージョンへの提案」セクションの**推奨案: V36ベースでhip_pos強化（-0.8 → -1.0）**を採用。

| パラメータ | V36値 | V37値 | 変更理由 |
|-----------|-------|-------|---------|
| hip_pos | -0.8 | **-1.0** | hip_roll運動の抑制による横揺れ軽減。V36でhip_posはペナルティ1位（20.4%）であり、スケール強化により横方向の関節運動をさらに抑制する |
| 他のパラメータ | - | - | V36から全て維持 |
| 報酬項目数 | 16 | **16** | 推奨範囲（15-17）を維持 |

## 設計詳細

### hip_pos強化の根拠

hip_posペナルティはhip_yawとhip_rollの両関節の絶対値をペナルティする報酬項目である。V35-V36を通じてペナルティ中の1位（V35: 23.8%、V36: 20.4%）を占めており、横揺れ（hip_roll運動の増大）の主要な制御レバーとなる。

V36のhip_roll運動:
- L range: 0.319 rad（V35: 0.487から-34%改善）
- R range: 0.439 rad（V35: 0.416から+6%悪化）
- 左右差が拡大しており、特に右脚のhip_roll抑制が必要

hip_posスケールを-0.8→-1.0（+25%強化）に変更することで:
1. hip_roll運動範囲の縮小を促し、横揺れを軽減
2. hip_yawも同時に抑制されるため、歩行ダイナミクス全体への影響に注意が必要

## 考察と改善案

### 成功点

1. **hip_roll R rangeの大幅縮小**: 0.439→0.310 rad（-29%）。hip_pos強化の意図通り、右脚のhip_rollが顕著に抑制された
2. **hip_roll mean値の中立化**: 左右ともhip_roll平均値の絶対値が縮小（L: -0.202→-0.137、R: +0.208→+0.140）し、中立位置へ接近
3. **スイング持続時間の回復**: 0.31s→0.44s（+40%）。V36で失われた「長時間空中保持」戦略がV37で再獲得された
4. **L/R接地バランスの均等化**: 46.2/52.8→48.5/49.8。ほぼ完全に均等な片足接地分布を達成
5. **hip_pitch非対称度の改善**: +21.0%→+14.0%。依然として残存するが7ポイント縮小
6. **Base height安定性の改善**: std 0.0096→0.0079m（-18%）
7. **タップダンス解消**: ユーザーが目視で確認。スイング持続時間の回復と整合

### 課題

1. **Yawドリフトの大幅悪化**: -4.92°→-17.35°（3.5倍悪化）。t=4sで-36.51°に達する初期急速ドリフトが発生。過去最悪
2. **Roll std/横揺れの悪化**: Roll std 8.21°→9.29°（+13%）、base_pos_y std 0.055→0.164m（3倍）。hip_rollは縮小したにもかかわらず胴体の横揺れは増大
3. **Pitch stdの大幅悪化**: 1.60°→5.24°（+228%）。前後方向の姿勢安定性が著しく低下
4. **knee_pitch非対称度の大幅悪化**: +2.2%→+27.5%。右膝のrange（0.923→0.521 rad）が大幅に縮小し、右脚全体の運動が制限された
5. **X速度の低下**: 0.168→0.149 m/s（-11%）。目標値0.15m/sをわずかに下回る
6. **ang_vel_xyペナルティの増大**: -0.0810→-0.0917（+13%）。Roll/Pitch角速度の増加を定量的に裏付け
7. **orientationペナルティの増大**: -0.0130→-0.0157（+21%）。胴体傾斜の増大を反映

### 動作メカニズムの因果分析

#### hip_pos強化がhip_yawに与えた副作用

hip_posペナルティはhip_yawとhip_rollの**両方**を同時にペナルティする（`torch.sum(torch.square([L_hip_yaw, L_hip_roll, R_hip_yaw, R_hip_roll]))`）。-0.8→-1.0の強化はhip_rollだけでなくhip_yawも同等に制約した。

V37のhip_yaw range R（0.285 rad）はV36（0.354 rad）から-19%縮小した。hip_yawは歩行中の方向修正に使用される関節であり、その運動範囲の制約がYaw方向の修正能力を低下させ、Yawドリフトの急速な蓄積（0-4s: -36.51°）を引き起こした可能性がある。

#### Yawドリフトと横方向スウェイの連鎖

V37のbase_pos_y std（0.164m）がV36（0.055m）の3倍に達した主因として、Yawドリフトによる斜行の効果を考慮する必要がある。

t=4sでYaw=-36.51°に達した状態では、ロボットの前進方向がY軸に対して大きく傾斜する。このとき、通常のX方向前進が見かけ上のY方向変位として現れる。base_pos_y stdの大幅増加はこのYawドリフトによる斜行効果を含んでおり、胴体自体のRoll振動の増加（Roll std +13%）だけでは説明できない。

したがって、V37の横方向スウェイ悪化は以下の2つの要因の複合と推定される:
1. **Yawドリフトによる斜行効果**: base_pos_y std増加の主因（Yaw最大偏差-36.51°による見かけの横移動）
2. **Roll振動の純増**: Roll std 8.21°→9.29°（+13%）。hip_rollは縮小したが、それを上回るPitch方向の不安定化がRoll-Pitch結合を通じてRollにも波及した可能性

#### 右脚運動制限のメカニズム

V37ではknee_pitch R range（0.950→0.665 rad、-30%）やankle_pitch R vel std（2.587→1.739、-33%）など、右脚全体の運動が一貫して低下した。全関節のL/R角速度比が1.0を超え（最大: knee_pitch 1.69）、左脚主導の歩行パターンが形成されている。

これはRL確率的変動の影響もあるが、hip_pos強化が右脚のhip_roll/hip_yawをより強く制約し（右脚のhip_roll rangeがV36で支配的だったため制約の影響が大きい）、その結果として右脚下流の関節も連鎖的に運動が縮小したと解釈できる。

#### hip_rollは縮小したのにRoll stdが悪化した理由

V37ではhip_roll R rangeが-29%と大幅に縮小し、hip_roll meanも中立方向に改善した。にもかかわらずRoll std（胴体のRoll角振幅）が8.21°→9.29°に悪化した。

この乖離は、**hip_rollの縮小がRoll安定性に必ずしも直結しない**ことを示している。hip_rollは歩行中の動的バランスの一部として機能しており、過度な制約は逆に以下のメカニズムを通じてRoll振動を増幅する可能性がある:
1. hip_rollの自由度が制約されると、片足支持時のCoM（重心）横方向シフトの調整能力が低下する
2. その結果、胴体が受動的にRoll方向に振動する（制御不能なRoll）
3. Pitch stdの大幅悪化（+228%）はhip_yaw制約がPitch方向の安定性にも波及したことを示唆し、Pitch不安定化がRollへもカップリングしている

これはサーベイSection 3.4（hip_pos）の実装が示すように、hip_posはhip_yawとhip_rollを無差別にペナルティするため、**歩行に必要なhip_yawの適応的運動まで抑制してしまう構造的限界**を示している。

### サーベイ知見との照合

1. **Section 3.4（hip_pos）**: hip_posがhip_yawとhip_rollを同時にペナルティする構造が確認された。V37の結果は、この「無差別ペナルティ」の副作用（hip_yaw制約→Yaw修正能力低下→Yawドリフト悪化）を実証した

2. **Section 2.2.2（ang_vel_xy）**: デフォルトスケール-0.05はUnitreeベースラインであり、BSL-Droidでも同値を使用中。V37のang_vel_xyペナルティ増大（-0.0810→-0.0917）は、Roll/Pitch角速度が増加していることを反映。ang_vel_xyスケールの強化はRoll角速度を直接抑制する手段として有効であり、hip_yawへの副作用がない

3. **Section 2.2.3（orientation）**: BSL-Droid用に-0.5に調整済み（Unitree humanoid: -1.0）。V37のorientationペナルティ増大（-0.0130→-0.0157）は胴体傾斜の増加を反映。orientationの強化はRoll/Pitchの位置偏差を直接ペナルティする

4. **Section 7.2.3（対称等変ポリシー）**: V37のknee_pitch非対称度+27.5%は報酬shapingの限界を再確認。同一の報酬構成でも左右の運動パターンに大きな偏りが生じうる

5. **Section 6.2（BSL-Droid推奨パラメータ）**: ang_vel_xy=-0.05はUnitree準拠であるが、BSL-Droidのサイズ・質量分布ではより強い値が適切である可能性がある

#### 横揺れ関連報酬の体系図

横揺れ（Roll振動・横方向スウェイ）に関連する報酬項目とその因果関係:

- **直接制御**: ang_vel_xy（Roll/Pitch角速度→-0.05）、orientation（Roll/Pitch傾斜→-0.5）
- **間接制御（関節経由）**: hip_pos（hip_yaw+hip_roll→-1.0）、swing_foot_lateral_velocity（スイング足横速度→-0.5）
- **非関連**: 他のペナルティ項目は横揺れに対する直接的な制御力を持たない

V37の結果から、**間接制御（hip_pos経由）の限界**が明確になった。hip_posはhip_yawを巻き込むため副作用が大きく、hip_roll縮小がRoll振動改善に直結しない。**直接制御（ang_vel_xy、orientation）の強化**が次の論理的なステップである。

## まとめ

V37はhip_posスケールを-0.8→-1.0に強化し、hip_roll R rangeの-29%縮小、スイング持続時間の回復（0.31→0.44s）、L/R接地バランスの均等化（48.5/49.8）など、複数の歩行品質指標で改善を達成した。ユーザーの目視でもタップダンスの解消が確認された。

しかし、hip_posペナルティがhip_yawとhip_rollを無差別にペナルティする構造的特性により、hip_yaw運動の制約がYawドリフトの大幅悪化（-4.92°→-17.35°、最大偏差-36.51°）を引き起こした。さらに、hip_rollの縮小にもかかわらずRoll std（+13%）およびbase_pos_y std（3倍）が悪化し、「hip_roll縮小≠Roll振動改善」という重要な知見が得られた。これはhip_rollが歩行中の動的バランス調整に寄与しており、過度な制約が逆にRoll振動を増幅するメカニズムを示唆する。

V38では、V37をベースにang_vel_xy強化（-0.05→-0.1）によるRoll/Pitch角速度の直接抑制を推奨する。タップダンス解消というV37の成果を維持しつつ、横揺れの根本原因であるRoll角速度を直接制約する。V38で改善が不十分な場合は、hip_posの段階的緩和（-1.0→-0.9）やhip_roll/hip_yawへの分離を次段階として検討する。

## 備考

### V37の教訓: hip_pos強化の構造的限界

V37の結果は「hip_pos（hip_yaw + hip_roll同時ペナルティ）による横揺れ制御」の構造的限界を明確にした。hip_rollのみを選択的にペナルティする報酬項目は現在の報酬設計に存在しないため、hip_posの強化はhip_yawへの副作用を避けられない。

将来的には以下の選択肢が考えられる:
1. **hip_roll専用ペナルティの新設**: hip_yawとhip_rollを分離して制御する。報酬項目数は17になるが推奨範囲内
2. **直接制御の活用**: ang_vel_xy（Roll角速度）やorientation（Roll位置）など、胴体レベルの制御で関節レベルの副作用を回避
3. **等変ネットワーク**: 構造的に左右対称な出力を保証し、報酬shapingの限界を超越
