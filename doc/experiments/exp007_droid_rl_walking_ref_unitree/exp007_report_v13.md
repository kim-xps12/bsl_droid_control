# V13: V1回帰 + ゆっくり大股歩行の試行

## 概要

V12レポートの「次バージョンへの提案」に基づき、V1に近い設定に戻しつつ「ゆっくり大股」歩行を目指した。しかし、**速度目標とgait_frequencyを下げすぎた**ことで、V1の健全な歩行パターンを再現できず、目視で確認された問題は改善されなかった。

**方針**:
- 報酬項目数: 17 → 18（action_rate復活）
- V1のaction_rate、hip_pos、tracking_sigmaに戻す
- gait_frequencyを0.8Hzに下げて「ゆっくり」を実現
- lin_vel_x_rangeを[0.12, 0.18]に下げて速度を抑制

**結論**: 「V1に戻す」方針自体は正しかったが、同時に速度・周波数を下げる変更を加えたため、V1の歩行パターンを再現できなかった。

## 前バージョンからの改善事項

V12レポート（exp007_report_v12.md）の「次バージョンへの提案」セクションに基づく改善。

### V12の課題と対策

| パラメータ | V12値 | V13値 | 変更理由 |
|-----------|-------|-------|---------|
| action_rate | なし | **-0.01** | 【復活】急激な動きを抑制、V1と同じ |
| hip_pos | -0.3 | **-0.5** | 【強化】内股抑制を強化、V1と同じ |
| tracking_sigma | 0.10 | **0.25** | 【緩和】緩やかな追従、V1と同じ |
| gait_frequency | 1.0 Hz | **0.8 Hz** | 【調整】さらにゆっくり歩行（周期1.25秒） |
| single_foot_contact | 1.0 | **0.5** | やや緩和、過剰な動作を抑制 |
| lin_vel_x_range | [0.15, 0.25] | **[0.12, 0.18]** | 速度を下げて安定性優先 |
| foot_flat | -0.3 | **-0.5** | 【強化】爪先立ち抑制を強化 |

**報酬項目数**: 17 → 18（+1: action_rate復活）

## 考察

### V13の成果と限界

**成果**:
1. **hip_rollの改善**: 両脚ともデフォルト(0)を跨ぐようになった
2. **DOF range sumの適正化**: 6.851 → 3.331 rad（過剰な動きが抑制された）
3. **Yawドリフトの改善**: +5.92° → +4.46°
4. **hip_pitch相関の改善**: -0.208 → -0.382（より交互傾向）

**限界**:
1. **X速度の低下**: 0.213 → 0.106 m/s（目標0.15 m/sを下回る）
2. **爪先立ち問題の継続**: 右ankle_pitch max=1.147 rad（依然高い）
3. **目視での問題が継続**: 内股、爪先立ち、小刻み、胴体傾斜

### 根本原因分析

#### 「V1に戻す」は正しかったが、同時に「速度・周波数を下げる」変更を加えたことが問題

V13の設計思想は「V1回帰 + ゆっくり大股」だった。しかし、以下の2つの変更を**同時に**行ったことで、V1の歩行パターンを再現できなかった：

1. **gait_frequency低下**: 1.5 Hz → 0.8 Hz（約半分）
2. **lin_vel_x_range低下**: [0.2, 0.3] → [0.12, 0.18]（約半分）

これらの変更により：
- **歩行サイクルが遅くなりすぎた**（周期1.25秒）
- **速度目標が低すぎて、動く動機が弱まった**
- 結果として「ゆっくり」ではなく「小刻みで遅い」動作になった

#### V1が「動いていた」理由

V1の評価結果を詳細に分析すると：

| 指標 | V1値 | 意味 |
|------|------|------|
| X速度 | 0.263 m/s | 目標0.2-0.3を達成 |
| gait_frequency | 1.5 Hz | 周期0.67秒の速いリズム |
| DOF velocity RMS | 1.331 rad/s | 活発な関節動作 |
| ankle_pitch | [0.451, 0.785] | 安定した足首角度 |

V1は**高速・高周波数**で動いていたからこそ、安定した歩行パターンを維持できていた。低速・低周波数にすると、バランスを取る機会が減り、かえって不安定になる。

### 教訓：「ゆっくり」と「大股」は別の課題

ユーザーの要望は「ゆっくり大股で速度を下げたい」だったが、これを達成するには：

1. **まずV1を完全に再現する**（高速・高周波数で動く歩行）
2. **その後で速度だけを下げる**（周波数は維持）
3. **「大股」は歩幅の問題**であり、速度と周波数の調整だけでは実現できない

V2-V13で行った「速度を下げる」「周波数を下げる」「報酬を追加する」変更は、**V1が持っていた健全な歩行パターンを壊してしまった**。

### 報酬項目数の問題

| バージョン | 報酬項目数 | X速度 | 評価 |
|-----------|-----------|-------|------|
| V1 | 15 | 0.263 m/s | 動いていた |
| V3-V4 | 16-17 | 0.15-0.19 m/s | 動いていた |
| V7-V10 | 22 | 0.006-0.21 m/s | 不安定 |
| V11 | 15 | 0.212 m/s | 復活 |
| V12-V13 | 17-18 | 0.11-0.21 m/s | 不安定 |

**exp007_unitree_rl_gym_survey.mdの知見**によれば：
- Unitree G1/H1は**13-14項目**で実機歩行を達成
- BSL-Droidは**15-17項目**が推奨

V13の18項目は境界線上であり、報酬項目の相互作用が予測困難になっている可能性がある。

## 次バージョン（V14）への提案

### 方針：V1ベースで1変更1検証

V1は既に学習済みで動いている（X速度0.263 m/s）。同じ設定で再学習しても価値がない。

V14からは**V1をベースに1変更1検証**で「ゆっくり大股」を段階的に実現する。

### V14: 速度目標のみ低下

**変更点は1つだけ**：

| パラメータ | V1値 | V14値 | 変更理由 |
|-----------|------|-------|---------|
| **lin_vel_x_range** | [0.2, 0.3] | **[0.15, 0.20]** | 速度を下げる（これだけ） |

**他は全てV1と同じ**（gait_frequency=1.5Hz、報酬設計15項目、tracking_sigma=0.25 等）

### V14の報酬設計（V1と同じ、15項目）

```python
reward_scales = {
    "tracking_lin_vel": 1.0,
    "tracking_ang_vel": 0.5,
    "feet_air_time": 1.0,
    "contact": 0.2,
    "alive": 0.1,
    "lin_vel_z": -2.0,
    "ang_vel_xy": -0.05,
    "orientation": -0.5,
    "base_height": -5.0,
    "feet_swing_height": -5.0,
    "contact_no_vel": -0.1,
    "hip_pos": -0.5,
    "torques": -1e-5,
    "action_rate": -0.01,
    "dof_acc": -2.5e-7,
}
```

### V14の成功基準

| 指標 | V1値 | V14目標 | 判定基準 |
|------|------|---------|---------|
| X速度 | 0.263 m/s | 0.15-0.20 m/s | 目標速度に追従 |
| hip_roll L | [-0.172, 0.006] | デフォルト跨ぐ | V1レベル維持 |
| ankle_pitch R max | 0.785 rad | < 0.85 rad | V1レベル維持 |
| DOF range sum | 2.290 rad | 2-3 rad | V1レベル維持 |
| 目視評価 | - | V1と同等 | 歩行品質維持 |

### 「ゆっくり大股」の実現ロードマップ

1. **V14: 速度のみ低下**
   - lin_vel_x_range: [0.2, 0.3] → [0.15, 0.20]
   - **他は一切変更しない**
   - 速度低下の影響のみを観察

2. **V15以降: 必要に応じて調整**
   - V14で歩行品質が維持できていれば、さらに速度を下げる
   - 歩幅が足りなければ、feet_air_timeを調整
   - **常に1変更1検証**

### 設計思想

**「1変更1検証」の厳守**

V2-V13では、複数の変更を同時に行い、何が効いて何が悪影響かわからなくなった。

V14以降は：
1. **V1をベースラインとして固定**
2. **1バージョン1変更**
3. **変更が悪影響を与えたら即座にロールバック**

## まとめ

V13は、V12レポートの「V1回帰」提案に基づき、action_rate、hip_pos、tracking_sigmaをV1の値に戻した。しかし、**同時にgait_frequencyとlin_vel_x_rangeを下げた**ことで、V1の歩行パターンを再現できなかった。

**数値上の改善**:
- hip_roll: 両脚ともデフォルト(0)を跨ぐようになった
- DOF range sum: 6.851 → 3.331 rad（適正範囲）
- hip_pitch相関: -0.208 → -0.382（より交互傾向）

**改善されなかった問題**:
- X速度: 0.213 → 0.106 m/s（低下、目標未達成）
- 爪先立ち: 右ankle_pitch max=1.147 rad（依然高い）
- 目視での評価は「悪い」のまま（内股、爪先立ち、小刻み、胴体傾斜）

**根本原因**:
「V1に戻す」と「速度・周波数を下げる」を同時に行ったため、V1の歩行パターンを再現できなかった。

**V14への提案**:
V1の設定を**完全にそのまま**使用する。追加・変更・削除は一切行わない。「ゆっくり大股」はV15以降で、V1の再現に成功した後に段階的に実現する。
