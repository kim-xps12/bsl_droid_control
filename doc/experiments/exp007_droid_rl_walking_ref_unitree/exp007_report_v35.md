# V35: swing_contact_penalty強化（-0.5 → -0.7）

## 概要

V35では、V34で残存した左足タップダンスの直接抑制を目的として、swing_contact_penaltyを-0.5から-0.7に強化する。

V30-V34の実験で、swing_contact_penalty=-0.5が片側タップダンス抑制の最低有効閾値であることが判明している。V32/V34では右足タップダンスは改善されたが左足が残存しており、より強いペナルティが必要と判断された。-0.5→-0.7の変化幅はV30→V31（-0.5→-0.3）の逆方向と同程度であり、1変更1検証の原則に適合する。

**1変更1検証の原則**に従い、今回はswing_contact_penaltyの強化のみを行う。

## 前バージョンからの改善事項

### V34の評価結果

| X速度 | 0.175 m/s | 0.173 m/s | -1% | ほぼ維持 |

| hip_pitch相関 | -0.544 | -0.602 | 改善 | V32水準 |

### V34で観測された課題

1. **左足タップダンスの残存**
   - ユーザー目視: 「左足だけ」にタップダンスが発現
   - 両足接地率4.4%がV32の2.8%に回復していない
   - knee_pitch L/R角速度比=2.09と左膝の過剰活動が継続

2. **swing_contact_penalty=-0.5の限界**
   - V30で-0.5が右足タップダンス抑制に有効と実証済み
   - V32/V34で左足が残存していることから、-0.5は片側抑制の閾値であるがもう片側には不十分

### 改善策（V35）

V34レポートの「次バージョンへの提案」セクションの**推奨案: swing_contact_penalty強化（-0.5 → -0.7）**を採用。

| パラメータ | V34値 | V35値 | 変更理由 |
|-----------|-------|-------|---------|
| swing_contact_penalty | -0.5 | **-0.7** | 左足タップダンスの直接抑制 |
| 他のパラメータ | - | - | V34から全て維持 |
| 報酬項目数 | 20 | 20 | 変更なし |

## 設計詳細

### swing_contact_penalty強化の根拠

1. **V30-V31の実験結果**: V30（-0.5）で右足タップダンス改善、V31（-0.3）で悪化。-0.5が最低有効閾値
2. **V32-V34の観察**: -0.5では左足タップダンスが残存し続ける（両足接地率4.4%）
3. **変化幅の妥当性**: -0.5→-0.7（変化幅0.2）はV30→V31（-0.5→-0.3、変化幅0.2）と同程度
4. **リスク評価**: 中。-0.7は未検証領域だが、swing_contact_penaltyのペナルティ中の割合は8.8%（V34）であり、0.2の強化による全体バランスへの影響は限定的

## 考察と改善案

### 成功点

1. **タップダンスの大幅改善**: 両足接地率がV34の4.4%からV35の1.4%に低下し、V32の2.8%を下回った。両足接地イベント数もV34の20回からV35の7回に65%減少し、swing_contact_penalty=-0.7はタップダンス抑制に極めて有効であった
2. **knee_pitch対称性の劇的改善**: knee_pitch非対称度がV34の+21.4%（L>R）からV35の+0.4%に改善し、V34で問題だった左膝の過剰活動が解消された。knee_pitch velocity L/R比もV34の2.09からV35の0.95にほぼ均等化
3. **X速度の改善**: 0.178 m/sはV32の0.175 m/sを上回り、過去最高を記録した
4. **Base heightの回復**: 0.274 mはV34の0.250 mから24 mm上昇し、V32の0.262 mも上回った。高さの安定性（std=0.003）もV34（0.006）から改善
5. **hip_pitch交互歩行の改善**: 相関係数が-0.669（V34: -0.602）に改善し、ローリング相関のstdもV34の0.085から0.040に半減。位相関係の安定性が向上した
6. **接地パターンの対称化**: 片足接地がL49.0%/R48.8%とほぼ均等（V34: L45.5%/R50.2%）

### 課題

1. **hip_roll関節の運動範囲増大**: hip_roll rangeがV34比でL: +20%（0.405→0.487 rad）、R: +26%（0.331→0.416 rad）と増加。hip_roll velocity RMSもL: +28%、R: +33%増加。ユーザーが「体を左右に揺らして歩く」と表現した横揺れの主因
2. **Yawドリフトの悪化**: ドリフト速度の絶対値がV34の1.42°/sからV35の2.36°/sに増加。最終値-9.07°も大きい
3. **hip_pitch非対称の方向逆転**: V34の+5.9%（L>R）からV35の-12.0%（R>L）に逆転。RL訓練の確率的変動による
4. **hip_posペナルティの増大**: -0.0937→-0.1143と22%増加し、ペナルティ中のトップに。hip_roll運動の増大を反映
5. **ペナルティ/主報酬比の上昇**: 0.327→0.350。feet_swing_heightとhip_posの増大が主因
6. **報酬項目数**: 20項目は推奨上限（15-17項目）を依然として超過

### 動作メカニズムの因果分析

#### swing_contact_penalty強化の波及効果

swing_contact_penalty=-0.7への強化は以下の因果連鎖を引き起こした:

1. **直接効果**: 遊脚のスイング中の接地がより強くペナルティされる
2. **行動変化**: ポリシーが遊脚をより長時間空中に保持する戦略を学習（スイング持続: 0.26s→0.49s、約2倍）
3. **swing_duration報酬の増加**: 0.1377→0.2354（+71%）。空中時間が長くなりswing_duration報酬を多く獲得
4. **重心移動の必要性**: 片足での支持時間が約2倍に延長されたため、重心を支持脚上により大きく移動させる必要が生じた
5. **hip_roll運動の増大**: 重心移動を実現するためにhip_roll関節の運動範囲が拡大（L: +20%、R: +26%）
6. **hip_posペナルティの増大**: hip_rollの増大により-0.0937→-0.1143に増加

**核心**: swing_contact_penalty強化はタップダンスの直接抑制に成功したが、その代償として片足支持時間が延長され、動的バランスのためのhip_roll運動が増大した。これは**タップダンスと横揺れのトレードオフ**であり、同一の報酬構成内では不可避な関係である。

#### swing_contact_penaltyの減衰パラドックス

swing_contact_penaltyの最終値は-0.0352（V34: -0.0416）と、スケール強化（-0.5→-0.7）にもかかわらず**絶対値が減少**した。これはスイング中の接地イベント自体が大幅に減少したためであり、ペナルティが有効に機能した結果である。ペナルティ値の減少はタップダンスの改善と整合する。

#### Yawドリフト方向逆転のメカニズム

V35のhip_pitch非対称度は-12.0%（R>L）であり、V34の+5.9%（L>R）から逆転した。V32でもL>R（+13.9%）であったことから、V35の右脚優位は新しいパターンである。

既知のパターンとの対応:
- V32: L>R非対称 → 負方向Yaw（-2.00°）
- V34: L>R非対称（弱い） → 正方向Yaw（+7.35°）
- V35: R>L非対称 → 負方向Yaw（-9.07°）

V34の例外（L>Rなのに正方向Yaw）を除き、hip_pitch非対称の方向とYawドリフト方向に相関がある。しかし、RL訓練の確率的変動が非対称方向を決定するため、報酬設計による直接的な制御は困難である。

### サーベイ知見との照合

1. **Section 2.2.2（ang_vel_xy）**: ang_vel_xyはRoll/Pitch角速度を抑制する。V35のang_vel_xy値（-0.0834）はV34（-0.0912）からやや減少しており、Roll角速度自体はわずかに改善している。ユーザーが体感した横揺れはRoll角速度ではなく、hip_roll関節の運動によるものであり、ang_vel_xy報酬の直接的な対象外

2. **Section 3.4（hip_pos）**: hip_posはhip_yawとhip_rollの変位をペナルティする。V35でhip_posが-0.0937→-0.1143に増加したのはhip_roll rangeの増大を反映している。hip_posのスケール強化が横揺れ対策の直接的なレバーとなり得る

3. **Section 7.4.2（角運動量バランシング）**: BSL-Droidには腕がないため、遊脚スイングの角運動量をカウンターバランスする手段が限られている。スイング持続の約2倍化は角運動量の不均衡を拡大し、Yawドリフト悪化の一因となった可能性がある

4. **Section 7.2.3（対称等変ポリシー）**: V35でknee_pitchは均等化（0.4%）したが、hip_pitchは-12.0%と逆方向に非対称化した。これは報酬shapingの限界を再確認するものであり、構造的な対称性保証には等変ネットワークが必要

5. **Section 7.6（改善策の優先順位）**: 現段階でhip_roll運動の増大を抑制する最も直接的な手法はhip_posスケールの強化であり、これはサーベイの短期改善策に該当する。等変ネットワークは長期改善策

## まとめ

V35はswing_contact_penaltyを-0.5→-0.7に強化した結果、タップダンスの大幅改善を達成した。両足接地率はV34の4.4%からV35の1.4%に低下し、V32の2.8%を下回って過去最良となった。knee_pitch非対称度もV34の+21.4%から+0.4%に劇的に改善し、左膝の過剰活動が解消された。X速度は0.178 m/sで過去最高を記録し、Base height（0.274 m）も回復した。

CSV時系列分析により以下の新知見を得た:
- **スイング持続時間がV34の約2倍**（R: 0.267→0.492s、L: 0.245→0.474s）に延長され、ポリシーが「遊脚をより長く空中に保持する」戦略を学習した
