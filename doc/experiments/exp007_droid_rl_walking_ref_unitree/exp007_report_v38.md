# V38: ang_vel_xy強化によるRoll/Pitch角速度の直接抑制（-0.05 → -0.1）

## 概要

V38では、V37をベースにang_vel_xyスケールを-0.05から-0.1に2倍強化し、Roll/Pitch角速度を胴体レベルで直接ペナルティすることで、V37の横揺れ・Pitch不安定化に対処する。

V37ではhip_pos強化（-0.8→-1.0）により、タップダンス解消・スイング持続時間回復（0.44s）・L/R接地バランス均等化に成功したが、hip_posがhip_yaw+hip_rollを同時にペナルティする構造的欠陥により、Yawドリフト-17.35°（過去最悪）とPitch std+228%の重大な劣化が発生した。

V38では「間接制御（hip_pos: 関節レベル）」から「直接制御（ang_vel_xy: 胴体レベル）」へのアプローチ転換を行う。ang_vel_xyはhip_yawに影響しないため、Yaw修正能力を保持したままRoll/Pitch安定性を改善できる。

**1変更1検証の原則**に従い、今回はang_vel_xyスケールの変更のみを行う。

## 設計詳細

### ang_vel_xy強化の根拠

ang_vel_xyペナルティは`torch.sum(torch.square(base_ang_vel[:, :2]))`を計算し、胴体のRoll角速度とPitch角速度を直接ペナルティする。

V37での因果分析により、以下が判明:
- **hip_pos（間接制御）の限界**: hip_yaw+hip_rollを同時にペナルティする構造的欠陥。hip_roll縮小（-29%）を達成したが、hip_yaw制約がYaw修正能力を破壊
- **「hip_roll縮小≠Roll振動改善」**: hip_rollが縮小してもRoll stdは悪化（+13%）。関節レベルの制御では胴体の振動を直接抑制できない
- **直接制御の必要性**: 胴体レベルでRoll/Pitch角速度を直接ペナルティすることで、関節の自由度を維持しつつ安定性を確保

### V38の報酬構成（V37からの変更箇所のみ）

| カテゴリ | 項目 | V37値 | V38値 | 変更 |
|---------|------|-------|-------|------|

他の15項目は全てV37と同一。

### リスク分析

- **Pitch角速度への影響**: ang_vel_xyはPitch angular velocityもペナルティするため、前進歩行のダイナミクスが制約される可能性。X速度（V37: 0.149 m/s）のさらなる低下リスク
- **ペナルティ比率の変化**: V37でang_vel_xyはペナルティ2位（18.9%）。2倍強化でペナルティ1位に浮上する可能性があり、報酬バランスの変化に注意

## 考察と改善案

### 成功点

1. **Roll/Pitch安定性の大幅改善**: Roll std -30%（9.29°→6.53°）、Pitch std -40%（5.24°→3.16°）。ang_vel_xy強化の直接的効果として、胴体の角速度振動を効果的に抑制
2. **横揺れ改善**: base_pos_y std -16%（0.164→0.138m）、base_vel_y std -22%。V37で失われた横方向安定性を部分回復
3. **hip_pitch相関の大幅回復**: -0.427→-0.801（+87%）。交互歩行パターンが明確に復活。V32（-0.544）を大幅に上回る過去最高水準
4. **X速度の回復**: 0.149→0.168 m/s（+13%）。V36水準に復帰。ang_vel_xyによるPitch制約が前進速度を阻害するリスクは顕在化せず
5. **L/R均等性の改善**: 関節角速度L/R比が全関節で1.0に近づく。hip_yaw 1.23→0.96、hip_roll 1.34→1.14
6. **orientation改善**: ペナルティ-30.6%減少。Roll/Pitch角速度の抑制がorientation（姿勢角の偏差）にも波及

### 重大な問題

1. **Yawドリフトが過去最悪を更新**: -42.71°（V37: -17.35°の2.5倍）。定常ドリフト速度-2.955°/s
2. **スイング持続時間の再短縮**: 0.44s→0.35s/0.30s。V37で回復した「長時間空中保持」戦略が再び崩壊
3. **hip_roll rangeの増加**: L+24%、R+17%。ただしRoll stdは改善しており、実害は限定的

### 因果メカニズム分析

#### なぜang_vel_xy強化がRoll/Pitch改善とYaw悪化を同時に引き起こしたか

ang_vel_xyはbase_ang_vel[:, :2]（Roll角速度とPitch角速度）をペナルティし、Yaw角速度（base_ang_vel[:, 2]）は制御しない。

**Roll/Pitch改善のメカニズム**:
- スケール2倍により、角速度二乗和が約34%減少（ペナルティ+31.6%はスケール2倍×角速度0.66倍²に対応）
- 関節可動域（hip_roll range）が増加したにもかかわらずRoll振動が抑制されたのは、関節の「動かし方」（角速度）が滑らかになったため
- これはV37の「hip_roll縮小≠Roll改善」の逆命題「角速度抑制→Roll改善」を実証

**Yaw悪化のメカニズム**:
- hip_pos=-1.0（V37から維持）がhip_yaw rangeを制約し続けている（L: 0.289, R: 0.308 rad）
- ang_vel_xy強化はRoll/Pitch angular velocityを直接抑制するが、これにより歩行のダイナミクスが変化
- **Yaw回復フェーズの消失**: V37は0-4sでYaw=-36.51°まで悪化後、4-10sで-17.35°に回復した。V38は回復フェーズが存在せず、7s以降さらに加速
- 仮説: Roll/Pitch角速度の抑制がYaw方向のトルク補正に使われていた「結合効果」を弱めた可能性。Roll振動がYaw修正の間接的メカニズムとして機能していた可能性がある

#### hip_roll range増加の解釈

V37→V38でhip_roll rangeが増加（L: 0.345→0.428, R: 0.310→0.362 rad）した理由:
- ang_vel_xy強化により歩行パターンが変化し、より大きなhip_roll可動域を使うが低速で動かす戦略に移行
- hip_roll mean（V37: L=-0.137, R=+0.140 → V38: L=-0.230, R=+0.201）が増加し、より大きなオフセット角を持つ歩行に
- hip_posペナルティ増加（-0.1003→-0.1143、+14%）はこのhip_roll range増加を反映

#### スイング持続時間短縮の原因

V38でスイングが0.44s→0.30-0.35sに短縮した理由:
- ang_vel_xy強化がスイング中のPitch angular velocityもペナルティするため、脚の振り出し速度が制約される
- 結果として短い振り出しで素早く接地する戦略に移行
- swing_duration報酬は低下（0.1718→0.1587、-7.6%）

### 報酬設計のシステム分析

V38の結果は「直接制御vs間接制御」の対比を鮮明にした:

| アプローチ | V37（hip_pos間接制御） | V38（ang_vel_xy直接制御） |
|-----------|---------------------|------------------------|
| Roll std | 9.29°（悪化） | 6.53°（**改善**） |
| Pitch std | 5.24°（悪化） | 3.16°（**改善**） |
| hip_roll range | 縮小（意図通り） | 増加（逆方向） |
| Yaw drift | -17.35°（悪化） | -42.71°（**さらに悪化**） |
| hip_pitch相関 | -0.427（悪化） | -0.801（**大幅改善**） |
| X速度 | 0.149（低下） | 0.168（**回復**） |

**結論**: 直接制御（ang_vel_xy）はRoll/Pitch安定性とhip_pitch相関に明確な効果を示したが、Yaw制御は別メカニズムが必要。Yawドリフトの根本原因はhip_pos=-1.0によるhip_yaw制約にある。

### 次バージョンへの提案

V38はRoll/Pitch安定性・hip_pitch相関・X速度で明確な改善を達成した。最大の残存課題はYawドリフト（-42.71°）であり、これはhip_pos=-1.0によるhip_yaw制約が根本原因。

#### 推奨案: V38ベース + hip_pos緩和（-1.0 → -0.8）

```
V39: V38ベース + hip_pos: -1.0 → -0.8
```

**根拠**:
- Yawドリフトの根本原因はhip_pos=-1.0によるhip_yaw制約
- V36ではhip_pos=-0.8でYaw=-4.92°だった（許容範囲）
- V38のang_vel_xy=-0.1でRoll/Pitch直接制御が機能することが実証済み
- hip_pos緩和でhip_yaw freedomを回復すれば、Yaw修正能力が復活する期待
- ang_vel_xy=-0.1がRoll安定性を担保するため、hip_pos緩和によるhip_roll増加のリスクは低減
- 「1変更1検証」原則に従い、hip_posのみ変更
- 報酬項目数: 16項目維持

**期待される効果**:
- Yawドリフトの大幅改善（V36水準-5°以下を目標）
- Roll/Pitch安定性はang_vel_xy=-0.1で維持
- hip_pitch相関はhip_pos緩和の影響を受ける可能性あり

**リスク**: hip_pos緩和でhip_roll rangeがさらに増加し、ang_vel_xy=-0.1では抑えきれない可能性

#### 代替案A: hip_yaw/hip_rollの分離（17項目）

hip_posを廃止し、hip_rollとhip_yawを独立した報酬項目に分離:
- hip_roll: -0.8（横揺れ抑制を維持）
- hip_yaw: -0.2（軽いペナルティでYaw自由度を確保）
- 報酬項目数: 17項目（推奨範囲上限）

**利点**: hip_yawとhip_rollを独立に制御可能
**リスク**: 新しい報酬関数の実装が必要。17項目は推奨範囲上限

#### 代替案B: tracking_ang_vel強化（0.5 → 1.0）

Yaw角速度追従報酬を強化してYawドリフトを抑制:
- tracking_ang_velはYaw追従を含む（コマンドは角速度0なので、Yaw角速度0に誘導）
- 報酬項目数: 16項目維持

**リスク**: tracking_ang_velはYaw以外の角速度も含む可能性。効果が限定的な可能性

## まとめ

V38ではang_vel_xyスケールを-0.05から-0.1に2倍強化し、「直接制御（胴体レベル）」アプローチの有効性を実証した。

**達成した改善**:
- Roll std -30%（9.29°→6.53°）、Pitch std -40%（5.24°→3.16°）: 横揺れ・前後揺れの大幅改善
- hip_pitch相関 -0.427→-0.801（+87%）: 過去最高の交互歩行パターン明確さ
- X速度 0.149→0.168 m/s（+13%）: V36水準に回復
- base_pos_y std -16%、base_vel_y std -22%: 横方向安定性の改善
- 全関節のL/R角速度比が1.0に近づき均等化が進行

**残存する重大課題**:
- Yawドリフト -42.71°: 過去最悪を大幅更新。hip_pos=-1.0によるhip_yaw制約が根本原因
- スイング持続時間再短縮: 0.44s→0.30-0.35s

**結論**: ang_vel_xy直接制御はRoll/Pitch安定性に明確な効果を示した。次のV39では、Yaw問題の根本原因であるhip_pos=-1.0の緩和（-0.8へ戻す）が推奨される。ang_vel_xy=-0.1がRoll安定性を担保するため、hip_pos緩和に伴うhip_roll増加リスクは低減されている。

### 「直接制御vs間接制御」の重要な知見

V37-V38の比較から、BSL-Droidの歩行安定性制御における重要な原則が確立された:

1. **関節レベルの間接制御（hip_pos）**: 関節可動域を制約できるが、胴体振動を直接抑制できない。複合ペナルティ（hip_yaw+hip_roll同時）の構造的限界あり
2. **胴体レベルの直接制御（ang_vel_xy）**: 胴体の角速度を直接抑制でき、関節可動域にかかわらず安定性を改善。ただし制御対象外の軸（Yaw）には効果なし
3. **Yaw制御は独立した設計が必要**: ang_vel_xyもhip_posもYawドリフトの直接的解決策にはならない。hip_yawの自由度確保（hip_pos緩和）またはYaw専用報酬が必要
