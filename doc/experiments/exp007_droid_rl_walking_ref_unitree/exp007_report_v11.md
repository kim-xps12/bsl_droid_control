# V11: 最小有効報酬セットによる静止ポリシー回避

## 概要

V10は静止ポリシーへの回帰という深刻な問題が発生した（X速度: 0.006 m/s）。原因はペナルティの累積効果であり、5項目を同時に強化/追加したことで「動くこと自体が大きなペナルティを受ける」状態になった。

V11では、V10レポートの「提案3: 最小有効報酬セット」に基づき、報酬設計を大幅に簡素化する。

**方針**:
1. 報酬項目数を22 → 15に削減（7項目削減）
2. Unitree G1/H1の成功パターンに準拠（13-14項目で実機歩行達成）
3. V3-V4の実績に基づく設定（16-17項目で0.15-0.19 m/s達成）
4. ペナルティ累積効果の回避

## 前バージョンからの改善事項

V10レポート（exp007_report_v10.md）の「次バージョンへの提案」セクションに基づく改善。

### V10の失敗原因分析

1. **ペナルティの累積効果**（根本原因）
   - V10では複数のペナルティを同時に強化・追加
   - 動くこと自体が大きなペナルティを受ける状態に
   - 報酬最大化（98.12）≠ 歩行品質最大化

2. **feet_stumbleの副作用**
   - 歩行動作自体にペナルティがかかり、静止が最適解に

3. **swing_height_targetの過剰引き上げ**
   - 達成困難な目標設定で常時ペナルティ発生

### V11: 報酬項目の削減

**削除する要素（7項目）**:

| 削除要素 | 理由 | 根拠 |
|---------|------|------|
| **symmetry** | hip_pitch同期誘発 | V9で実証（相関+0.605に悪化）、Unitreeになし |
| **hip_pitch_antiphase_v2** | 効果なし | V7-V9で悪化継続、Unitreeになし |
| **both_legs_active** | 効果不明 | Unitreeになし |
| **feet_stumble** | 静止誘発 | V10で実証 |
| **hip_pos** | V3になく不要 | 効果限定的 |
| **action_rate** | ペナルティ累積 | V3-V4では不要 |
| **dof_vel** | V3になく不要 | ペナルティ累積の一因 |

## 考察と改善案

### V11の成果と限界

V11は**静止ポリシー回避に成功**し、X速度0.212 m/sを達成した。これはV10（0.006 m/s）からの大幅な改善であり、V7（0.214 m/s）と同等の水準に回復した。

しかし、**歩行品質の問題**が新たに顕在化した：

1. **内股問題**: hip_rollが両脚とも内側に偏っている
2. **爪先立ち問題**: 右足のankle_pitchが異常に大きく、爪先立ちで接地
3. **小刻み歩行**: hip_pitch相関が+0.245で両脚が同期方向に動く

### 問題原因の分析

#### 1. 内股問題の原因（hip_roll + hip_yaw）

V11では`hip_pos`報酬を削除したため、hip_roll/hip_yawの過度な変位を抑制する機構がなくなった。

**hip_roll（内股の直接原因）**:
- 左: [-0.442, 0.0]（常に負 = 内転方向のみ）
- 右: [0.0, 0.332]（常に正 = 内転方向のみ）
- デフォルト(0)を跨がず、**両脚とも常に内側に偏向**

**hip_yaw（追加の非対称性）**:
- 左: [-0.444, 0.284]（両方向に動く）
- 右: [-0.542, 0.000]（**常に負方向のみ**）
- 右脚の回旋が制限され、歩行パターンの非対称性に寄与

サーベイ（exp007_unitree_rl_gym_survey.md）セクション3.4より：
> **hip_pos（股関節位置ペナルティ）**: 股関節（hip_yaw, hip_roll）の過度な変位をペナルティ。目的: 脚が大きく開くことを防止

**対策**: `hip_pos`報酬を復活させる。V10の-0.5は静止ポリシーを誘発したため、-0.3から開始。hip_rollとhip_yaw両方の偏向を是正。

#### 2. 爪先立ち問題の原因（ankle_pitch）

ankle_pitch（背屈/底屈）の**可動域の非対称性**と**デフォルトからの逸脱**が問題。

**関節データ詳細**:
| 指標 | 左ankle_pitch | 右ankle_pitch | 差異 |
|------|--------------|---------------|------|
| デフォルト | 0.785 rad (45°) | 0.785 rad (45°) | - |
| min | 0.242 rad | 0.242 rad | 同等 |
| max | 0.819 rad | **1.269 rad** | 右が+0.45 rad |
| range | 0.577 rad | **1.027 rad** | 右が1.78倍 |
| デフォルトからの最大逸脱 | +0.034 rad (+2°) | **+0.484 rad (+28°)** | 右が14倍 |

**メカニズム解釈**:
- ankle_pitch正（背屈）= 足がshankに対して前方回転
- 右足のmax値1.269radはデフォルト(0.785rad)から**28°も超過**
- 歩行サイクル中、右足が不安定に動き、一部のタイミングで爪先接地が発生
- V11では`foot_flat`報酬を削除したため、ankle_pitchの逸脱を抑制する機構がない

**対策**: `foot_flat`報酬を**復活**（V7で実装済み、V11で削除）。ankle_pitchのデフォルトからの乖離をペナルティ化し、足裏の平坦接地を促進。

**symmetryを使用しない理由**:
- V9でsymmetry=1.0を使用した際、hip_pitch相関が-0.382 → +0.605に悪化
- 全関節の対称性を強制すると、hip_pitchの逆相関（交互歩行）も抑制してしまう
- 爪先立ち問題はankle_pitchに限定されており、foot_flatで直接対処するのが適切

#### 3. 小刻み歩行の原因（hip_pitch）

2つの問題が複合している：

**問題A: 左脚の可動域不足**
- 左hip_pitch range: **0.218rad（12.5°）** ← 極めて小さい
- 右hip_pitch range: 0.643rad（36.8°）
- **左脚がほとんど動いておらず、右脚主導の歩行**

**問題B: 両脚の同期傾向**
- hip_pitch相関: **+0.245**（理想は-1.0に近い負の値）
- 両脚が逆方向ではなく、同じ方向に動く傾向
- 交互歩行パターンが確立されていない

サーベイ セクション7.3.3より：
> **feet_air_time報酬のオフセット調整**: 遊脚時間の目標を調整することで、より大きなストライドを誘導可能

**対策**:
- `air_time_offset`を調整（0.25 → 0.20）: より短い遊脚時間でも報酬を獲得可能にし、歩行サイクルを促進
- `single_foot_contact`の重みを増加（0.8 → 1.0）: 片足接地状態をより強く報酬化し、交互歩行を誘導
- **注意**: 複雑なantiphase報酬（V7-V9で試行）は逆効果だったため、シンプルなアプローチを維持

### 成功点

1. **静止ポリシー回避**: X速度0.212 m/s達成（V10: 0.006 m/s）
2. **報酬簡素化**: 22項目 → 15項目で歩行回復
3. **直進性**: Yawドリフト+5.35°（V10: +14.52°から改善）
4. **安定性**: エピソード長1001（最大）、Roll/Pitch stdも良好
5. **単足接地率**: 98.6%と高く、交互歩行の基本パターンは確立

### 課題

1. **内股（hip_roll + hip_yaw）**:
   - hip_roll: 両脚ともデフォルト(0)を跨がず、常に内側に偏向
   - hip_yaw: 右脚が常に負方向のみ（正方向に動かない非対称性）
   - 原因: hip_pos報酬削除による副作用
2. **爪先立ち（ankle_pitch）**:
   - 右ankle_pitchがデフォルトから最大+28°逸脱（左は+2°）
   - 右ankle_pitchの可動域が左の1.78倍と非対称
   - 原因: foot_flat報酬削除による副作用
3. **小刻み歩行（hip_pitch）**:
   - 左hip_pitch rangeが0.218rad（12.5°）と極小（右は0.643rad）
   - hip_pitch相関+0.245（同期方向、理想は-1.0に近い値）
   - 左脚がほぼ動いておらず、右脚主導の非対称歩行

**方針**: V11をベースに、内股・爪先立ち・小刻み歩行の3問題に対処


- 片足接地状態をより強く報酬化
- hip_pitch相関+0.245（同期方向）を改善
- 複雑なantiphase報酬は過去の実験で逆効果だったため、シンプルなアプローチを維持

#### 提案5: swing_height_targetの微増（足上げ高さ対策）

```python
"swing_height_target": 0.04,  # V11: 0.03 → V12: 0.04
```

- 足上げ高さを微増して引きずりを防止
- V10の0.05は過剰だったため、0.04で試行

#### V12パラメータ変更一覧（V11からの差分）

| パラメータ | V11値 | V12提案値 | 変更理由 |
|-----------|-------|----------|---------|
| hip_pos | なし | **-0.3** | 内股対策（hip_roll/hip_yaw偏向是正） |
| foot_flat | なし | **-0.3** | 【復活】爪先立ち対策（ankle_pitch逸脱抑制） |
| single_foot_contact | 0.8 | **1.0** | 交互歩行促進（左脚の動作不足改善） |
| air_time_offset | 0.25 | **0.20** | 小刻み歩行対策（歩行サイクル促進） |
| swing_height_target | 0.03 | **0.04** | 足上げ高さ増加（引きずり防止） |

**報酬項目数**: 15 → 17（復活2: hip_pos, foot_flat）

#### symmetryを不採用とする理由

symmetry報酬は一見、左右非対称性の解決に有効に見えるが、以下の理由で採用しない：

1. **V9での悪影響**: symmetry=1.0でhip_pitch相関が-0.382 → +0.605に悪化（同期方向へ）
2. **hip_pitch同期リスク**: 両脚の関節角度を揃えようとすると、交互歩行ではなく同期歩行を誘発
3. **問題の切り分け**: 爪先立ち問題はankle_pitchに限定されており、全関節の対称性を報酬化する必要はない

代わりに、問題の根本原因（ankle_pitchの過度な変位）を直接ペナルティ化する`foot_flat`報酬を提案する。

## まとめ

V11は**静止ポリシー回避に成功**し、X速度0.212 m/sを達成した：

- 報酬項目数: 22 → 15（7項目削減）で歩行回復
- V10の0.006 m/sから**3433%改善**
- Yawドリフト+5.35°で直進性も回復
- 単足接地率98.6%で交互歩行の基本パターンは確立

しかし、3つの歩行品質問題が顕在化した：

1. **内股（hip_roll + hip_yaw）**: hip_pos報酬削除により、hip_rollが両脚とも内側に偏向、hip_yawも右脚が非対称
2. **爪先立ち（ankle_pitch）**: foot_flat報酬削除により、右ankle_pitchがデフォルトから最大+28°逸脱
3. **小刻み歩行（hip_pitch）**: 左脚のhip_pitch rangeが12.5°と極小、相関+0.245で同期傾向

**問題と原因の対応関係**：

| 問題 | 原因となる関節 | データ | 解決策 |
|------|--------------|--------|--------|
| 内股 | hip_roll偏向 | L:常に負、R:常に正 | hip_pos復活(-0.3) |
| 　〃 | hip_yaw非対称 | R:常に負方向のみ | 同上 |
| 爪先立ち | ankle_pitch逸脱 | R:デフォルト+28° | foot_flat復活(-0.3) |
| 小刻み | hip_pitch低可動域 | L:0.218rad(12.5°) | single_foot_contact強化 |
| 　〃 | hip_pitch同期 | 相関+0.245 | air_time_offset調整 |

**重要な教訓**：

1. **最小報酬セットの罠**: 報酬項目の削減は静止ポリシー回避に有効だが、**必要な報酬を削除しすぎると別の問題が発生**する
2. **hip_posとfoot_flatの必要性**: これらは「品質を保証する報酬」であり、最小セットから外すべきではなかった
3. **symmetryの危険性**: 全関節の対称性を強制すると、hip_pitchの逆相関（交互歩行）も抑制してしまう。問題は関節ごとに切り分けて対処すべき
4. **「有効」の定義**: 最小有効報酬セットの「有効」とは、単に動くだけでなく**品質も保証する**ことを意味する

V12では、V11をベースにhip_posとfoot_flatを**復活**し、歩行品質の改善を図る。
