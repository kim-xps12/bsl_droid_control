# V28: air_time_offset延長 + contact報酬強化 + 内股対策強化

## 概要

V28では、V27の分析に基づき、タップダンス再発の原因であった`air_time_offset`短縮を元に戻し、さらに`contact`報酬強化と`hip_pos`ペナルティ強化を行う:

1. `air_time_offset`を0.15→0.25秒に延長（短縮副作用の解消）
2. `contact`を0.2→0.4に強化（位相整合性の強化）
3. `hip_pos`を-0.5→-0.8に強化（内股対策）
4. `action_rate`, `dof_acc`は維持（hip_pitch相関改善の成果を保持）

V27で発見された「エネルギーペナルティ緩和によるhip_pitch相関改善」は成功であり、これを継承しつつ、`air_time_offset`短縮の副作用を解消する。

## 設計詳細

### air_time_offset延長の根拠

**V27の問題分析**:
- air_time_offset = 0.15秒は小型ロボットでも報酬獲得が容易
- しかし、「足を上げてすぐ下ろす」動作でも報酬獲得可能に
- feet_air_time報酬のfirst_contact構造（接地時に報酬付与）との相互作用
- 同じ足で複数回接地すると複数回報酬を獲得 → タップダンス促進

**改善策**:
- air_time_offset = 0.25秒に延長
- 0.25秒以上の滞空がないと報酬獲得不可
- 「足を上げてすぐ下ろす」動作では報酬獲得できない
- 結果としてタップダンスが抑制される

### contact報酬強化の根拠

**V27の問題分析**:
- contact報酬 = 0.2は位相整合性への報酬が弱い
- V27でcontact報酬値が34%悪化（0.2007→0.1327）
- 正しいタイミングでの接地が報酬的に十分に重視されていない

**改善策**:
- contact報酬 = 0.4に強化（2倍）
- 歩行フェーズ（stance/swing）と実際の接地状態の整合性を重視
- 正しいタイミングでの接地を促進

### hip_pos強化の根拠

**V27の問題分析**:
- hip_pos報酬は38%改善（-0.0408→-0.0251）したが、目視では内股継続
- 数値改善と目視評価の乖離
- ペナルティが十分に強くない

**改善策**:
- hip_pos = -0.8に強化（60%増加）
- hip_yaw, hip_rollの過度な変位をより強くペナルティ
- 内股軌道を抑制

### エネルギーペナルティ維持の根拠

V27で発見した重要な成果:
- action_rate緩和（-0.01→-0.005）とdof_acc緩和（-2.5e-7→-1.0e-7）により、hip_pitch相関が大幅改善
- この成果を失わないために、エネルギーペナルティはV27値を維持

## 考察と改善案

### 成功点

1. **hip_pitch相関のさらなる改善**: -0.931 → -0.951に改善し、左右脚の交互性が向上
2. **contact報酬の大幅改善**: 0.1327 → 0.5500（スケール変更の影響を含むが、位相整合性が向上）
3. **安定した歩行**: エピソード長1001（最大）で転倒なし
4. **Y方向ドリフト改善**: -0.005 → 0.003 m/sに改善

### 課題

1. **タップダンス継続**:
   - feet_air_time = **-0.0427**（負の値）は滞空時間が`air_time_offset=0.25`秒より短いことを示す
   - air_time_offset延長（0.15→0.25）でも解消せず
   - **根本原因**: `feet_air_time`報酬の`first_contact`構造
     - 接地時にのみ報酬を付与するため、同じ足で複数回接地すると複数回報酬計算が行われる
     - 短い滞空でも「接地」すれば報酬（負でも）が発生
     - 結果として「足を上げてすぐ下ろす」動作が学習される

2. **内股軌道継続**:
   - hip_pos = -0.0292（ペナルティ継続）
   - swing_foot_lateral_velocity = -0.0088
   - hip_rollの非対称性: L=-0.220~0, R=0~0.253

3. **片足接地率低下**: 91.4% → 87.8%に低下

### 根本原因の特定

**タップダンスの根本原因は`feet_air_time`報酬の`first_contact`構造にある**:

```python
# 現在の実装
rew_air_time = (self.feet_air_time - self.air_time_offset) * self._first_contact.float()
```

- `first_contact`は接地開始時にのみTrueになる
- 同じ足で複数回接地すると複数回報酬計算が行われる
- air_time_offsetをいくら延長しても、「接地→短い滞空→接地→短い滞空→接地」のパターンでは報酬計算が複数回発生
- 結果として、タップダンスは構造的に解消されない

### 次バージョンへの提案（V29）

**アプローチ: `swing_duration`報酬の導入**

`first_contact`構造の問題を解決するため、「空中にいる間ずっと報酬」方式の新しい報酬項目を追加する:

```python
def _reward_swing_duration(self):
    """スイング期間中の累積報酬

    - 遊脚が空中にいる時間がair_time_offsetを超えた場合に報酬
    - 毎ステップ報酬を与えることで、長い滞空を促進
    - first_contactによる「接地直後の一回だけ報酬」ではないため、
      タップダンスのメリットがなくなる
    """
    reward_per_foot = torch.clamp(self.feet_air_time - self.air_time_offset, min=0.0)
    reward = torch.sum(reward_per_foot, dim=1) * is_moving
    return reward
```

| パラメータ | V28値 | V29値 | 変更理由 |
|-----------|-------|-------|---------|
| feet_air_time | 1.5 | **0** | first_contact構造を無効化 |
| swing_duration | - | **1.0** | 新規追加、空中報酬方式 |
| gait_frequency | 1.2 | **0.9** | 歩行周期延長（ゆったり） |
| step_length | 0.5 | **0.8** | 歩幅報酬強化（大股） |

## まとめ

V28では、V27の分析に基づき、以下の3点を変更した:

1. **air_time_offset**: 0.15 → 0.25秒に延長（タップダンス抑制）
2. **contact**: 0.2 → 0.4に強化（位相整合性の強化）
3. **hip_pos**: -0.5 → -0.8に強化（内股対策）

**設計根拠**:
- V27のair_time_offset短縮がタップダンスを促進した
- feet_air_time報酬のfirst_contact構造との相互作用が原因
- air_time_offset延長により、長い滞空を要求
- contact報酬強化により、正しいタイミングでの接地を促進
- hip_pos強化で内股軌道を抑制
- エネルギーペナルティはV27値を維持（hip_pitch相関改善の成果を保持）

**結果**:
- hip_pitch相関: -0.931 → -0.951（2%改善）
- contact報酬: 大幅改善（位相整合性向上）
- **タップダンス**: 継続（feet_air_time=-0.0427で負のまま）
- **内股軌道**: 継続
- **根本原因特定**: `feet_air_time`報酬の`first_contact`構造がタップダンスの根本原因
