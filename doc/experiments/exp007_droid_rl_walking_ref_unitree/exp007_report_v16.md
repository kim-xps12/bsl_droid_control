# V16: V15ベース + 振動抑制・大股歩行強化

## 概要

V15は0.198 m/sで歩行に成功し、静止ポリシー回避に成功した。しかし、「小刻み振動」という問題が残った（DOF range sum 1.834 rad、V3の2.43 radより小さい）。V16では**関節角速度の制御**と**ストライド長の増加**の2軸で「ゆったりと大股の歩容」を実現する。

**方針**:
- V15の静止ポリシー回避機構を維持
- `dof_vel`ペナルティで高周波振動を抑制
- `step_length`報酬で大股歩行を促進
- `action_rate`緩和と`air_time_offset`調整で大きな関節動作を許容
- 報酬項目数: 16 → 18項目（+2: dof_vel, step_length）

## 前バージョンからの改善事項

V15レポート（exp007_report_v15.md）の「次バージョン（V16）への提案」セクションに基づいて改善を実施。

### V15の問題分析

| 要素 | 観察された問題 |
|------|--------------|
| knee_pitch可動域 | 0.099 rad（V3の0.305 radの約1/3） |
| DOF range sum | 1.834 rad（V3の2.43 radより小さい） |
| hip_pitch可動域 | 0.246 rad（V3より改善、しかし膝が動いていない） |
| 目視評価 | 小刻み振動歩行 |

### V15の問題の根本原因

1. **dof_velペナルティ未使用**: 高周波振動が抑制されていない
2. **feet_air_timeオフセット0.5秒**: BSL-Droidには大きすぎ、滞空報酬を獲得しにくい
3. **action_rate=-0.01**: 大きな関節動作を抑制している

### V16での変更点（5点）

| パラメータ | V15値 | V16値 | 変更理由 |
|-----------|-------|-------|---------|
| **dof_vel** | (なし) | **-0.005** | 振動抑制 |
| **step_length** | (なし) | **0.5** | ストライド長報酬 |
| **action_rate** | -0.01 | **-0.005** | 大きな動作許容 |
| **swing_height_target** | 0.03 m | **0.04 m** | 足上げ誘導 |
| **air_time_offset** | 0.5秒 | **0.3秒** | 滞空報酬獲得しやすく |

### 5. 改善策の根拠

#### 5.1 dof_velペナルティの追加

**根拠**:
- V5/V6で使用され、足首振動抑制に効果があった
  - V4: ankle velocity std = 2.7-3.6 rad/s
  - V5（dof_vel追加後）: ankle velocity std = 0.2-0.6 rad/s
- 関節速度の二乗和にペナルティを与えることで、高周波振動を抑制

**unitree_rl_gymからの参考値**:
| ロボット | dof_vel | 備考 |
|---------|---------|------|
| H1_2 | -1e-3 | 二脚ヒューマノイド |
| G1 | -1e-3 | 二脚ヒューマノイド |

BSL-Droidは小型ロボットのため、`-0.005`（unitreeの5倍）を採用。

#### 5.2 step_length報酬の追加

**根拠**:
- 左右の足間距離（前後方向）を報酬化し、大股歩行を促進
- V7で環境クラスに実装済み
- V15は交互傾向（hip_pitch相関-0.165）があるため、効果が期待できる

#### 5.3 action_rateの緩和

**根拠**:
- 現在のペナルティ（-0.01）が大きな関節動作を抑制している可能性
- V7では-0.005を使用し、より大きな動作を許容
- 緩和することで、大きな歩幅を取る動作が許容される

#### 5.4 air_time_offsetの調整

**根拠**:
- 現在のオフセット0.5秒はUnitree G1/H1（大型ロボット）向け
- BSL-Droidの歩行周期1.0秒（gait_frequency=1.0Hz）では遊脚期は約0.45秒
- 0.5秒オフセットでは遊脚期全体でも報酬が得られにくい

**報酬計算式**:
```python
rew_airTime = (self.feet_air_time - offset) * first_contact
# offset=0.5: 0.45秒滞空 → 報酬-0.05（ペナルティ）
# offset=0.3: 0.45秒滞空 → 報酬+0.15（報酬）
```

#### 5.5 swing_height_targetの増加

**根拠**:
- 足上げが不十分で大股が取れていない
- unitree H1_2: 目標0.08m（BSL-Droidは小型のため半分程度が適切）
- 0.03m → 0.04mに増加して足上げを誘導

## 考察と改善案

### 成功点

1. **数値的指標は大幅に改善**:
   - DOF range sum: 1.834 → 2.724 rad（+48%）
   - knee_pitch可動域: 0.099 → 0.201-0.290 rad（2-3倍）
   - 関節速度std: 全て < 0.8 rad/s

2. **hip_pitch相関が改善**: -0.165 → -0.419（交互歩行傾向の強化）

3. **X速度は目標範囲内**: 0.185 m/s

### 課題

#### 1. 「前足と後ろ足が固定される」問題

**データからの根拠**:
- hip_pitch可動域: 左=0.524 rad vs 右=0.197 rad（**約2.7倍の差**）
- 左脚だけが大きく動き、右脚はほとんど固定されている
- これは目視で「前足と後ろ足が固定される」として観察される

**原因分析**:
- V16では**左右対称性を促す報酬がない**
- ポリシーが「片脚を大きく動かす」という局所最適に収束
- symmetry報酬がないため、非対称動作が許容されている

#### 2. 「細かい振動」問題

**データからの根拠**:
- 接地パターン: 100%が両足空中（実際にはジャンプ動作？）
- DOF速度std: 0.3-0.8 rad/s（振動成分が存在）

**原因分析**:
- dof_vel=-0.005を追加したが、振動を完全には抑制できていない
- 接地検出が機能していないため、スタンス相での安定化ができていない

#### 3. feet_air_time/single_foot_contact報酬が機能していない

**データからの根拠**:
- 報酬成分: feet_air_time=0.0, single_foot_contact=0.0
- 接地パターン: 100%が両足空中

**原因分析**:
- contact_threshold=0.025mが足の実際の高さに対して適切でない可能性
- または、air_time_offset=0.3秒の変更が環境クラスで正しく反映されていない
- Genesis物理エンジンでの接地力検出の問題

#### 4. Yawドリフトの悪化

**データからの根拠**:
- V15: -4.92° → V16: +13.57°（方向が逆転し、絶対値も増加）

**原因分析**:
- hip_pitch可動域の非対称により、左右の推進力が異なる
- 左脚が大きく動くことで、Yaw方向のトルクが発生

### サーベイ（exp007_unitree_rl_gym_survey.md）からの知見

#### 7.2.3 Symmetry Equivariant Policy（対称性報酬）

> 「対称性を『報酬shaping』で誘導する従来手法はソフト制約に過ぎず、完全な対称性は保証されない」

**適用**: symmetry報酬を追加して左右脚の対称動作を促進

#### 7.4 Yawドリフト抑制

> 「直線歩行を実現するために、Yaw角と横方向ドリフトの両方を修正する必要がある」

**適用**: tracking_ang_velの強化、またはYaw修正報酬の追加

#### 8.2 Single Foot Contact Reward

> 「移動コマンド時に『片足のみ接地』を報酬化」

**問題**: 現在の実装では接地検出自体が機能していないため、この報酬も機能していない。

### 次バージョンへの提案

#### V17への改善策（優先度順）

**優先度1（必須）: 左右対称性の強化**

```python
# symmetry報酬の追加
"symmetry": 0.3,  # 左右脚の対称動作を報酬化
```

**実装案**:
```python
def _reward_symmetry(self):
    """左右対称性報酬"""
    # 左右のhip_pitch位相差を評価
    left_hip_pitch = self.dof_pos[:, 2]   # left_hip_pitch_joint
    right_hip_pitch = self.dof_pos[:, 7]  # right_hip_pitch_joint

    # 理想的には180°位相差（半周期ずれ）
    # 可動域の差を最小化
    range_diff = torch.abs(
        torch.abs(left_hip_pitch - left_hip_pitch.mean()) -
        torch.abs(right_hip_pitch - right_hip_pitch.mean())
    )
    return -range_diff
# scale: 0.3
```

**優先度2（推奨）: 接地検出の修正**

```python
# contact_thresholdの調整
"contact_threshold": 0.035,  # 0.025 → 0.035 m（より緩い閾値）
```

または、環境クラスで接地検出のデバッグログを追加して原因を特定。

**優先度3（推奨）: Yawドリフト対策**

```python
# tracking_ang_velの強化
"tracking_ang_vel": 0.8,  # 0.5 → 0.8
```

または、明示的なYaw安定化報酬を追加:
```python

```

現在の-0.005では振動抑制が不十分。

## まとめ

V16は、V15の「小刻み振動」問題を解決するため、以下の2軸アプローチを採用した：

| 軸 | 改善策 | 変更内容 | 効果 |
|---|--------|---------|------|
| **振動抑制** | dof_vel追加 | (なし) → -0.005 | 部分的効果（関節速度std < 0.8） |
| **大股歩行** | step_length追加 | (なし) → 0.5 | ✅ DOF range sum 1.834→2.724 |
| **大股歩行** | action_rate緩和 | -0.01 → -0.005 | ✅ 関節可動域増加 |
| **大股歩行** | swing_height増加 | 0.03 → 0.04 m | 効果不明 |
| **大股歩行** | air_time_offset調整 | 0.5 → 0.3秒 | ❌ 報酬0.0（機能せず） |

### 達成したこと

- **数値的指標は大幅改善**: DOF range sum +48%、knee_pitch可動域 2-3倍
- **hip_pitch相関改善**: -0.165 → -0.419（交互傾向強化）
- **関節速度抑制**: 全関節で < 0.8 rad/s

### 達成できなかったこと

- **目視で「ゆったり大股」を実現できず**: 細かい振動と脚固定が継続
- **左右対称性の確保**: hip_pitch可動域 L=0.524 vs R=0.197（約2.7倍の差）
- **接地報酬の機能**: feet_air_time, single_foot_contactが共に0.0
- **Yawドリフト抑制**: -4.92° → +13.57°（悪化）

### 根本原因

1. **symmetry報酬の欠如**: 左右非対称動作が許容され、片脚固定の局所最適に収束
2. **接地検出の問題**: contact_threshold=0.025mが不適切、または物理エンジンの問題
3. **非対称推進力**: 左脚のみ大きく動くことでYawトルクが発生

### 次バージョン（V17）への主要提案

1. **symmetry報酬追加** (0.3): 左右対称性を明示的に報酬化
2. **contact_threshold調整** (0.025→0.035m): 接地検出の修正
3. **tracking_ang_vel強化** (0.5→0.8): Yawドリフト対策
