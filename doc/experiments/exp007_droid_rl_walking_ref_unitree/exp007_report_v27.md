# V27: エネルギーペナルティ緩和 + air_time_offset短縮

## 概要

V27では、V26の根本原因分析に基づき、「案A（推奨）」を採用する:

1. `action_rate`を-0.01→-0.005に緩和（遊脚維持コスト半減）
2. `dof_acc`を-2.5e-7→-1.0e-7に緩和（加速度ペナルティ緩和）
3. `air_time_offset`を0.2→0.15秒に短縮（報酬獲得閾値を下げる）

V26の分析により、タップダンスの根本原因は「エネルギー最小化ペナルティが滞空報酬より優位」であることが判明した。本バージョンでは、エネルギーペナルティを緩和することで報酬バランスを是正し、「足を上げ続ける」動作が報酬的に有利になる構造を目指す。

### V26で観測された課題

1. **タップダンス継続・悪化**: feet_air_time報酬を1.0→1.5に強化したにも関わらず、報酬値は-0.0205→-0.0275と**35%悪化**した
   - hip_pitch相関も-0.851→-0.318と大幅に悪化

2. **内股軌道継続**: hip_pos報酬は-0.0471→-0.0408と13%改善したが、目視では内股軌道が依然として観察される

3. **歩行の非対称性**: hip_yaw、hip_rollの左右非対称な可動域が顕著

### V26から得られた教訓

1. **根本原因の特定**: 仮説B「エネルギー最小化が優位」が根本原因
   - 仮説A「左右非対称動作が原因」は症状
2. エネルギー関連ペナルティ（action_rate, dof_acc）が滞空報酬より優位なため、遊脚中にトルクを抜いて「サボる」動作が報酬最大化となりタップダンスが発生
3. feet_air_timeスケール強化だけでは滞空時間は延びない（エネルギーペナルティ優位のため）
4. **報酬バランスの是正**が根本解決に必要

## 設計詳細

### エネルギーペナルティ緩和の根拠

**問題の分析**:
- V26の報酬バランス:
  - feet_air_time報酬: -0.0275（スケール1.5、実質-0.018/step）
  - action_rate + dof_accペナルティ: -0.0072
- 遊脚中に足を上げ続けるには関節を能動的に制御する必要があり、action_rate/dof_accペナルティが発生
- 一方、トルクを抜いて足を下ろせばペナルティ回避可能
- 結果: 「足を上げ続ける」より「素早く上げ下げする」方が報酬的に有利 → タップダンス発生

**改善策**:
1. **action_rate緩和**（-0.01→-0.005）: 遊脚維持のコストを半減
2. **dof_acc緩和**（-2.5e-7→-1.0e-7）: 加速度ペナルティを緩和
3. **air_time_offset短縮**（0.2→0.15秒）: 小型ロボットでも報酬獲得が容易に

**期待効果**:
- エネルギーペナルティ緩和により、「足を上げ続ける」動作が報酬的に有利に
- 滞空時間延長 → タップダンス解消
- 左右非対称動作も自然に改善される可能性（根本原因への対処により）

### 副作用のリスク評価

1. **action_rate緩和**: 高周波振動のリスクがあるが、-0.005でも十分な抑制効果を維持
2. **dof_acc緩和**: 動作の滑らかさが若干低下する可能性があるが、許容範囲内

## 考察と改善案

### 成功点

1. **エネルギーペナルティ緩和の効果**:
   - action_rate, dof_accの報酬値が改善（51%, 69%改善）
   - feet_air_time報酬も49%改善
   - hip_pitch相関が-0.318→**-0.931**と大幅改善（左右交互歩行が改善）

2. **X速度の微改善**: 0.208→0.217 m/s（+4%）

### 課題

1. **タップダンス再発（悪化）**:
   - ユーザー目視評価: 「右足で地面を2回叩いて、左足で一回叩く」パターン
   - hip_pitch相関は改善したが、**片足内での2回接地**が発生
   - 位相整合性（contact報酬）が34%悪化

2. **内股軌道継続**:
   - hip_pos報酬は38%改善したが、目視では内股軌道が依然として観察される
   - 数値改善と目視評価の乖離

3. **速度追従・歩行品質報酬の低下**:
   - tracking_lin_vel: 34%悪化
   - contact: 34%悪化
   - single_foot_contact: 33%悪化

### 根本原因の再分析

**仮説B「air_time_offset短縮の副作用」が有力**:

1. **v27でair_time_offsetを0.2→0.15秒に短縮**
2. **短い滞空時間でも報酬獲得可能に**
3. **feet_air_time報酬のfirst_contact構造**との相互作用:
   - first_contact時に報酬が付与される構造
   - 同じ足で複数回接地すると複数回報酬を獲得
   - 「タタン、タン」パターンは右足で2回first_contactを得て、左足で1回得るパターン
4. **結果**: 「足を上げてすぐ下ろす」動作でも報酬獲得可能になり、タップダンスが促進

**矛盾の解消**:
- hip_pitch相関改善（左右交互性）と「タップダンス再発」は矛盾しない
- 左右の交互性は保たれている（-0.931）が、**右足が1サイクル内で2回接地**している
- これはhip_pitch相関では検出できない

### 次バージョンへの提案（V28）

**案A（推奨）: air_time_offset延長 + contact報酬強化**

| パラメータ | V27値 | V28値 | 変更理由 |
|-----------|-------|-------|---------|
| air_time_offset | 0.15 | **0.25** | 短縮副作用の解消、長い滞空を要求 |
| contact | 0.2 | **0.4** | 位相整合性の強化 |
| action_rate | -0.005 | -0.005 | 維持（hip_pitch相関改善を保持） |
| dof_acc | -1.0e-7 | -1.0e-7 | 維持 |
| hip_pos | -0.5 | **-0.8** | 内股対策強化 |

**設計根拠**:
1. air_time_offset延長により、「足を上げてすぐ下ろす」動作では報酬獲得不可に
2. contact報酬強化により、正しいタイミングでの接地を促進
3. エネルギーペナルティはv27値を維持（hip_pitch相関改善の成果を保持）
4. hip_pos強化で内股軌道を抑制

## まとめ

V27では、V26の根本原因分析に基づき、以下の3点を変更した:

1. **action_rate**: -0.01 → -0.005に緩和（遊脚維持コスト半減）
2. **dof_acc**: -2.5e-7 → -1.0e-7に緩和（加速度ペナルティ緩和）
3. **air_time_offset**: 0.2 → 0.15秒に短縮（報酬獲得閾値を下げる）

**設計根拠**:
- V26の分析により、タップダンスの根本原因は「エネルギー最小化ペナルティが滞空報酬より優位」と特定
- 「足を上げ続ける」動作が報酬的に有利になる報酬構造への転換を目指す

**結果**:
- **部分的成功**: hip_pitch相関が-0.318→**-0.931**と大幅改善（左右交互歩行の改善）
- **失敗**: タップダンス再発（右足2回蹴り）、内股軌道継続
- **原因**: air_time_offset短縮の副作用により、短い滞空でも報酬獲得可能になりタップダンスが促進

**教訓**:
- エネルギーペナルティ緩和は左右交互歩行（hip_pitch相関）に効果あり
- air_time_offset短縮は逆効果（feet_air_time報酬のfirst_contact構造との相互作用でタップダンス促進）
- V28ではair_time_offsetを延長し、contact報酬を強化する方針

## 備考

- V24〜V26の3バージョンで「報酬スケール強化」アプローチを試したが効果がなかった
- V27では「エネルギーペナルティ緩和」により報酬バランスを是正するアプローチに転換
- **V27の成果**: エネルギーペナルティ緩和によるhip_pitch相関改善はV28に継承する
- **V27の失敗**: air_time_offset短縮はタップダンスを促進したため、V28では延長する
