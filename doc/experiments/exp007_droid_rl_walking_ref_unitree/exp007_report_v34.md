# V34: symmetry_rangeの無効化（V32ベースへの回帰）

## 概要

V34では、V33で発生したsymmetry_range報酬の副作用（振幅縮小→タップダンス化、Yawドリフト悪化）を解消するため、symmetry_rangeを無効化してV32ベースに回帰する。

V33でsymmetry_range=0.3を有効化した結果、左右対称性は達成されたものの、「振幅を大きくする」のではなく「大きい方を小さくする」学習戦略が採用され、DOF range sum -15%、Yawドリフト -2.00°→-18.69°という深刻な悪化が発生した。この"race to bottom"効果は報酬shapingによる対称性誘導の構造的限界を示しており（サーベイSection 7.2.3と一致）、symmetry_rangeの無効化が最も安全な回帰策である。

**1変更1検証の原則**に従い、今回はsymmetry_rangeの無効化のみを行う。

## 前バージョンからの改善事項

### V33で観測された課題

1. **振幅の縮小による「タップダンス」化**
   - hip_pitch range: 左0.716→0.396 rad（-45%）、右0.616→0.431 rad（-30%）
   - symmetry_rangeの"race to bottom"効果が原因
   - 振幅が小さくなったため、足が十分に持ち上がらず地面を叩く動作が発生

2. **Yawドリフトの大幅悪化**: -2.00° → -18.69°
   - 振幅縮小による前進推力の低下が主因

3. **報酬項目数の超過**: 21項目（推奨上限15-17項目を超過）

### 改善策（V34）

V33レポートの「次バージョンへの提案」セクションの**推奨案A: symmetry_range=0に戻す（V32ベースへの回帰）**を採用。

| パラメータ | V33値 | V34値 | 変更理由 |
|-----------|-------|-------|---------|
| symmetry_range | 0.3 | **0** | 副作用（振幅縮小）を解消、V32ベースに回帰 |
| 他のパラメータ | - | - | V33/V32から維持 |
| 報酬項目数 | 21 | **20** | symmetry_range無効化 |

## 設計詳細

### symmetry_range無効化の根拠

1. **副作用の深刻さ**: symmetry_rangeは左右均等化に成功したが、振幅縮小→タップダンス化→Yawドリフト悪化という連鎖的な副作用が発生
2. **V32の実績**: symmetry_range=0のV32は現時点でYawドリフト最良（-2.00°）、X速度0.175 m/sを達成
3. **構造的限界**: 報酬shapingによる対称性誘導は対称化の「方法」を制御できない（サーベイSection 7.2.3）
4. **項目数削減**: 21→20項目に削減し、推奨上限（15-17項目）に近づける

## 考察と改善案

### 成功点

1. **V33の振幅縮小の解消**: DOF range sumがV33の4.830 radからV34の6.021 radに回復し、V32の5.651 radを上回った。symmetry_range無効化による振幅回復は成功した
2. **X速度の回復**: 0.173 m/sはV32の0.175 m/sとほぼ同水準であり、V33の0.161 m/sからの回復を確認した
3. **Yawドリフトの改善**: V33の-18.69°から+7.35°に大幅改善した。ただし方向がV32の-2.00°と逆転しており、V32水準には未達
4. **タップダンスの片側化**: ユーザー目視で「左足だけ」に改善を確認。hip_pitch velocity std比が1.52（V33の0.95から非対称に戻る）であり、左足の活動が右足より大きい状態に戻った
5. **hip_pitch相関の安定**: -0.602はV32の-0.544と同水準であり、V33の-0.964（過度に強い交互歩行）から適度な水準に戻った

### 課題

1. **Yawドリフトの残存**: +7.35°はV32の-2.00°と比較して悪化している。方向も逆転（負→正）しており、RL訓練の確率的変動の影響がある
2. **両足接地率の非回復**: 4.4%はV33と同率で、V32の2.8%に回復していない。タップダンス自体は左足に限定されたが、発生頻度はV32水準に戻っていない
3. **左右非対称性の残存**: hip_pitch velocity std比=1.52（左>右）。V32の1.82よりは改善しているが、完全な対称性（1.0）には遠い
4. **Base heightの低下**: 0.250 m（V32: 0.262 m）。重心が12 mm下がっており、Roll stdも9.20°（V32: 8.38°）と増加している
5. **報酬項目数**: 20項目は推奨上限（15-17項目）を依然として超過している

### 動作メカニズムの因果分析

#### Yawドリフト方向逆転の要因

CSV時系列分析により、Yawドリフトの因果構造は当初の想定より複雑であることが判明した。

**full-duration vs 定常状態のhip_pitch非対称の乖離**: full-duration（0-10s）では右hip_pitch rangeが左を上回る（0.742 vs 0.660 rad）が、定常状態（t>2s）では左が右を上回る（0.660 vs 0.587 rad、非対称度+5.9%）。これは過渡状態（0-2s）で右脚がより大きな振幅を示すためであり、定常状態のランニング動態と過渡応答を区別して分析する必要性を示す。

**Yaw振動の存在**: V34のYawは-4.96°から+7.35°まで振動しており、単調なドリフトではない。V32も-7.48°から+5.04°の範囲で振動する。最終値（V34: +7.35°, V32: -2.00°）は振動の位相に依存しており、ドリフト速度（V34: +1.42°/s, V32: -0.30°/s）がより信頼性の高い指標である。

**左右位相リーダーの逆転**: 相互相関ラグの符号が逆転（V34: R先行, V32: L先行）しており、歩行の主導脚が入れ替わっている。定常状態のknee_pitch非対称度はV34で+21.4%とV32の+15.5%を上回り、膝関節の左右差が拡大している。hip_pitch rangeの左右関係が逆転したのは、同一の報酬構成でも異なるランダムシードによるRL訓練の確率的変動で説明される。symmetry_rangeの有無とは独立した現象である。

#### タップダンス残存の構造的要因

CSV時系列分析で、タップダンスの発生メカニズムをより詳細に特定できた。

**定量的特徴**: 定常状態で20回の両足接地イベントが検出されたが、全て1-2 steps（0.02-0.04s）の瞬間的な同時接地であり、持続的に両足が地面についている状態ではない。これは歩行サイクルの脚交換時に発生する「オーバーラップ」に近い。

**左右スイング時間の非対称**: 右足スイング（左足接地時）平均0.267s vs 左足スイング（右足接地時）平均0.245sで、右足スイングが0.022s長い。左足の接地時間がやや長いことは、左足がスイング開始時に地面を離れにくい傾向と整合する。

**関節角速度の非対称**: knee_pitchのRMS速度比がL/R=2.09と極めて大きく、左膝の運動が右膝の2倍以上活発。しかしankle_pitchはL/R=0.80と逆転しており、右足首が左より活発。この「膝は左優位、足首は右優位」のパターンは左足タップダンスの力学と一致する可能性がある（左膝が大きく動くが足首は地面を離しきれない）。

両足接地率4.4%が維持された構造的要因:

1. **leg_phase初期値の固定**: 左leg_phase=0、右leg_phase=0.5の固定初期化が、左右脚に異なる学習軌跡を与える
2. **MLPアーキテクチャの限界**: 512-256-128のMLPは左右入力に対して対称な出力を保証しない（サーベイSection 7.2.3）
3. **swing_contact_penalty=-0.5の到達点**: V30/V31の実験で、-0.5が片側のタップダンス抑制に有効な最低閾値であることが判明しているが、もう片側への効果は不十分

### サーベイ知見との照合

1. **Section 7.2.3（対称等変ポリシー）**: V34の結果は、symmetry_range無効化によるV32ベース回帰が妥当であったことを確認した。左右非対称性の根本解決には等変ネットワークアーキテクチャが必要であるという知見と一致する

2. **Section 7.4.1（Yaw速度コマンド訓練）**: V34のYawドリフト+7.35°は、Yaw速度コマンド=0での訓練限界を示している。意図的にYaw速度コマンドを変動させる訓練（Section 7.4.1）はYawドリフト修正能力の獲得に有効と予測される。ただし中期的改善策であり、V35の範囲外

3. **Section 7.4.3（Symmetric Gait Reward）**: 位相差と足間距離に基づく対称性報酬は、symmetry_rangeの振幅ベースアプローチとは異なる。"race to bottom"のリスクが低い可能性があるが、検証は未実施

4. **Section 8.4（ROGER）**: 報酬スケールの自動調整は、20項目の手動チューニングの限界を超えるための将来的選択肢。ただし実装コストが高く、現段階では手動での削減・調整が現実的

5. **Section 7.6（改善策の優先順位）**: V34の段階では、短期改善策（タップダンス抑制の強化）と報酬項目削減の組み合わせが最も効果的と判断される

### 次バージョンへの提案

#### 推奨案: swing_contact_penalty強化（-0.5 → -0.7）

- **目的**: 残存する左足タップダンスの直接抑制
- **根拠**: V30で-0.5が右足タップダンス抑制に有効と実証済み。V32/V34で左足が残存していることから、より強いペナルティが必要
- **リスク**: 中。-0.7は未検証領域だが、-0.5→-0.7の変化幅はV30→V31（-0.5→-0.3）の逆方向と同程度であり、1変更1検証の原則に適合
- **予測効果**: 両足接地率の低下（4.4%→目標2%以下）、左足の地面叩き動作の低減

#### 代替案A: negligible報酬項目の削減（20→18項目）

- **削除候補**: dof_vel_limits（0.0000）、torques（-0.0002）
- **目的**: 報酬構造の簡素化、推奨項目数への段階的接近
- **リスク**: 低。両項目は実質的にゼロであり、削除の影響は無視可能
- **留意点**: 1変更1検証の原則においてゼロ寄与項目の削除が「変更」にカウントされるかは判断が分かれる

#### 代替案B: swing_contact_penalty強化 + ゼロ寄与2項目削除の同時実施

- **変更**: swing_contact_penalty: -0.5→-0.7、dof_vel_limits削除、torques削除
- **目的**: 実質的にはswing_contact_penalty強化のみ（ゼロ寄与項目削除はno-op）
- **リスク**: 低〜中。ゼロ寄与項目の削除は機能的にno-opであり、swing_contact_penalty強化が唯一の実質変更

## まとめ

V34はsymmetry_range=0への回帰により、V33の振幅縮小副作用（DOF range sum: 4.830→6.021 rad）とYawドリフト悪化（-18.69°→+7.35°）を大幅に改善した。X速度は0.173 m/sでV32水準（0.175 m/s）をほぼ回復し、タップダンスも左足のみに限定された。

CSV時系列分析により以下の新知見を得た:
- 歩行周期は0.88 Hz（1.143s）でV32と完全一致し、報酬変更による歩行基本構造の安定性を確認
- 両足接地イベントは全て1-2 steps（0.02-0.04s）の瞬間的オーバーラップであり、持続的な両足接地ではない
- Yawは-5°〜+7°の範囲で大きく振動しており、最終値（+7.35°）よりもドリフト速度（+1.42°/s）がより信頼性の高い評価指標
- 定常状態のhip_pitch非対称度はV34で+5.9%（L>R）とV32の+13.9%から改善。ただしknee_pitchは+21.4%と拡大
- knee_pitchのL/R角速度比が2.09と極端で、左膝の過剰活動と足首の左右逆転パターンが左足タップダンスの力学メカニズムを示唆

報酬設計の体系的整理により、20項目中6項目がnegligible（寄与率2%未満）であることを特定した。V35ではswing_contact_penaltyを-0.5→-0.7に強化して左足タップダンスの直接抑制を試み、V36で不要項目の削減（20→16項目）を行うロードマップを提案する。
