# V19: 目標速度引き上げ（V15ベース）

## 概要

V15の報酬構成（base_vel_y含む18項目）・PDクランプ設定をそのまま維持し、目標速度のみを引き上げる実験。V15ではX速度が0.146 m/sと目標範囲[0.15, 0.25]の下限付近にとどまっていたため、目標速度を+0.10 m/sシフトしてポリシーがより速い歩行を学習するかを検証する。

## 前バージョンからの改善事項

V15をベースに、目標速度のみを変更する。報酬構成・PDクランプは一切変更なし（1変更1検証原則）。

| パラメータ | V15値 | V19値 | 変更理由 |
|-----------|-------|-------|---------|
| lin_vel_x_range | [0.15, 0.25] | [0.25, 0.35] | V15実績X=0.146m/sが目標下限付近。より高い目標速度でポリシーの速度追従能力を検証 |

## 設計詳細

### 目標速度変更の根拠

V15の評価結果:
- X速度（定常mean）: 0.146 m/s（目標範囲[0.15, 0.25]の下限を下回る）
- tracking_lin_vel報酬: 1.2357（正報酬中2位）
- velocity_deficit: -0.0033（小さい＝速度未達ペナルティは軽微）

目標速度を[0.25, 0.35]に引き上げることで:
1. ポリシーがより大きな歩幅を学習する可能性
2. 歩行周波数の変化（0.88Hz→上昇の可能性）
3. velocity_deficitペナルティ増加による速度追従圧力の強化

### 報酬構成（V15と同一）

有効報酬項目数: 18（V15から変更なし）

主報酬:
- tracking_lin_vel: 1.5
- tracking_ang_vel: 0.5

歩行品質報酬:
- swing_duration: 2.0, contact: 0.4, single_foot_contact: 0.5
- step_length: 0.8, swing_contact_penalty: -0.7

安定性ペナルティ:
- lin_vel_z: -2.0, ang_vel_xy: -0.1, orientation: -2.0, base_height: -5.0

歩行品質ペナルティ:
- feet_swing_height: -8.0, contact_no_vel: -0.1, hip_yaw_pos: -0.8
- stance_foot_lateral_position: -0.5, velocity_deficit: -0.5
- foot_lateral_velocity: -0.5, base_vel_y: -1.0

エネルギー効率:
- action_rate: -0.005

PDクランプ: hip_roll_inward_limit = -0.05 rad（V14から維持）

## 考察と改善案

### 成功点

1. **X速度の大幅向上**: 0.162→0.255 m/s (+57.7%)。目標速度引き上げ[0.25, 0.35]に対して0.255 m/sは目標範囲の下限に達しており、V15での0.146 m/s（目標範囲[0.15, 0.25]の下限未達）から大きく改善。目標速度の引き上げが速度追従に直接寄与することを確認
2. **歩行周波数維持**: 0.88 Hz。速度向上は主に歩幅増大によるものであり、歩行周波数を変えずに達成された
3. **swing_duration改善**: +15.4% (0.212→0.245)。V15の歩幅短縮傾向が回復方向に向かった
4. **swing_contact_penalty改善**: -18.6% (0.046→0.038)。遊脚接地タイミングの規則性向上
5. **内股着地維持**: stance hip_roll L -9.6°, R +10.3°。PDクランプの効果が速度変更に影響されないことを確認
6. **Yawドリフト絶対値改善**: |21.07|°→|16.08|°（符号反転はRL確率性）。ドリフト速度は|2.57|→|3.80|°/sと微増だが、最終Yaw絶対値は小さくなった
7. **action_rate改善**: -8.1%。速度追従圧力が強まったことでジャーク動作が減少
8. **hip_pitch非対称度改善**: |16.3%|→|6.2%|。左右の歩幅差が縮小

### 課題

1. **hip_pitch相関の大幅悪化（-0.804→-0.619、-23.0%）**
   - 交互歩行の質が明確に低下。L/R接地比率の非対称化（L 43.5% vs R 53.8%）と連動
   - R swing duration 0.488→0.395s (-19.1%)、L swing duration 0.438→0.482s (+10.0%): 右足の振り出し時間が短縮し、左右のタイミングが非対称化
   - ローリング相関のstdも0.030→0.089に増大（3倍）し、位相関係の不安定性を示す

2. **横方向揺れの悪化（lateral std +29.7%、R² -49.6%）**
   - detrended lateral std: 44.7→58.0 mm (+29.7%)。p2p: 168→235 mm (+39.8%)
   - Roll-横方向R²: 0.751→0.379。残差std: 22.3→45.8 mm (+105.4%)
   - R²低下 + 残差倍増は、横方向揺れの非Roll成分（L/R非対称歩行パターンによる運動学的揺れ）が大幅増加したことを示す

3. **Roll std悪化（+15.0%、5.18→5.95°）**
   - Roll角速度RMS +14.5%。速度向上に伴う運動エネルギー増大がRoll振動を増幅
   - ang_vel_xy +4.3%（小幅だが増加方向）

4. **foot_lateral_velocity悪化（+43.2%）**
   - 最も増加率が大きいペナルティ。速度向上により足先の横方向速度成分が増大
   - これが横方向揺れ悪化の直接的な寄与因子の一つ

### 根本原因分析

**1. X速度向上と歩行品質のトレードオフ構造**

目標速度引き上げにより、ポリシーはtracking_lin_vel報酬を最大化するために歩幅を増大させた。しかし、歩行周波数は0.88 Hzで維持されたため、速度向上は純粋に歩幅増大によるものである。歩幅増大は以下のカスケード効果を発生させた:

```
目標速度↑ → 歩幅↑（周波数維持）
  → 遊脚振り出し角度↑ → hip_pitch range L 0.567→0.866 rad (+52.7%)
  → 運動エネルギー↑ → Roll振動↑ (+15%)
  → 足先横方向速度↑ → foot_lateral_velocity +43.2%
  → L/R非対称化 → hip_pitch corr -23%
```

**2. hip_pitch相関悪化のメカニズム**

V15のhip_pitch相関-0.804は、左右均等な接地比率（L 49.8%, R 48.8%）と均等なswing duration（L 0.438s, R 0.488s）に支えられていた。V19では:
- R swing duration短縮（-19.1%）とL swing duration伸長（+10.0%）
- 接地比率の非対称化（L 43.5% vs R 53.8%）
- 相互相関ラグ変化（0.540s→0.440s）: 位相タイミング自体が変化

サーベイSection 7.2.3の指摘通り、MLPは左右対称出力を保証しないため、速度追従圧力が増すとポリシーが非対称な戦略を採用しやすくなる。特にtracking_lin_vel報酬は左右対称性を考慮しないため、片側の足で強く蹴り出す方がエネルギー効率が良い場合、その戦略が選択される。

**3. R²低下の解釈**

V15のR²=0.751（横方向揺れの75%がRoll角で説明可能）からV19のR²=0.379（38%のみ）への低下は、横方向揺れの主因が変化したことを示す。V19では残差std=45.8mm（Roll成分std=35.7mmを上回る）であり、非Roll成分が主体となっている。これはL/R非対称歩行パターンによる運動学的揺れ（左右の足の出し方が非対称なために、接地位置が左右非均等になる揺れ）であり、Rollペナルティ（ang_vel_xy）やbase_vel_yでは直接制御できない成分である。

### サーベイ知見との照合

1. **tracking_lin_vel報酬のGaussian特性（サーベイ2.1.1）**: tracking_sigma=0.25のGaussian報酬は、目標速度が高いほどerrorが大きくなり報酬値が低下する。V19のtracking_lin_vel=1.1972（V15: 1.2357、-3.1%）は、目標速度[0.25, 0.35]に対して実速度0.255 m/sが目標範囲の下限であることと整合する。ポリシーは速度追従を改善したが、Gaussian報酬の性質上、高い目標速度での報酬取得は構造的に困難
2. **velocity_deficit増加（+63.6%）**: サーベイ8.7で設計された速度未達ペナルティが、目標速度引き上げにより機能した。ただしvelocity_deficit=-0.0054は依然小さく、速度追従圧力の主体はtracking_lin_velである
3. **MLP非対称性（サーベイ7.2.3）**: hip_pitch相関の悪化は、equivariant networkなしのMLPでは高速歩行時に左右対称性が崩れやすいという予測と一致する
4. **ロボットサイズと目標速度（サーベイ7.3）**: BSL-Droid Simplified (5.8kg)の適正速度範囲は0.1-0.3 m/s。V19の目標[0.25, 0.35]は上限付近であり、0.255 m/sの実現は妥当な結果

### 次バージョンへの提案

**推奨案A: 歩行周波数の引き上げ（gait_frequency: 0.9→1.2 Hz）**

- V19ではX速度向上が歩幅増大のみで達成され、歩行周波数は0.88 Hzに固定された。歩幅増大はhip_pitch range +52.7%、foot_lateral_velocity +43.2%、L/R非対称化といった副作用を伴った
- 歩行周波数を引き上げることで、歩幅を抑えつつ速度を維持する戦略への移行を促す
- 歩行周波数↑ → 1歩あたりの振り出し角度↓ → 運動エネルギー↓ → Roll振動↓、foot_lateral_velocity↓
- さらに高い周波数はswing duration短縮により接地切替が速くなり、L/R位相関係の安定化（hip_pitch相関改善）が期待される
- 0.9→1.2 Hzは+33%の変更だが、gait_frequencyはcontact報酬・swing_duration報酬の位相基準にのみ作用し、報酬項目の追加/削除を伴わないため、1変更1検証原則に適合
- リスク: 周波数が高すぎると小刻みな歩行（V15で観察されたPitch std悪化と類似のパターン）になる可能性。また、0.88 Hzは本ロボットの自然共振周波数である可能性があり、強制的な周波数変更がgait品質を低下させるリスクがある

**代替案B: 位相条件付きPDクランプ（V15レポート推奨案A）**

- 接地相のみhip_roll_inward_limit=-0.05 radを適用し、遊脚相はクランプ解除
- 遊脚相のhip_roll自由度を回復し、V4レベルの位相依存バランス機構を復元
- 横方向揺れの根本原因（PDクランプによるhip_roll動的範囲制限）に直接対処
- 報酬変更なし（コード変更のみ）、1変更1検証原則に適合

**代替案C: symmetry_range報酬の復活**

- V34で無効化されたsymmetry_range報酬を再有効化し、L/R非対称性を直接ペナルティ化
- V19のhip_pitch非対称度は|6.2%|と比較的軽微だが、接地比率の非対称（43.5% vs 53.8%）は有意
- サーベイ7.4.3の知見に基づくが、報酬項目数が19になり推奨範囲を超過するリスク

**推奨理由**: 案Aは、V19で確認された「速度向上=歩幅増大」の一本足打法を、「歩幅×周波数」の二軸制御に転換する試み。歩行周波数はV1以降0.88 Hzに固定されており、周波数軸の探索は未実施。歩幅増大に伴う副作用（hip_pitch相関悪化、横方向揺れ増大）を周波数側で吸収できるかを検証する。

## まとめ

V19は目標速度を[0.15, 0.25]→[0.25, 0.35]に引き上げ、X速度を0.162→0.255 m/s (+57.7%) と大幅に向上させることに成功した。歩行周波数0.88 Hzは維持されており、速度向上は純粋に歩幅増大によるもの。swing_duration (+15.4%)、swing_contact_penalty (-18.6%)、action_rate (-8.1%)、hip_pitch非対称度（|16.3%|→|6.2%|）も改善した。Yawドリフト最終値は|21.07|°→|16.08|°と絶対値が改善（符号反転はRL確率性）。内股着地（stance hip_roll ±9.6-10.3°）はPDクランプにより維持された。

一方、hip_pitch相関の悪化（-0.804→-0.619、-23.0%）が最大の課題。接地比率の非対称化（L 43.5% vs R 53.8%）とswing duration非対称化が原因。横方向揺れも悪化（lateral std +29.7%、R² -49.6%）し、横方向揺れの主因がRoll角からL/R非対称歩行パターンによる運動学的揺れに移行した。Roll std +15.0%、foot_lateral_velocity +43.2%も速度向上のコストとして観察された。

目標速度引き上げはX速度向上に直接有効であるが、歩行品質（hip_pitch相関、横方向揺れ）とのトレードオフが確認された。速度向上が歩幅増大のみに依存している点が副作用の根本原因であり、次バージョンでは歩行周波数の引き上げ（0.9→1.2 Hz）により、歩幅と周波数の二軸で速度を制御し、歩幅増大に伴う副作用の軽減を目指すことを推奨する。

## 備考

- V15との比較により、目標速度の影響を純粋に評価できる
- 報酬項目数: 18（active non-zero）、V15から変更なし
