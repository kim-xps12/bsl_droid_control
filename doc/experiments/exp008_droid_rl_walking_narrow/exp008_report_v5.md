# V5: stance_hip_roll_targetスケール強化（-0.5→-2.0）

## 概要

V4で確認された内股offset悪化（L: -24.41°, R: +24.93°）を解消するため、stance_hip_roll_targetのスケールを-0.5から-2.0に強化する。V4でのankle_roll→stance_hip_roll_target置換時に実効ペナルティが半減（-0.5→-0.25）したことが内股悪化の主因であり、スケール4倍化で実効ペナルティを-1.0に引き上げ、V1水準（実効-1.3）に近いhip_roll制約を接地相限定で実現することを目指す。

## 前バージョンからの改善事項

V4レポートの「次バージョンへの提案」推奨案に基づく変更:

> 推奨案: stance_hip_roll_targetスケール強化（-0.5→-1.0）
> - V4で内股offsetが悪化した主因は実効ペナルティ不足（接地相限定により約50%に低下）

ただし精査の結果、-1.0（実効-0.5）ではV3のankle_roll=-0.5（実効-0.5）と同水準であり、V3で~23°程度にとどまっていたため、-2.0（実効-1.0）に引き上げる判断を行った。

| パラメータ | V4値 | V5値 | 変更理由 |
|-----------|------|------|---------|
| stance_hip_roll_target | -0.5 | **-2.0** | 接地相hip_rollペナルティ強化（内股offset低減） |

### 実効ペナルティの系統的分析

各バージョンのhip_rollに対する実効ペナルティと内股offsetの関係:

| Version | hip_roll制御メカニズム | 実効ペナルティ | 内股offset |
|---------|----------------------|---------------|-----------|
| V1 | hip_pos=-0.8(常時,compound) + ankle_roll=-0.5(常時) | -1.3 (常時) | ~15° |
| V3 | hip_yaw_pos=-0.8(yawのみ) + ankle_roll=-0.5(常時) | -0.5 (常時) | ~23° |
| V4 | hip_yaw_pos=-0.8(yawのみ) + stance_hip_roll_target=-0.5(接地相~50%) | -0.25 (実効) | ~25° |
| **V5** | **hip_yaw_pos=-0.8 + stance_hip_roll_target=-2.0(接地相~50%)** | **-1.0 (実効)** | **~17°を期待** |

### 内股問題の根本原因分析

1. **トルク不足ではない**: PD制御のkp/kdは全関節一律であり、RLポリシーが意図的にhip_roll ±25°をターゲットしている
2. **物理的制約**: narrow URDF（hip_offset_y ≈ 0.0675m）では片足支持時にhip_roll ~15°が物理的に必要（0°は構造的に不可能）
3. **報酬インセンティブ不足**: V4の実効ペナルティ-0.25では、ポリシーが25°付近を最適解として学習。-1.0に引き上げることで物理下限（~15°）に近い解を誘導する

### 期待される効果

- 内股offset: 25°→17°以下への低減を期待（V1の15°に近い水準）
- Yaw/Roll: V4水準を維持することを期待（接地相限定によるスイング相自由度維持）

### リスク

## 考察と改善案

### 成功点

1. **内股offset劇的改善**: L: -13.4°→-3.5°（-74%）, R: +17.0°→+4.4°（-74%）。exp008全体best。ほぼ解消
2. **Yawドリフトexp008全体best大幅更新**: +7.51°→+0.69°（-90.8%）。定常ドリフト速度も-0.596°/sと低い
3. **X速度維持**: 0.167→0.170（+1.8%）。ペナルティ増大にもかかわらず速度を維持
4. **stance_hip_roll_target raw penalty -83%**: hip_roll角度自体が大幅に縮小し、ペナルティの設計意図が正確に達成

### 課題

1. **Roll std大幅悪化**: 3.92°→6.95°（+77.3%）。接地相hip_roll硬直化によりbody oscillation増大
2. **Pitch std大幅悪化**: 0.98°→2.27°（+131.6%）。V4で全実験best(0.98°)から大幅後退
3. **横方向揺れ大幅悪化**: base_pos_y std 14.9→93.0mm（+524%）。内股offsetは解消したが横方向安定性が崩壊
4. **スイング持続時間短縮**: 0.49-0.54s→0.24-0.29s。「短頻度」戦略（V36パターン）に遷移。ペナルティ回避戦略
5. **タップダンス増加**: 両足接地率0.8%→3.5%、swing_contact_penalty +393.8%
6. **hip_pitch corr低下**: -0.787→-0.673。交互歩行パターンの明確さが低下
7. **報酬バランス悪化**: ペナルティ/正報酬比率0.100→0.186（exp008最悪）

### 因果メカニズム分析

#### stance_hip_roll_target 4倍強化の効果ツリー

```
stance_hip_roll_target -0.5→-2.0（4倍強化）

 │    └─→ 接地相hip_roll硬直化（バランス利用不可）
 │         ├─→ Roll std +77%（body oscillation増大）
 │         ├─→ base_pos_y std +524%（横方向揺れ）
 │         ├─→ Pitch std +132%（姿勢安定性低下）
 │         └─→ ang_vel_xy +95.2%（角速度増大）
 ├─→ [間接効果B] ペナルティ回避戦略への遷移
 │    ├─→ スイング持続時間短縮（0.49-0.54s→0.24-0.29s）

 │    ├─→ 両足接地率増加（0.8%→3.5%）= タップダンス増加
 │    │    └─→ swing_contact_penalty +393.8%
 │    └─→ hip_pitch corr低下（-0.787→-0.673）= 交互歩行パターン不明確化
 └─→ [副産物C] Yaw大幅改善（+7.51°→+0.69°）
      ├─→ 短スイングによる角運動量削減（V36パターン）
      └─→ hip_roll硬直化によるhip_yaw間接影響減少
```

#### 「内股解消」と「歩行品質」のトレードオフ

V5はstance_hip_roll_target=-2.0（実効-1.0）で内股をほぼ解消した一方、接地相でのhip_rollバランス利用を過度に制約した。サーベイSection 7.2.3の「race to bottom」パターンの変形版：

- V33: symmetry_range → 振幅縮小（関節が動かなくなる）
- V5: stance_hip_roll_target=-2.0 → 接地相hip_roll硬直化（バランスに使えなくなる）

いずれも「特定の関節自由度を過度に制約 → 他の指標で補償動作 → 全体の歩行品質低下」という共通メカニズム。

#### Yaw改善は「副産物」

V5のYaw改善（+7.51°→+0.69°）は内股改善の直接効果ではなく、主に2つの間接効果による:
1. スイング持続時間短縮による角運動量削減（V36で同パターン確認）
2. Yaw推移パターンの「上昇→下降」相殺（0-4s: +17°方向、4-10s: 下降して+0.7°で着地）

V6でスケールを緩和した際、Yawが悪化する可能性があることに注意。

### サーベイ知見との照合

1. **Section 7.2（報酬スケール影響伝搬）**: 一致。4倍スケール変更が直接効果（内股改善）と間接効果（gait strategy変更）の両方を伴う
2. **Section 7.2.3（対称性報酬race to bottom）**: 一致。V33パターンの再現。hip_roll offset改善とRoll std悪化がトレードオフ
3. **Section 7.4.2（Yawドリフトメカニズム）**: 一致。短スイングによる角運動量削減でYaw改善（V36教訓と整合）
4. **Section 8.1（ペナルティ過大のgait degradation）**: 部分一致。完全静止ではないがgait degradation（短頻度戦略、タップダンス増加）は確認
5. **Section 7.2.1（Periodic Reward Composition）**: 方向性は正しいがスケール-2.0が過大。フェーズ条件付きペナルティの適正スケール探索が必要

### 次バージョンへの提案

#### 方針転換: 関節レベルからボディレベルへ

**V5の目視確認の結果、hip_roll数値の改善（±4°）にもかかわらず、視覚的な内股は全く解消されていないことが判明した。** これはhip_roll関節角度≠視覚的内股であり、**胴体の傾き（Roll oscillation）が主因**であることを示唆する。V5のRoll std 6.95°（V4比+77%）が大きな体の揺れを生み、脚全体が交互に傾くことで内股に見える。

このことから、**hip_roll関節を直接制約するアプローチ（stance_hip_roll_target強化）は限界**であり、V6ではボディレベル（胴体の鉛直性保持）にアプローチを変更する。

#### 推奨案: V4ベース + orientation強化（-0.5→-2.0）

**V5のstance_hip_roll_target=-2.0は失敗実験のため破棄し、V4をベースとする。** V4→V6の変更差分は1項目のみ。

| パラメータ | V4値 | V6値 | 変更理由 |
|-----------|------|------|---------|
| orientation | -0.5 | **-2.0** | 胴体の鉛直性保持を4倍強化 |
| stance_hip_roll_target | -0.5 | -0.5 | V4と同値（V5の-2.0は破棄） |

**根拠**:
1. `_reward_orientation`は`projected_gravity[:,:2]²`を罰するため、胴体のRoll/Pitch両方の傾きを直接抑制する
2. 胴体が鉛直に近づけば、脚の見かけの内股が軽減される（体の傾き分の視覚的補正）
3. V4のRoll std 3.92°、Pitch std 0.98°をベースに、orientation強化でさらなる安定化を期待
4. V4→V6の1変更で、orientationスケール変更の純粋な効果を測定可能
5. 既存報酬関数の活用であり、新規実装不要

**期待される効果**:
- Roll std/Pitch stdの更なる低減（胴体鉛直性向上）
- 視覚的な内股の軽減（体傾斜起因分）
- V4の良好な歩行品質（Yaw +7.51°, 長振りスイング）を概ね維持

**リスク**:
- orientation過大でgait degradation（V5パターン）の可能性。ただし-2.0はV5の-2.0とは異なる報酬項目で、orientation -2.0はUnitree RL Gymデフォルト値(-5.0)より穏やか
- X速度低下の可能性（胴体制約による自由度縮小）

#### 代替案: swing_foot_lateral_velocity強化（-0.5→-1.5）

足先軌道の横方向成分を3倍に制約。遊脚が内側を通る軌道を直接抑制する。orientation強化で効果不足の場合の次ステップ。

## まとめ

exp008 V5は、V4のstance_hip_roll_target（接地相限定hip_rollペナルティ）のスケールを-0.5から-2.0に4倍強化した。

**成果**: 内股offset劇的改善（L: -13.4°→-3.5°, R: +17.0°→+4.4°, exp008全体best）、Yawドリフトexp008全体best大幅更新（+7.51°→+0.69°, -90.8%）、X速度維持（+1.8%）。hip_roll raw penalty -83%で設計意図の達成を確認。
