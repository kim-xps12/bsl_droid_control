# V20: 歩行周波数引き上げ（0.9→1.2 Hz）

## 概要

V19をベースに、歩行周波数（gait_frequency）のみを0.9→1.2 Hzに引き上げる実験。V19ではX速度+57.7%を達成したが、速度向上が歩幅増大のみに依存し、hip_pitch相関-23%、横方向揺れ+29.7%の副作用が発生した。歩行周波数を引き上げることで、歩幅×周波数の二軸制御への転換を狙い、歩幅増大に伴う副作用の軽減を目指す。

## 前バージョンからの改善事項

V19レポートの「次バージョンへの提案」推奨案Aに基づく。V19の課題（hip_pitch相関悪化、横方向揺れ増大）に対し、歩行周波数の引き上げで対処する。

| パラメータ | V19値 | V20値 | 変更理由 |
|-----------|-------|-------|---------|
| gait_frequency | 0.9 Hz | 1.2 Hz | 歩幅のみの速度戦略から歩幅×周波数の二軸制御に転換。1歩あたりの振り出し角を抑制し、hip_pitch相関・横方向揺れの改善を狙う |

## 設計詳細

### 歩行周波数変更の理論的根拠

V19の分析で判明した因果関係:

```
目標速度↑ → 歩幅↑（周波数0.88Hz維持）
  → hip_pitch range L +52.7% (0.567→0.866 rad)
  → 運動エネルギー↑ → Roll振動↑ (+15%)
  → foot_lateral_velocity +43.2%
  → L/R非対称化 → hip_pitch corr -23%
```

gait_frequency引き上げによる期待される効果:

```
gait_frequency 1.2Hz → 周波数↑ + 歩幅↓（速度維持）
  → 1歩あたり振り出し角↓ → 運動エネルギー↓
  → Roll振動↓, foot_lateral_velocity↓
  → L/R位相安定化 → hip_pitch corr改善
```

### gait_frequencyの影響範囲

gait_frequencyはcontact報酬・swing_duration報酬の位相基準に作用する:
- `contact`: 接地タイミングの位相判定基準
- `swing_duration`: 遊脚時間の位相判定基準
- 報酬項目の追加/削除は伴わないため、1変更1検証原則に適合

### リスク評価

- 周波数が高すぎると小刻みな歩行になる可能性（V15で観察されたPitch std悪化パターン）
- 0.88 Hzが本ロボットの自然共振周波数である可能性があり、強制的な変更がgait品質を低下させるリスク
- +33%の変更幅（0.9→1.2）は比較的大きいが、周波数は連続変数であり段階的調整が可能

## 考察と改善案

### 成功点

1. **歩行周波数の劇的変化**: 0.88→1.25 Hz (+42%)。gait_frequency=1.2の設定に対して実測1.25 Hzは極めて良好な追従。V1以降19バージョンにわたり0.88 Hzに固定されていた歩行周波数が初めて変化した
2. **X速度の更なる向上**: 0.255→0.312 m/s (+22.4%)。周波数増加による速度向上が歩幅減少を上回った（velocity = stride × frequency: 歩幅 -18% × 周波数 +42% ≈ +16-22%）
3. **hip_pitch相関の劇的改善**: -0.619→-0.877 (+41.7%)。V4水準（-0.787）を大幅に上回り、V3の-0.805も超えるexp008史上最高値
4. **Yawドリフトほぼゼロ**: -16.08°→+0.05°。ドリフト速度も-3.795→-0.231 °/s (-93.9%)。exp008全バージョン中最良
5. **横方向揺れの大幅改善**: lateral std -52.9% (58.0→27.3mm)、p2p -61.4% (235→91mm)。exp008全バージョン中最良
6. **R²回復**: 0.379→0.903。横方向揺れの90%がRoll角で説明可能な状態に回復。残差std 45.8→8.5mm (-81.4%)は非Roll成分がほぼ消失したことを示す
7. **Roll std/Pitch std同時改善**: Roll -29.8% (5.95→4.18°)、Pitch -54.3% (2.19→1.00°)。Pitch stdはV7（0.70°）に次ぐ好成績
8. **接地パターン対称化**: L43.5%/R53.8%→L47.2%/R49.5%。L/R差が10.3pp→2.3ppに縮小
9. **penalty/positive ratio 0.079**: V13（0.070）に次ぐ全バージョン2位。ペナルティ合計が-56.5%と大幅減少
10. **内股着地維持**: stance hip_roll L -9.3°, R +9.3°。PDクランプの効果が周波数変更に影響されないことを確認。L/R対称性も改善（R 10.3°→9.3°）
11. **ほぼ全ペナルティ指標改善**: 13ペナルティ中12が改善。action_rate -86.2%、foot_lateral_velocity -81.6%、stance_foot_lateral_position -96.9%、swing_contact_penalty -90.7%が特に顕著

### 課題

1. **feet_swing_height悪化（+17.5%）**
   - 唯一悪化したペナルティ項目（-0.0452→-0.0531）
   - 1.25 Hzでのスイング時間は0.343-0.366s（V19: 0.395-0.482s）と短縮されており、swing_height_target=0.05mを達成する時間的余裕が減少
   - ペナルティ全体に占める割合は22.2%（V20最大のペナルティ要因）

2. **step_length報酬低下（-34.6%）**
   - 0.0706→0.0462。歩幅縮小の直接的反映
   - 正報酬への影響は小さい（正報酬全体の1.5%）

3. **base_height微低下（-3.6%、0.280→0.270m）**
   - 高周波歩行により平均重心高がやや低下
   - base_height penalty自体は-9.5%改善しており、目標値（0.20m）との乖離は十分小さい

### 根本原因分析

**歩行周波数引き上げによるカスケード改善メカニズム**

```
gait_frequency 0.9→1.2 Hz → 実測歩行周波数 0.88→1.25 Hz
  → スイング時間短縮 (0.44-0.48s → 0.34-0.37s)
  → 1歩あたりの振り出し角↓ (hip_pitch range L 0.866→0.701 rad, -19.1%)
  → 運動エネルギー↓（per step）
    → Roll std -29.8%, Pitch std -54.3%
    → foot_lateral_velocity -81.6%, action_rate -86.2%
  → 短いスイング時間 → MLP非対称性の蓄積時間↓
    → L/R接地比率対称化 (差10.3pp→2.3pp)
    → hip_pitch corr -0.619→-0.877
    → Yawドリフト角運動量不均衡↓ → -16.08°→+0.05°
  → 非Roll横方向成分↓ → R² 0.379→0.903
  → 速度 = 歩幅 × 周波数: 0.81 × 1.42 ≈ +16-22%
```

**V19→V20の対比が示す構造的知見**: 同じ報酬構成・同じPDクランプで、gait_frequencyの1パラメータ変更だけで主要指標がほぼ全面的に改善した。これは「報酬整形だけではアクセスできない改善領域がgait_frequency軸に存在していた」ことを意味する。V1以降0.9 Hz固定だったgait_frequencyは、0.88 Hzの自然共振を前提に報酬設計が最適化されていた。1.2 Hz設定はBSL-Droid Simplified V2の動的特性により適合していた可能性がある。

### サーベイ知見との照合

1. **MLPの左右非対称性（サーベイ7.2.3）**: V19でhip_pitch corr -0.619に悪化した原因として「MLPは対称出力を保証しない」と分析していた。V20では同じMLPアーキテクチャで-0.877を達成。高周波歩行はスイング時間短縮によりMLP非対称性の蓄積を抑制し、equivariant networkなしでも対称性を大幅改善できることを実証した
2. **角運動量不均衡によるYawドリフト（サーベイ7.4.2）**: V19の-16.08°ドリフトは「遊脚の角運動量不均衡」と分析されていた。V20では短いスイング時間×対称接地比率の積によりYaw角運動量不均衡がほぼゼロ化（+0.05°）
3. **Gaussian報酬構造（サーベイ2.1.1）**: tracking_lin_vel報酬が-1.4%の微減にとどまった（1.1972→1.1809）。目標速度[0.25, 0.35]に対して実速度0.312 m/sは目標中央付近であり、Gaussian報酬の最適動作点に位置する
4. **penalty/positive ratio**: 0.079はV5のoverkill zone（0.186）を大幅に下回る健全水準。ペナルティ削減は報酬削除ではなく動作改善による自然低下であり、Goodhart's Law回避の理想形
5. **Physical structure > reward shaping（サーベイ8.1）**: exp007-exp008を通じた「物理構造 > 報酬整形」原則は、「動的パラメータ（タイミング含む）> 報酬整形」に拡張される。gait_frequency変更は物理構造変更ではないが、ロボットの動的特性との整合を改善する点で同等の効果を発揮した

#### 機能別詳細分析

##### 1. 主目的（2項目）: 速度追従

**tracking_lin_vel** (+1.1809, scale +1.5) — **KEEP**
- 設計意図: XY線速度の指令追従（Gaussian報酬、sigma=0.25）
- V19→V20: -1.4%微減。目標[0.25, 0.35]に対し実速度0.312m/sは中央付近でGaussian最適動作点
- 冗長性: なし。唯一の直接速度報酬
- リスク: 削除不可。全報酬の根幹

**tracking_ang_vel** (+0.4333, scale +0.5) — **KEEP**
- 設計意図: Yaw角速度の指令追従。ang_vel_range=[0,0]なので、実質的にYaw安定性報酬として機能
- V19→V20: **+55.7%改善**。Yawドリフト≈0°が直接反映（ドリフトがないため角速度がゼロに近い→Gaussian報酬最大化）
- 冗長性: hip_yaw_posと部分重複（Yaw制御の角速度側 vs 関節角側）。ただしhip_yaw_posは関節空間、tracking_ang_velはタスク空間であり補完関係
- リスク: 削除するとYaw安定性の報酬インセンティブが消失

##### 2. 歩行品質報酬（6項目）: 歩容構造の形成

**contact** (+0.7005, scale +0.4) — **KEEP**
- 設計意図: 位相-接地状態の整合報酬。接地相で接地、遊脚相で離地を報酬
- V19→V20: +1.0%安定
- 冗長性: swing_contact_penaltyと部分重複（遊脚相の接地に対する正/負のインセンティブ）。ただし本項目は正報酬として接地相もカバーするため、機能範囲はより広い

#### 項目整理の必要性に対する結論

**明確な削除候補は1項目のみ: stance_foot_lateral_position**

1. **stance_foot_lateral_position (scale -0.5 → 0)**: V14以降のPDクランプ (hip_roll_inward_limit=-0.05 rad) により機能的に完全代替されている。V20での寄与は-0.0014（ペナルティ全体の0.6%）であり、V19→V20で-96.9%減少してほぼゼロ化。削除により報酬項目数が19→18となり、推奨範囲15-17の上限に近づく。リスクは極めて低い（7バージョンの検証実績）

2. **その他の低寄与項目は維持すべき**: foot_lateral_velocity (V17教訓)、velocity_deficit (安全弁)、contact_no_vel・swing_contact_penalty・action_rate (歴史的検証不足) は、magnitude ≠ importance の原則に基づき削除すべきでない。低い寄与度は「ポリシーが該当する悪行動をほぼ行わなくなった」ことの反映であり、報酬自体が不要になったわけではない

3. **「magnitude ≠ importance」原則**: 寄与度ランキング下位の項目の中に、安全弁として必須のもの（velocity_deficit, 0.0006）、削除で破局的失敗を招くもの（foot_lateral_velocity, 0.0081）、そして真に冗長なもの（stance_foot_lateral_position, 0.0014）が混在する。寄与度の大きさだけでは削除判断は不可能であり、冗長性と歴史的教訓に基づく多次元評価が必須

4. **V21への推奨**: stance_foot_lateral_position (scale -0.5→0) の削除を1変更1検証として実施。コード変更不要（スケール値変更のみ）。期待効果: penalty/positive ratio 0.079→~0.075、報酬項目数18に最適化

## まとめ

V20はgait_frequencyを0.9→1.2 Hzに引き上げ、exp008全20バージョン中最も広範で劇的な改善を達成した。主要指標の大半が改善し、悪化したのはfeet_swing_height (+17.5%) のみであった。

最も顕著な成果:
- **X速度 +22.4%** (0.255→0.312 m/s): 歩幅減少を周波数増加が上回り、速度と品質の同時向上を実現
- **hip_pitch相関 +41.7%** (-0.619→-0.877): exp008史上最高値。交互歩行の品質が劇的に改善
- **Yawドリフト ≈0°** (-16.08°→+0.05°): 事実上のゼロドリフトを達成
- **横方向揺れ -52.9%** (58.0→27.3 mm): 非Roll成分がほぼ消失（残差std 45.8→8.5 mm）
- **penalty/positive ratio 0.079**: V13に次ぐ歴代2位の健全水準

これらの改善は単一パラメータ（gait_frequency）の変更のみで達成された。V1以降0.9 Hz固定だったgait_frequencyが最適でなかったことが判明し、1.2 Hz設定がBSL-Droid Simplified V2の動的特性に、より適合していることが示された。報酬整形の限界を超えた改善領域が環境パラメータ軸に存在することを実証し、「動的パラメータ > 報酬整形」の原則がexp007-exp008の知見を拡張する。
