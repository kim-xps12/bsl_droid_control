# V15: 横方向速度ペナルティ新設 (base_vel_y)

## 概要

V14でPDターゲットクランプ強化により内股がFK参照値(-12°)に到達したが、横方向並進揺れが新たな主課題として顕在化した（lateral std 48.0mm, peak-to-peak 181mm, Y速度std 0.225 m/s）。V15ではY方向並進速度を直接ペナルティ化する`base_vel_y`報酬を新設し、横方向揺れの抑制を狙う。

## 前バージョンからの改善事項

V14レポートの「次バージョンへの提案」推奨案Bに基づき、横方向並進速度ペナルティを新設する。

| パラメータ | V14値 | V15値 | 変更理由 |
|-----------|-------|-------|---------|
| base_vel_y (新設) | なし | -1.0 | Y方向並進速度 std 0.225m/s (V4比+162%) の直接抑制 |

他のパラメータは一切変更なし（1変更1検証原則）。報酬項目数は17→18。

## 設計詳細

### base_vel_yの設計根拠

V14の横方向揺れの定量的根拠:
- lateral detrended std: 48.0mm（V4の14.9mmの3.2倍）
- Y速度 std: 0.225 m/s（V4の0.086 m/sの2.6倍）
- lateral peak-to-peak: 181mm（目視で非常に顕著）

既存のang_vel_xyペナルティは角速度（回転）を制約するが、横方向並進速度は直接制約しない。体がゆっくり大きく揺れる場合、角速度は低くても並進揺れは大きいため、base_vel_yで並進速度を直接ペナルティ化する。

### 報酬関数の実装

```python
def _reward_base_vel_y(self) -> torch.Tensor:
    """Y方向並進速度ペナルティ（横方向揺れ抑制）"""
    return torch.square(self.base_lin_vel[:, 1])
```

既存の`_reward_lin_vel_z`（Z方向速度ペナルティ）と同じ構造。二乗ペナルティにより大きな横方向速度ほど強くペナルティ化される。

### スケール値-1.0の根拠

- lin_vel_z（Z方向）が-2.0であるのに対し、base_vel_yは-1.0で控えめに開始
- V14のpenalty/positive ratio 0.150に対し、base_vel_y追加で~0.16-0.17を予測
- V5のoverkill zone (0.186)を超えない設計
- 横方向並進速度を減らすために、ポリシーが現在放棄している外向きhip_roll修正を再発見する可能性を期待

### 報酬項目数

V15: 18項目（V14の17項目 + base_vel_y 1項目）。推奨範囲15-17の上限を1項目超過するが、21+の危険域にはまだ余裕がある。

## 考察と改善案

### 成功点

- **横方向位置振幅の改善**: lateral detrended std -17.7% (48.0→39.5mm)、lateral p2p -32.5% (181.8→122.7mm)。base_vel_yの主目的であった横方向揺れの位置振幅が明確に改善された
- **Roll振動の改善**: Roll std -13.1% (5.73→4.98°)、Roll p2p -21.2% (18.03→14.20°)。Roll角速度RMS -11.9%。横方向並進速度を制約することでRoll振動も間接的に抑制された
- **非Roll成分の大幅抑制**: 残差std -48.5% (26.0→13.4mm)。Roll-横方向R²が0.707→0.885に上昇し、横方向揺れのうちRollで説明できない成分（非予測的なドリフト・不規則振動）が半減した
- **歩行周波数・内股維持**: 0.88Hz、stance hip_roll ±10°台を完全維持。PDクランプの効果はbase_vel_y追加に影響されない
- **hip_pitch相関改善**: -0.800→-0.817 (+2.1%)。左右非対称度も-15.1%→-11.9%に改善

### 課題

- **Yaw drift微増**: -8.12°→+15.15°（符号反転はRL確率性）。符号をまたいでいるため絶対値比率(+86.6%)は過大評価。実質的な絶対値差は+7.0°であり、drift速度も|0.885|→|1.656|°/sと絶対値差+0.77°/sの増加にとどまる
- **Pitch std大幅悪化**: 1.07°→1.98° (+85.0%)。V14のPitch std=1.07°はexp008中best水準であったが、V15で大きく後退
- **Y速度stdは改善されず**: 0.224→0.233 m/s (+3.7%)。base_vel_yで直接ペナルティ化した指標が改善されていない。位置振幅は改善されたが速度振幅は維持または微増
- **swing_duration大幅低下**: 0.3773→0.2120 (-43.8%)。L swing duration 0.538→0.472s (-12.3%)、R 0.488→0.444s (-9.0%)。歩幅が短縮し、歩行が「小刻み」になった
- **swing_contact_penalty倍増**: -0.0224→-0.0463 (+106.7%)。遊脚相の接地タイミングが不規則化
- **penalty/positive ratio増加**: 0.150→0.183。V5のoverkill zone (0.186) に接近。base_vel_yの-0.0382に加え、action_rate (+21.8%)、lin_vel_z (+42.8%)の増加が寄与

### 因果分析

1. **base_vel_yの作用メカニズム**: Y方向並進速度の二乗ペナルティは、瞬時速度のピーク値を特に強く抑制する（二乗の性質）。これにより横方向の位置振幅は改善されたが、Y速度stdが改善されない理由は、ポリシーが**高周波・小振幅の横方向振動**に移行した可能性がある。位置の積分特性により、高周波振動は低周波振動と同じ速度stdでも位置振幅は小さくなる

2. **Yaw driftの符号反転と微増**: V14 -8.12°→V15 +15.15°は符号をまたぐため絶対値比率(+86.6%)は過大評価。実質的な変化は絶対値差+7.0°。符号反転自体はRL確率性（MEMORY: 同一報酬構成でも左右非対称パターンが反転する現象）による。ただし絶対値の微増(8°→15°)はbase_vel_yが修正的なY方向速度（Yawドリフト補正のための横方向ステップ）も同時にペナルティ化する副作用を示唆する

3. **Pitch悪化のメカニズム**: swing_duration低下とswing_contact_penalty増加は、ポリシーがY速度を抑制するために足の接地時間を短縮し、より頻繁に足を交替させる戦略を採用したことを示唆する。短い遊脚時間は制御マージンの低下を意味し、Pitch方向の安定性が犠牲になった

4. **base_vel_yとstance_foot_lateral_positionの重複**: stance_foot_lateral_positionのペナルティが-0.0624→-0.0472 (-24.4%) に低下した。base_vel_yが横方向運動を制約することでstance_foot_lateral_positionの違反が減少しており、両報酬の制約対象が部分的に重複していることが確認された

### 横方向揺れの根本原因再分析

V15の横方向揺れ（lateral std 39.5mm）はV4（14.9mm）の2.6倍であり、base_vel_yによる-17.7%改善では不十分である。根本原因の再分析を行う。

**V4が最良の横方向安定性を示した理由**（サーベイ・過去レポート照合）:
1. **位相依存hip_roll制御**: V4はstance_hip_roll_target（接地相限定ペナルティ）により、接地相ではhip_rollを制約しつつ遊脚相では自由を許容した。hip_roll L range +33.8%（0.517→0.692 rad）の遊脚相自由度が、次の着地位置の能動的調整（lateral foot placement）を可能にした
2. **二段階横方向制御戦略**: 遊脚相でhip_rollを使って次の足位置を調整 → 接地相でhip_roll制約により安定化。この動的な位相依存制御がlateral std 14.9mmを達成した

**V14-V15で横方向揺れが悪化した原因**:
- V14のPDターゲットクランプ（hip_roll_inward_limit=-0.05 rad）は全位相でhip_rollを制約する
- hip_roll動的範囲が0.38→0.21 rad (-44%)に低下し、V4の位相依存バランス機構が消失
- V15のbase_vel_yは症状（横方向速度）を抑制するが、原因（hip_rollバランス機構の消失）には対処していない

**結論**: 横方向揺れの根本原因はPDクランプによるhip_roll動的範囲の制限であり、報酬スケール調整（base_vel_y, ang_vel_xy等）は対症療法にとどまる。根本的解決には**位相条件付きPDクランプ**が必要。

### 次バージョンへの提案

**推奨案A（推奨）: 位相条件付きPDクランプ**
- 現在のPDクランプは全位相で適用されるが、内股が問題になるのは**接地相**（足が地面に着く瞬間の向き）のみ
- 遊脚相ではhip_rollを自由にし、V4の位相依存バランス機構を回復
- 実装: `step()`内のPDターゲットクランプを接地状態で条件分岐
  - 接地相: hip_roll_inward_limit=-0.05 rad（V14維持、内股防止）
  - 遊脚相: クランプなし（lateral foot placement自由度回復）
- 期待効果: lateral std 39.5mm → 20-25mm（V4の14.9mmとV14の48.0mmの中間域）
- リスク: 遊脚相の自由hip_rollにより内股着地が部分的に再発する可能性。ただしPDクランプは接地遷移時にも作用するため、着地直前の制約が残る
- 報酬変更なし（コード変更のみ）。1変更1検証原則に適合
- V14提案C（長期案）の前倒し実施

**案B: stance_foot_lateral_position削除（penalty budget整理）**
- base_vel_yとstance_foot_lateral_positionの制約重複がV15で確認された（ペナルティ-24.4%低下）
- 削除により報酬項目数18→17（推奨範囲内に復帰）、penalty/positive ratio 0.183→0.16前後に改善
- 横方向揺れへの直接効果は小さいが、penalty budget解放によりポリシーの自由度が改善
- A案と同時実施可能（A案はコード変更、B案は報酬パラメータ変更で独立）

**案C: ang_vel_xy強化 (-0.1→-0.2)**
- V14提案Aの実施。V38で-0.05→-0.1の2倍化がRoll std -30%を達成した実績
- PDクランプ下でhip_rollの自然減衰が失われた状態で、ang_vel_xyが外部減衰として機能
- サーベイ2.2.2節: ang_vel_xyはUnitree Go2/H1の標準的Roll抑制手法
- A案（位相条件付きクランプ）が困難な場合のフォールバック
- base_vel_yを維持するか削除するかは別途検討（2変更になるためA/B案との組合せは非推奨）

## まとめ

V15はbase_vel_y=-1.0の新設により、横方向位置振幅を-17.7%（lateral std）、-32.5%（lateral p2p）改善し、Roll振動も-13.1%（Roll std）抑制することに成功した。特に非Roll成分の残差stdが-48.5%と大幅に改善され、横方向揺れの予測可能性が向上した。

しかし、Yaw drift微増（符号反転+絶対値差7.0°）、Pitch std (+85.0%)の悪化、swing_duration -43.8%の歩幅短縮という副作用を伴った。Yaw driftの符号反転はRL確率性であり、絶対値比率(+86.6%)は符号をまたぐため過大評価である。Y速度std自体は改善されず（+3.7%）、ポリシーが高周波振動に移行した可能性がある。penalty/positive ratio 0.183はV5のoverkill zone (0.186)に接近している。

横方向揺れの根本原因はPDクランプによるhip_roll動的範囲の制限（V14で-44%）であり、base_vel_yやang_vel_xy等の報酬スケール調整は対症療法にとどまる。V4（lateral std 14.9mm, exp008 best）が示した位相依存hip_roll制御（遊脚相自由・接地相制約）を回復するため、次バージョンでは位相条件付きPDクランプ（A案）の実施を推奨する。

- base_vel_yは`droid_env_unitree.py`に`_reward_base_vel_y()`として実装
- `torch.square(self.base_lin_vel[:, 1])`（Y方向並進速度の二乗）
- 既存の`_reward_lin_vel_z`（Z方向）と同じパターン
- PDクランプ設定はV14から維持: hip_roll_inward_limit=-0.05 rad
- base_vel_yはUnitree Go2/H1では直接的な前例なし（survey参照）。類似のlin_vel_y制約はLiu et al. (2024)のmulti-skill agentで使用
