# V2: hip_pos緩和によるYawドリフト改善（失敗）

## 概要

exp008 V1をベースに、hip_posスケールを-0.8から-0.5に緩和する。narrow URDFの物理構造がhip_yaw twistを自然に抑制しているため、hip_pos制約を緩和してhip_yaw自由度を回復し、Yawドリフトの改善を狙った。

**結果**: Yawドリフトが壊滅的に悪化（+16.46°→+96.49°、ドリフト速度6.6倍）。narrow URDFのizz低下がYawドリフト感度を増幅し、hip_pos緩和が意図と逆の結果をもたらした。

## 前バージョンからの改善事項

V1レポートの「次バージョンへの提案」推奨案に基づく変更:

> 推奨案: hip_pos緩和（-0.8 → -0.5）
> - V1のhip_yaw stance twistは大幅改善済み（L: -2.03°、R: +3.28°）
> - narrow URDFの物理構造がhip_yaw twistを抑制 → hip_pos制約の必要性が低下
> - ankle_roll=-0.5がhip_roll制御を担っている → hip_posの役割がhip_yaw制約に偏っている
> - 緩和してhip_yaw自由度を回復しYaw修正能力を向上

| パラメータ | V1値 | V2値 | 変更理由 |
|-----------|------|------|---------|
| hip_pos | -0.8 | -0.5 | hip_yaw自由度を回復しYawドリフト改善を狙う |

## 考察と改善案

### 成功点

1. **Roll std改善**: 6.72°→5.37°（-20.1%）。hip_pos緩和によりhip_roll自由度が増し、Roll制御が改善
2. **Y速度改善**: +0.023→-0.004 m/s。横方向速度がほぼゼロに
3. **hip_pitch非対称度改善**: -11.9%→-4.0%。L/R対称性が向上
4. **両足接地率改善**: 0.8%→0.4%。さらに明確な片脚支持パターン

### 課題

1. **Yawドリフト壊滅的悪化**: +96.49°（V1: +16.46°、+486%）。ドリフト速度11.872°/s（V1: 1.801°/s、6.6倍）。10秒でほぼ直角回転
2. **Pitch std大幅悪化**: 5.02°（V1: 1.56°、+222%）。Yaw回転による二次効果（進行方向とロボット向きのずれで前進力がPitch方向に流入）
3. **X速度低下**: 0.156 m/s（V1: 0.190 m/s、-17.9%）。Yaw回転により見かけの前進速度が低下
4. **hip_pitch相関悪化**: -0.571（V1: -0.681、-16.2%）。交互歩行パターンの明確性が低下
5. **hip_roll stance offset増加**: L: -0.167→-0.199 rad、R: +0.157→+0.178 rad。hip_pos緩和でhip_roll制約も弱化

### 因果メカニズム分析（統合）

#### 3問題の統合因果ツリー

3つの問題は独立ではなく、hip_pos緩和という共通根源から分岐した因果ツリーを形成:

```
hip_pos -0.8 → -0.5（根源変更）
 ├─→ [経路A] hip_yaw制約弱化
 │    ├─→ L stance hip_yaw符号反転 (-2.12°→+2.43°)
 │    │    └─→ 両足hip_yawが同方向偏り → 自己修正能力消失 (r=-0.76→+0.33)
 │    │         └─→ [問題3] Yawドリフト加速 (+96.49°)
 │    │              └─→ Yaw 96°回転状態での座標系回転
 │    │                   ├─→ Pitch std +222%（二次効果）
 │    │                   └─→ L/R脚の力学的役割非対称化
 │    │                        ├─→ [問題1] 内股着地のL脚集中
 │    │                        └─→ [問題2] タッピングのL脚集中
 │    └─→ hip_yaw自由度増大 → stance leg twist
 │         └─→ [問題1] hip_roll stance offset +10.6%
 └─→ [経路B] hip_roll制約も同時弱化（compound penalty構造）
      └─→ hip_roll stance offset増加
           └─→ [問題1] 内股着地の直接的原因
```

#### L/R非対称性の原因

内股とタッピングがL脚に集中し、R脚はむしろ改善した理由: **正方向（反時計回り）のYaw回転が左右脚に異なる力学環境を生成**。L脚接地時はYaw回転がL脚を内側に引き込む方向、R脚接地時は外側に押し出す方向に作用。さらにYaw回転でL脚のswing軌道が変形し、着地方向が逆転（底屈→背屈）してバウンスが発生。

#### 因果の寄与度推定

- **内股**: hip_pos直接効果 ~30%、Yaw二次効果 ~70%
- **タッピング**: ほぼ100% Yaw二次効果（hip_pos復元で解消見込み）
- **Yawドリフト**: hip_pos直接効果 100%（正のフィードバックループで増幅）

#### narrow URDF × hip_pos緩和の非線形増幅メカニズム

| 環境 | hip_pos値 | Yawドリフト速度 | izz |
|------|----------|----------------|-----|
| exp007 | -0.8 | 0.810 °/s | 0.040 |
| exp007 | 0.0 | 5.683 °/s | 0.040 |
| exp008 | -0.8 | 1.801 °/s | 0.029 |
| exp008 | -0.5 | 11.872 °/s | 0.029 |

hip_pos=-0.5（exp008）がhip_pos=0.0（exp007）より2.1倍悪いのは、izz低下（-27.5%→角加速度1.38倍）だけでは説明できない。追加の非線形増幅要因:

1. **レバーアーム短縮**: hip_offset_y 0.10→0.075m（-25%）→ hip_yaw回転のYaw修正トルクが25%低下
2. **安定盆地の縮小**: 狭い脚間隔 → Yaw方向の安定盆地が浅い → わずかな擾乱で盆地外に脱出
3. **フィードバックループ閾値の低下**: V1(hip_pos=-0.8)で辛うじて安定盆地内。-0.5で即座に盆地外に脱出→単調加速

#### hip_pos compound penaltyのnarrow URDFでの構造的矛盾

exp007ではhip_posの compound penalty（hip_yaw²+hip_roll²）は「冗長だが無害」だった。narrow URDFでは:
- hip_rollの必要可動域が増加（狭い支持基底面でバランスを取るため）
- hip_posがhip_rollを制約 → バランス困難化 → より大きなhip_yaw twist（代償運動）
- → compound penaltyがhip_roll制約とhip_yaw増大を同時に引き起こす「構造的矛盾」
- **解決策: hip_yaw専用ペナルティとhip_roll専用ペナルティへの分離**

### 次バージョンへの提案

#### 推奨案: hip_pos→hip_yaw_pos分離（compound penalty構造の解消）

| パラメータ | V2値 | V3値 | 変更理由 |
|-----------|------|------|---------|
| hip_pos | -0.5 | **削除** | compound penalty（hip_yaw²+hip_roll²）の廃止 |
| hip_yaw_pos（新設） | - | **-0.8** | hip_yaw²のみをペナルティ化（分離制御） |
| ankle_roll | -0.5 | -0.5（維持） | hip_roll²制御を継続 |

**根拠**:

1. hip_posのcompound構造がnarrow URDFと非整合（V2で壊滅的に証明）。hip_yawとhip_rollを分離制御する
2. hip_yaw制約レベルは-0.8を維持（V2で-0.5の下限超過を実証）
3. hip_rollの制御はankle_roll=-0.5が既に担当（droid_env_unitree.py:969-990で確認。hip_roll²のみをペナルティ化）
4. 実質1変更（hip_pos→hip_yaw_posの置換）で「1変更1検証」原則に適合
5. 報酬項目数17維持（hip_pos削除+hip_yaw_pos追加=±0、推奨範囲内）

**期待される効果**:
- hip_yaw制約をV1相当に維持 → Yawドリフト≤±20°（V1水準）
- hip_rollがhip_pos制約から解放 → ankle_rollによる独立制御が効きやすくなる
- 内股着地のhip_pos直接効果（~30%）が改善（hip_roll自由度増大 ≠ 内股増加、ankle_rollが制御）

- hip_rollが完全にankle_rollのみの制御になるため、ankle_roll=-0.5で十分かの検証が必要

## まとめ

exp008 V2は、V1からhip_posを-0.8→-0.5に緩和し、hip_yaw自由度の回復によるYawドリフト改善を試みた。

**結果は失敗**: Yawドリフトが壊滅的に悪化（+16.46°→+96.49°、ドリフト速度6.6倍）。さらに目視で「内股着地」「足首タッピング」が確認された。

**3問題の統合因果分析**:
- 3問題は独立ではなく、hip_pos緩和を共通根源とする因果ツリーを形成
- **Yawドリフト**: L stance hip_yaw符号反転（-2.12°→+2.43°）により自己修正機構が崩壊（相関r=-0.76→+0.33）、正のフィードバックループで加速
- **内股着地**: hip_pos直接効果（hip_roll offset +10.6%）~30% + Yaw二次効果（L/R非対称化）~70%
- **足首タッピング**: ほぼ100% Yaw二次効果。L脚のstance中ankle_pitch振動が+32%増大、着地方向逆転（底屈→背屈）によるバウンス

**得られた重要な知見**:
1. **narrow URDFではhip_pos=-0.8が実質的な下限**: izz低下+レバーアーム短縮による非線形増幅
2. **hip_posのcompound penalty（hip_yaw²+hip_roll²）はnarrow URDFと構造的に非整合**: hip_rollの必要自由度増大とhip_yaw制約の必要性が矛盾
3. **hip_yaw自己修正機構の存在と崩壊条件**: V1はr=-0.76で自己修正、V2は制約不足で崩壊→フィードバックループ
4. **Yaw回転はPitch・内股・タッピングを連鎖的に引き起こす**: Yaw制御の喪失は全指標に波及

次のV3ではhip_posのcompound penaltyを解消し、hip_yaw_pos（新設、-0.8）とankle_roll（-0.5、既存）による分離制御を導入する。
