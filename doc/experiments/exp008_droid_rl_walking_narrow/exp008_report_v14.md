# V14: hip_roll PDターゲットクランプ強化

## 概要

V13でPDターゲットクランプの安全性を確認した（歩行崩壊なし、gait frequency/接地パターン完全維持）。しかしオーバーシュート10.6°により内股改善は限定的（±25°→±22°、3°改善のみ）。V14ではクランプ値を-0.20 rad (-11.5°)から-0.05 rad (-2.9°)に強化し、オーバーシュート10°を見込んだ実効値≈-13°（FK解析で足が胴体直下の-12°付近）を狙う。

## 前バージョンからの改善事項

V13レポートの「次バージョンへの提案」の推奨案に基づき、PDターゲットクランプ値を強化する。

| パラメータ | V13値 | V14値 | 変更理由 |
|-----------|-------|-------|---------|
| hip_roll_inward_limit | -0.20 rad (-11.5°) | -0.05 rad (-2.9°) | オーバーシュート10°を考慮し、実効値-13°程度を狙う |

報酬パラメータはV13から一切変更なし（1変更1検証原則）。

## 設計詳細

### クランプ値強化の設計根拠

V13の実測データに基づく設計:

```
V13: target = -0.20 rad (-11.5°) → actual = -0.385 rad (-22.1°)
オーバーシュート = -22.1° - (-11.5°) = -10.6°

V14: target = -0.05 rad (-2.9°) → expected actual ≈ -2.9° + (-10°) ≈ -13°
```

- FK解析でhip_roll = -12°のとき足が胴体中心の真下に来る
- target=-0.05 radで実効値-13°付近を狙い、足を胴体直下近くに配置
- V13の-22°から約9°の追加改善を期待

### オーバーシュートの線形性仮定

V13の1データポイントしかないため、オーバーシュートが線形に維持される保証はない。target値が小さくなるほどPD制御器の目標が中立位置に近づくため、重力トルクに対する復元力の配分が変わる可能性がある。

### fallback計画

歩行品質が悪化した場合: hip_roll_inward_limit = -0.10 rad (-5.7°)に緩和

## 考察と改善案

### PDターゲットクランプ強化の効果チェーン

```

│   ├── オーバーシュート: 10.6°→9.4° (ターゲットが中立に近づき微減)
│   ├── hip_roll mean: -15.69°→-7.15° (全サイクル平均が大幅改善)
│   └── R hip_roll vel RMS: -37.5% (右脚のhip_roll動的活動が大幅減少)
├── [予想外の改善] hip_pitch corr: -0.686→-0.800 (+16.6%)
│   ├── hip_roll固定化→横方向「ノイズ」除去→交互パターン明確化
│   ├── knee_pitch corr: -0.657→-0.815 (膝も改善)

│   └── R hip_roll vel RMS -37.5% → R leg balancing能力低下

│   ├── L hip_pitch range: 0.710→0.604 (-14.9%) → 左脚ストライド短縮

│   └── hip_roll固定化→lateral CoM transfer困難→前方推進効率低下
```

### 核心的発見: オーバーシュートは準線形

2データポイントからのオーバーシュート分析:
- V13: target=-11.5° → actual=-22.1° → overshoot=10.6°
- V14: target=-2.9° → actual=-12.3° → overshoot=9.4°

ターゲットが8.6°変化し、実際値が9.8°変化（比率1.14）。オーバーシュートは10.6°→9.4°に減少（-1.2°）。ターゲットが中立位置に近づくほど重力モーメントアームが小さくなるため、オーバーシュートが微減する理にかなった結果。

### hip_pitch corr改善の解釈

V13では「hip_roll固定化→hip_pitch corr悪化(-13.1%)」が観測され、V14でも同じ方向が予測されていた。しかし実際にはV14で大幅改善（+16.6%）した。

原因仮説: V13ではhip_roll range=0.38 rad（中程度の制限）で「動的バランスに使おうとするが不十分」な状態→hip_pitchの交互パターンにノイズ。V14ではhip_roll range=0.21 rad（強い制限）で「hip_rollによる横方向バランスを諦めてhip_pitchに完全移行」→交互パターンがクリーンに。中間状態より完全な制約の方が安定した歩行戦略をRLが発見しやすい。

### サーベイ知見との照合

- **「joint range ≠ body oscillation」法則**: V14でも成立。hip_roll range -44%だがRoll std +50%（Section 2.2.2/2.2.3）
- **PD制御の定常偏差**: steady-state error ≈ gravity_torque / kp はtarget近傍で準線形。kp=35での限界は構造的（Section 2.5）
- **penalty/positive ratio 0.150はV5のoverkill zone (0.186)の手前**: gait degradationは発生していない（swing duration維持）が、注意領域に入っている

### 成功点

- **内股がFK参照値に到達**: hip_roll内向き ±22°→±12° (10°改善)。FK解析で足が胴体直下の-12°にほぼ一致
- **歩行パターン完全維持**: 0.88Hz、スイング0.49-0.54s、接地パターン正常、歩行崩壊なし
- **hip_pitch corr大幅改善**: -0.686→-0.800 (+16.6%)、knee_pitch corrも-0.657→-0.815。交互歩行パターンがV13より明確化
- **オーバーシュート予測が有効**: 10°を前提に設計し、実測9.4° → 実効値-12.3°でFK参照値-12°にほぼ到達

### 目視確認

内股の改善は確認されたが、**胴体が横方向に大きく並進でスイングしながら歩いている**。その結果、足は胴体の下に潜り込むような動きが見られ、依然として不自然な歩容である。

### 横方向並進揺れの詳細分析

#### 定量的構造

V14の横方向揺れは**歩行周期に完全同期**した周期的振動である:
- Roll自己相関: 第1ピーク lag=1.12s, 値=0.84（高い周期性）
- 周波数: 0.89 Hz ≈ 歩行周波数0.88Hz（1:1結合）
- ランダムノイズではなく、歩行構造に起因する**系統的振動**

| 指標 | V4 (best Roll) | V6 (best Roll) | V13 | **V14** |
|------|---------------|---------------|------|---------|
| Roll std (°) | 3.95 | 3.55 | 3.87 | **5.73** |
| Roll peak-to-peak (°) | 12.86 | 11.83 | 10.94 | **18.03** |
| Roll角速度 RMS (°/s) | 24.3 | 22.3 | 27.4 | **38.8** |
| 横方向並進 detrended std (mm) | 14.9 | 15.9 | 37.9 | **48.0** |
| 横方向並進 p2p (mm) | 47.4 | 53.8 | 146.5 | **181.4** |
| Y速度 std (m/s) | 0.086 | 0.090 | 0.143 | **0.225** |
| Roll-横方向 R² | 0.905 | 0.889 | 0.432 | **0.704** |
| Roll-横方向 slope (mm/°) | -3.60 | -4.21 | -6.46 | **-7.03** |

V14の横方向並進 peak-to-peak **181mm（約18cm）** は視覚的に非常に顕著。V4比で3.2倍。

#### Roll→横方向変換の増幅メカニズム

Roll-横方向変換slope（1°のRollで何mm横方向に動くか）がV4の-3.60からV14の-7.03に**1.95倍に増幅**した。幾何学的予測値（-4.99 mm/°）より40%大きい。

原因: V4では接地相のhip_rollが深い内向き（-24.4°）で、接地足をCoM直下に配置→Roll運動の横方向変位を能動的に打ち消していた。V14ではhip_rollが-10.3°でクランプされ、接地足は股関節直下（CoMの外側）にあるため、Rollが直接的に横方向変位に変換される。

#### 横方向バランス能力の喪失: 根本原因チェーン

```
PDターゲットクランプ強化 (-0.20→-0.05 rad)
│
├── hip_roll動的レンジ: 0.38rad→0.21rad (-44%)
│   ├── 接地-遊脚間hip_rollデルタ:
│   │   V4: L=21.75°, R=16.49° (能動的バランスストローク)
│   │   V14: L=6.34°, R=6.33° (ほぼ固定)
│   │   → 接地相でのCoM直下への足配置が不可能に
│   │
│   └── hip_roll外向き（バランス修正方向）の利用状況:
│       V14: L hip_roll max=0.000 (外向きゼロ), R hip_roll min=-0.000 (外向きゼロ)
│       → ポリシーが外向きhip_roll修正を完全に放棄
│
├── Roll std: 3.87°→5.73° (+48%)
│   ├── 単脚支持中の重力による横方向落下が無補正
│   ├── Roll角速度 +42% (27.4→38.8°/s)
│   └── Roll-横方向slope 2倍 → 同じRollでも横方向変位が増大
│
└── 横方向並進 detrended std: 37.9→48.0mm (+27%)
    └── Roll増加 × slope増幅の複合効果
```

#### 生体力学的解釈

二足歩行の横方向バランスには3段階の制御戦略がある:

1. **足首戦略（最速・最精密）**: 足首ロールによるGRF適用点の微調整 → BSL-Droidは**足首ロール関節なし**（不可能）
2. **股関節戦略（次速）**: hip_roll外転/内転でCoM横方向シフト → V14で**PDクランプにより大幅制限**
3. **足配置戦略（最遅）**: 次の着地位置でCoMを捕捉 → V14で**唯一の残存メカニズム**

V14は第1戦略が構造的に不在、第2戦略がクランプで無力化され、最も遅い第3戦略（~0.5秒の修正サイクル）のみでバランスを取っている。これが横方向揺れの根本原因。

逆ペンダムモデルでの検証:
- ω = √(g/h) = √(9.81/0.286) = 5.86 rad/s
- 有効歩幅 ≈ 54mm（hip_roll 12°での推算）
- 0.5秒の単脚支持中のCoM横方向変位: y(0.5) = 0.027×cosh(2.93) ≈ 255mm（無制御時）

物理法則上、54mmの歩幅で0.5秒の単脚支持は**能動的横方向制御なしでは不安定**。V14の48mm detrended stdはこの物理的制約下で足配置戦略のみで達成している限界値に近い。

#### V4の「パラドックス」: 内股がバランス機構として機能していた

V4（Roll std=3.95°, best）での接地相hip_rollの振る舞い:
- L接地時: L hip_roll = -24.4°（深い内向き）→ 接地足をCoM直下に配置
- L遊脚時: L hip_roll = -2.7°（ほぼ中立）→ 遊脚のスイングに開放
- **接地-遊脚デルタ 21.75°が「能動的バランスストローク」**として機能

つまりV4の「内股」は見た目の問題であると同時に、横方向バランスの能動的制御メカニズムでもあった。V14のPDクランプは内股を解消したが、同時にこのバランスメカニズムも除去した。

### 課題

- **横方向並進揺れ（primary）**: detrended std 48.0mm (V4比3.2倍)、peak-to-peak 181mm。目視で「体を大きく揺らしながら歩く」不自然な歩容。hip_rollバランス機構の喪失が根本原因
- **Roll std +50.0%悪化**: 3.80°→5.70°。hip_rollバランス修正能力の喪失。Roll角速度RMS +42% (27.4→38.8°/s)
- **X velocity -21.0%低下**: 0.200→0.158 m/s。L hip_pitch rangeの縮小(-14.9%)による左脚ストライド短縮が主因
- **Yaw drift +44.5%悪化**: -5.62°→-8.12°。hip_pitch非対称度の増大(-3.8%→-15.1%)による角運動量不均衡
- **penalty/positive ratio +114.3%増大**: 0.070→0.150。全ペナルティ項目が増大、ペナルティ予算の効率低下
- **base_height +10.9%**: 0.286mは目標0.20mから大幅乖離。base_height penalty +145%
- **外向きhip_rollの完全放棄**: L hip_roll max=0.000, R hip_roll min=0.000。現状のクランプは内向きのみ制限しているにもかかわらず、ポリシーが外向き修正を学習していない

### 次バージョンへの提案

#### 推奨案A（V15）: ang_vel_xyスケール強化 (-0.1→-0.2)

| パラメータ | V14値 | V15提案値 | 変更理由 |
|-----------|-------|----------|---------|
| ang_vel_xy | -0.1 | -0.2 | Roll角速度RMS +42%の直接的抑制 |

**設計根拠**:
- Roll角速度RMS: V4の24.3°/s → V14の38.8°/s (+60%)。ang_vel_xyはこの量を直接ペナルティ化する
- V38で2倍増（-0.05→-0.1）が成功した実績: Roll std -30%, 角速度 -34%
- hip_rollクランプ下ではhip_rollによる自然減衰が不在のため、ang_vel_xyのスケール増で**外部減衰力**を付与する設計思想
- penalty/positive ratio: ~0.17予測（V5 overkill zone 0.186の手前）

**期待効果**: Roll std 5.73°→4.0-4.5° (-20~30%改善)、lateral std 48→33-38mm
**リスク**: swing duration < 0.30sならV5パターン（gait degradation）→ fallback -0.15に緩和

#### 推奨案B（V15）: 横方向速度ペナルティ新設 (base_vel_y)

| パラメータ | V14値 | V15提案値 | 変更理由 |
|-----------|-------|----------|---------|
| base_vel_y (新設) | なし | -1.0 | 横方向並進速度の直接抑制 |

**設計根拠**:
- ang_vel_xyは**角速度**（回転）をペナルティ化するが、横方向**並進速度**は直接制約しない。体がゆっくり大きく揺れる場合、角速度は低くても並進揺れは大きい
- Y速度 std: V4=0.086 → V14=0.225 m/s (+162%)。この量を直接ペナルティ化
- 実装: `_reward_base_vel_y(): torch.square(self.base_lin_vel[:, 1])`
- 横方向並進速度を減らすために、ポリシーが現在放棄している**外向きhip_roll修正**を再発見する可能性

**期待効果**: lateral std直接的低減。ポリシーが外向きhip_rollを活用する学習が誘導される可能性
**リスク**: 報酬項目追加（現在17→18）。ただし21+の危険域にはまだ余裕

#### 推奨案A vs B の比較

| 観点 | A: ang_vel_xy強化 | B: base_vel_y新設 |
|------|-------------------|-------------------|
| 変更の種類 | 既存パラメータ調整 | 新規報酬項目追加 |
| 対象量 | Roll/Pitch角速度（回転） | Y方向並進速度 |
| V38実績 | あり（2x→-30% Roll std） | なし（新規） |
| 横方向並進への直接性 | 間接的（Roll→並進の連鎖） | 直接的 |
| リスク | V5 overkill pattern | 未知の相互作用 |
| penalty/positive ratio影響 | ~0.17 | ~0.16-0.17 |

**判断基準**: 確実性重視ならA（V38実績あり）、根本原因（並進揺れ）への直接介入ならB。

#### 中期案（V16-V17）: 接地相限定hip_rollクランプ

現在のPDクランプは歩行全周期で適用されるが、本来必要なのは**接地相のみの内向き制限**。遊脚相ではhip_rollの自由度が足配置戦略に必要。

```
接地相: 内向きクランプ -0.05 rad（内股防止）, 外向き制限なし（バランス修正許可）
遊脚相: 制限なし（足配置の自由度確保）
```

これはenv側の`step()`メソッド変更が必要（接触状態に基づくクランプ切替）。1変更1検証原則に基づき、V15のang_vel_xyまたはbase_vel_yの効果確認後に実施。

#### 長期案: タスク空間バランス報酬

- **Raibert式足配置報酬**: 遊脚着地位置がcapture point予測位置に近いほど正報酬
- **CoM-支持足オフセット報酬**: 単脚支持中にCoMが接地足直上にあるほど正報酬
- **等変ネットワーク (MS-PPO)**: L/R対称性を構造的に保証し、非対称問題を根本解決

#### 以前の提案の再評価

**stance_foot_lateral_position除去**: PDクランプにより冗長化したが、横方向並進揺れが主課題の現状では**除去の優先度は低い**。除去しても横方向揺れの根本原因（hip_rollバランス能力の喪失）は改善しない。ang_vel_xy強化またはbase_vel_y新設の効果を確認した後、ペナルティ予算の最適化として実施を検討。

**base_height_target調整 (0.20→0.26m)**: 依然として有効な提案だが、横方向揺れ対策を優先するため後回し。

## まとめ

V14はPDターゲットクランプ強化(-0.20→-0.05 rad)により、**内股がFK参照値(-12°)にほぼ到達**した。これはexp008の14バージョンにわたる内股問題への取り組みにおいて、初めて「足が胴体直下」のポジションに到達した画期的な成果である。

定量的には、hip_roll内向き値が±22°→±12°（10°改善）、hip_roll全サイクル平均も±15.7°→±7.2°に半減した。予想外にhip_pitch相関も改善(-0.686→-0.800)し、交互歩行パターンがよりクリーンになった。歩行パターン（0.88Hz、スイング時間、接地パターン）は完全に維持された。

一方、**横方向並進揺れが新たな主課題**として顕在化した。detrended lateral std 48.0mm（V4比3.2倍）、peak-to-peak 181mmは目視で非常に顕著な「体を揺らしながら歩く」不自然な歩容を生んでいる。根本原因は、PDクランプにより横方向バランスの唯一の高速制御機構（hip_roll能動修正）が無力化されたことにある。BSL-Droidは足首ロール関節を持たないため、hip_rollのクランプは第1・第2横方向バランス戦略の両方を封じ、最も遅い足配置戦略のみを残している。

次バージョンでは、横方向揺れを直接抑制するアプローチとして、ang_vel_xyスケール強化（既存パラメータ、V38実績あり）またはbase_vel_y新設（横方向並進速度の直接制約）を検討する。中期的には接地相限定クランプ（遊脚での自由度回復）、長期的にはタスク空間バランス報酬（Raibert式足配置等）への移行を視野に入れる。
