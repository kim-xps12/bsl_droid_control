# V11: 非対称hip_rollリミット（URDF構造変更）

## 概要

V4-V8の報酬チューニングおよびV9-V10の対称リミット縮小では内股着地を解消できなかった。本バージョンでは、URDFレベルで非対称hip_rollリミット（内向き-12°制限、外向き+25°維持）を導入し、内股を物理的に防止しつつlateral balanceを確保することを目指す。

## 前バージョンからの改善事項

V10レポートの「次バージョンへの提案」に基づき、非対称hip_rollリミットを採用。V7（best gait quality: Yaw=+1.79°, Roll std=3.51°, penalty/positive=0.102）をベースに、URDF変更のみ実施。

|-----------|------|-------|---------|
| URDF | bsl_droid_simplified_v2.urdf | bsl_droid_simplified_v5.urdf | 非対称hip_rollリミット導入 |
| L hip_roll lower | -25° (-0.4363 rad) | -12° (-0.2094 rad) | 内向き方向を制限 |
| L hip_roll upper | +25° (+0.4363 rad) | +25° (+0.4363 rad) | 外向き維持 |

| R hip_roll upper | +25° (+0.4363 rad) | +12° (+0.2094 rad) | 内向き方向を制限 |

## 設計詳細

### 非対称リミットの設計根拠

1. **内向き-12°制限**: hip_roll=-12°で足先がちょうど股関節直下に位置する（FK解析より）。-12°以下への内向き回転を物理的に禁止することで、足先が体の内側に入り込む内股姿勢を防止
2. **外向き+25°維持**: V9(±12°)およびV10(±18°)で判明した通り、lateral balanceには外向き方向の十分な可動域が不可欠。外向き+25°を維持することで、単脚支持相での横方向CoM移動に必要な自由度を確保
3. **報酬変更なし**: URDF構造変更の効果を単独で評価するため、報酬パラメータはV7から一切変更しない

### 符号規約

- Left hip_roll: 負=内向き（内股方向）、正=外向き
- Right hip_roll: 正=内向き（内股方向）、負=外向き

### リスク

- 非対称URDFがMLP学習を不安定化する可能性（非equivariant architecture）
- 内向き制限が急激すぎる場合、接地相での微小な内向き調整が制約される可能性

## 考察と改善案

### 失敗の因果分析

V11は**歩行が根本的に破綻**した。V9（対称±12°）と同一の崩壊パターンを示す。

**因果チェーン**:
1. 非対称hip_rollリミット（内向き-12°）→ Genesisのソフト制約により部分的にしか効かないが、PD制御のtarget位置が制約される
2. 接地脚は重力により内向き方向にhip_rollが偏位する → 内向き-12°制限により接地相でのCoM横方向微調整の自由度が不足
3. lateral balance喪失 → RLは安全な「L脚立脚」局所最適に収束（R足接地9.2%）
4. 片脚立脚 → hip_pitch同期パターン（corr=+0.836）、高速振動（12Hz）、横方向流れ（Y=0.288 m/s）

**V9との共通崩壊メカニズム**: 内向き-12°制限がV9（対称±12°）とV11（非対称: 内向き-12°/外向き+25°）で共通。外向き方向の自由度を維持しても、**接地脚の内向き制御余裕の不足**が歩行崩壊の直接原因。接地脚は内股方向に偏位しているため、内向き方向の制限はstance相の制御を直接制約する。

### Genesis関節リミットの限界（新知見）

Genesisソースコード調査により以下が判明:
- `control_dofs_position()`はtarget位置をURDFリミットでクランプしない
- 関節リミットはインピーダンスベースのソフト制約（`dmin=0.9, dmax=0.95`）で、完全なハードストップ（imp=1.0）ではない
- PD制御トルクがソフト制約反力を上回ると関節位置がリミットを超過する

これにより、V9/V10/V11のURDF関節リミット変更は意図した「物理的制約」としては不完全に機能していた。

### 報酬コンポーネント分析

- **stance_hip_roll_target -73.9%**: 片脚立脚（R足非接地9.2%）によりペナルティ計上機会が減少（Goodhart's Law）
- **foot_lateral_velocity -83.2%**: 同上。R足がほぼ接地しないため速度ペナルティ対象が減少
- **base_height +259.8%**: 片脚立脚で体が上方に持ち上がり、高さ基準から乖離
- **tracking_ang_vel +31.4%**: 角速度指令0に対し、片脚での回転が少ないため追従改善（偽陽性）

### 成功点
- 該当なし（歩行破綻）

### 課題
1. **hip_roll内向き-12°制限はV9と同一の崩壊メカニズムを引き起こす**: 対称・非対称を問わず、内向き-12°は接地相のCoM制御に不足
2. **Genesisの関節リミットはソフト制約**: URDFリミット変更による物理的制約は不完全
3. **MLP + 非対称URDF = 片脚立脚局所最適**: 非等変アーキテクチャは左右非対称制約で一方の脚のみに依存する戦略に収束

### 次バージョンへの提案

#### URDF関節リミット変更アプローチの全面CLOSED

| Version | hip_roll制限 | 結果 | hip_pitch corr |
|---------|-------------|------|---------------|
| V7 | ±25° | 正常歩行 | -0.783 |
| V10 | ±18° | 部分崩壊（2回転倒） | +0.409 |
| V9 | ±12° | 完全崩壊 | +0.752 |
| V11 | 非対称-12°/+25° | 完全崩壊 | +0.835 |

対称・非対称を問わず、内向き方向の制限が接地相のlateral balanceを破壊する。加えてGenesisの関節リミットがソフト制約である以上、URDF変更による物理的制約は構造的に不完全。本アプローチは完全にCLOSEDとする。

#### `stance_hip_roll_target=-1.0`が不十分な理由

V4-V8で試行した関節角度ペナルティ（stance_hip_roll_target）の単純スケール変更は、以下の理由で内股問題を解決できない:

1. **接地相hip_rollがリミット±25°に飽和（99.8%）**: V8の接地相抽出分析で判明。従来報告の「±15° offset」は全周期平均であり、接地相では常にリミット（L=-24.96°, R=+24.96°）に張り付いている。スケールを増加してもペナルティ勾配は「壁」に当たり、関節は動けない
2. **Pareto frontierは既に探索済み**: V4(-0.5→不足), V5(-2.0→歩行劣化), V8(target angle→非対称応答)。-1.0はV4とV5の中間点に過ぎず、同じ「効率 vs ペナルティ」曲線上の別の点
3. **RLの補償戦略**: hip_rollが飽和している以上、ペナルティ増加に対しRLはhip_rollを変えず他の関節（hip_pitch等）で補償する。V5ではhip_pitch range +27%でバランス補償し、内股は解消されなかった
4. **V8のターゲット角度も同じ限界**: (hip_roll - 5°)²としても、offset(25°)>>target(5°)のため(20°)² ≈ (25°)²。飽和領域ではターゲットシフトの効果が極めて小さい

**結論: hip_roll角度空間での操作は、飽和点での壁に押し続けるに等しく、スケール調整では突破不可能**

#### V12推奨案: パラダイムシフト — 関節角度→足位置

**根本的な問い**: 内股の視覚的問題は「足が体幹中心に近すぎる」こと。であれば関節角度（hip_roll）ではなく、**足の横方向位置**を直接ペナルティ化すべきである。

**stance_foot_lateral_position（接地足横位置ペナルティ）**:
- 接地足の横方向位置（体幹中心からの距離）をFK鎖の結果として直接計測
- 最小距離（50mm）以下の場合のみペナルティ（片側制約）
- stance_hip_roll_targetの1対1置換（他パラメータ変更なし）
- V8からコピー、stance_hip_roll_target→stance_foot_lateral_positionに置換

**足位置が関節角度より優れている理由**:

| 観点 | 関節角度（現行） | 足位置（提案） |
|------|----------------|--------------|
| 最適化対象 | hip_roll²（間接指標） | foot_lateral_distance（直接指標） |
| 関節自由度 | hip_rollのみ制約 | 任意の関節組合せで達成可能（hip_pitch, knee, hip_yawも寄与） |
| Pareto frontier | hip_roll飽和域で壁 | FK鎖全体の解空間が広く、新経路が存在しうる |
| 片側制約 | 0°からの二乗偏差（常にペナルティ） | 最小距離以下のみペナルティ（それ以上は完全に自由） |
| MLPの非対称性 | 対称targetでも非対称応答（V8実証） | 各足独立に位置で評価 |
| 非線形FK | hip_roll=-25°付近では角度微小変化の位置効果が小さい（cos減衰） | 位置空間で線形にペナルティが作用 |

**V8からの論理的派生**:
- V8: hip_roll角度に±5°ターゲット → 関節空間でのターゲット設定
- V12: 足位置に50mmターゲット → タスク空間（足位置）でのターゲット設定
- V8のFK幾何学的知見（hip_roll=-12°で足がヒップ真下、足幅77mm）を設計根拠として直接活用

**Pareto frontier突破の論拠**:
1. 解空間の拡大: hip_roll以外にhip_pitch, knee, hip_yawの組合せで足位置制御可能 → エネルギー効率を犠牲にしない新経路の存在可能性
2. 片側制約: 最小距離以上なら完全に自由 → 不必要な制約コストなし
3. 非線形FK: hip_roll=-25°の飽和領域でも、足位置ペナルティは直接的な勾配を提供（関節角度ペナルティのcos減衰を回避）

**foot_lateral_velocityとの報酬間競合に関する注記**:
V7で発見された通り、foot_lateral_velocityは横方向速度最小化のため足を内側に寄せる副作用がある。V12ではfoot_lateral_velocityは維持し（1変更1検証）、足位置ペナルティとの時間的分離（stance vs swing）で競合を軽減する。競合が問題となればV13で対処する。

#### V13以降への展望

- **速度圧力低減**: tracking_lin_velのスケール調整またはコマンド範囲縮小（[0.10, 0.15]等）。RLがエネルギー効率を過度に最適化する暗黙的圧力を軽減し、広い足幅のコストを受容しやすくする。明示的エネルギー報酬（action_rate=-0.005等）は既にほぼ無効化されており、真の駆動力は速度追従報酬+安定性ペナルティによる暗黙的効率圧力
- **foot_lateral_velocity競合解消**: V12で競合が確認された場合、scale低減または方向限定修正
- **env-level action clamping**: V12の足位置ペナルティが不十分な場合、Genesisソフト制約問題の根本修正としてenv-levelでPDターゲットのハードクランプを実装
- **長期**: equivariant network（MS-PPO/ROGER）導入で左右対称性の構造的保証

## まとめ

V11は非対称hip_rollリミット（内向き-12°制限、外向き+25°維持）を導入したが、**歩行が根本的に破綻**した。hip_pitch corr=+0.836（同期）、R足接地9.2%、両足空中41.6%、歩行周波数12Hz（振動）と、V9（対称±12°）と同一の崩壊パターンを示す。

原因は「外向き自由度を維持すればlateral balanceは確保できる」という設計仮説の誤り。接地脚は内股方向（内向き）に偏位するため、内向き-12°制限はstance相のCoM横方向制御を直接制約し、歩行を破壊する。

追加の発見として、Genesisシミュレータの関節リミットはソフト制約であり、関節位置がURDFリミットを大幅に超過する（R hip_roll: リミット-25°に対し実測-100°）。URDF関節リミット変更による内股対策は（対称・非対称を問わず）構造的に限界がある。

V12ではURDF変更にも関節角度ペナルティのスケール調整にも頼らず、報酬設計のパラダイムを根本的に転換する。V8の根本原因分析（接地相hip_roll飽和99.8%、FK幾何、Pareto frontier）を踏まえ、関節角度ではなく**足の横方向位置を直接ペナルティ化**する新報酬（stance_foot_lateral_position）を導入し、内股問題の構造的突破を試みる。
