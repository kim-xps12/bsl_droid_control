# V17: foot_lateral_velocity削除（バランシング衝突解消）

## 概要

V16の位相条件付きPDクランプにおいて、foot_lateral_velocityペナルティが遊脚hip_rollバランシング機構と直接衝突していることが判明（V16で+69%増加）。PDクランプにより内股防止は構造的に解決済みのため、foot_lateral_velocityの存在理由が消失。削除によりペナルティ予算の健全化と報酬項目数の推奨範囲復帰を狙う。

## 前バージョンからの改善事項

V16レポートの「次バージョンへの提案」推奨案D（V16後の追加分析により案Aから変更）に基づき、foot_lateral_velocityを無効化する。

| パラメータ | V16値 | V17値 | 変更理由 |
|-----------|-------|-------|---------|
| foot_lateral_velocity | -0.5 | 0（無効化） | V16で+69%増加＝遊脚hip_rollバランシングとの衝突。exp007 V7で反生産的と確認済み。PDクランプで内股防止は構造的に解決済み |

他のパラメータは一切変更なし（1変更1検証原則）。報酬項目数は18→17（有効13→12）。

## 設計詳細

### foot_lateral_velocity削除の設計根拠

1. **バランシング機構との直接衝突**: foot_lateral_velocityは全足・全位相の横方向速度をペナルティ化。V16の遊脚hip_roll自由化で横方向足速度が発生するのはバランシングの正常動作。V16でこのペナルティが+69%増加したのは、方策がバランシングと回避の矛盾する勾配信号を受けている証拠。
2. **歴史的に反生産的**: exp007 V7で「foot_lateral_velocity counterproductive: Promotes 内股」と確認済み。
3. **ペナルティ予算回復**: ratio 0.193→~0.153予測（健全域に回復）。
4. **報酬項目数**: 18→17（有効13→12、推奨範囲に回復）。

### 報酬項目数

V17: 17項目（有効12項目）。V16から1項目削除。

## 考察と改善案

### 成功点

1. **penalty/positive ratio大幅改善**: 0.193→0.153（-20.7%）。V5 overkill zone (0.186)から健全域に大幅回復。
2. **X速度改善**: +11.3%（0.151→0.168 m/s）。ペナルティ衝突解消によりforward progressが改善。
3. **Roll std改善**: -16.0%（8.25→6.93°）。V16で問題だった過剰Roll振動が軽減。
4. **Yaw drift改善**: -22.6%（絶対値20.07→15.53°）。
5. **orientation penalty減少**: -14.9%（-0.047→-0.040）。胴体の鉛直性が改善。
6. **tracking_ang_vel大幅向上**: +57.5%。角速度追従精度が改善。

### 課題

1. **遊脚hip_rollの過剰な外向き広がり（ユーザー目視で悪化と判断）**: 遊脚swing meanが+8.1°/-9.3°（外向き方向）、外向き最大±26-27°。balance strokeは+16.7°/-18.1°でV4基準値（5-10°）の2.6倍。毎サイクルで関節リミット近辺まで到達する「カタパルトパターン」を形成。

2. **横方向位置std悪化**: +20.2%（36.2→43.5mm）。Roll-lateral R²が0.640→0.038に崩壊。横方向変位がRoll経由ではなく遊脚hip_rollの運動連鎖による直接質量中心変位で発生（残差std: 21.7→42.7mm）。orientationペナルティでは検出・抑制不可能。

3. **接地相クランプ有効性の低下**: stance hip_roll stdが2.3→8.0°に3.5倍悪化。遊脚の大振幅慣性がstance開始時に持ち越される（stance outward到達率: L 4.6%, R 5.1%）。

4. **歩行規則性の更なる悪化**: swing duration半減（0.39→0.19s）、チャタリング13回（V16: 9回）。遊脚の最大外向き到達後の急速引き戻しがチャタリングを誘発。

5. **動作の非滑らかさ**: hip_roll加速度RMSがV15比4.2-6.6倍悪化。ジャーキーな動作は「ゆったり自然に可愛く歩く」目標に反する。

### 因果メカニズム

```
foot_lateral_velocity削除
├→ 遊脚の横方向速度に制約なし
│  ├→ 外向きhip_roll excursion爆発（±26-27°、balance stroke 2.6xV4）
│  │  ├→ 運動連鎖経由のCoM横方向変位 → lateral std +20.2%、R²→0.038
│  │  ├→ 遊脚慣性のstance持ち越し → stance std 3.5x悪化
│  │  └→ カタパルトパターン（最大外向き→急速引戻し）→ チャタリング増加
│  └→ 前進推力への変換 → X速度 +11.3%
├→ ペナルティ衝突解消 → 報酬最適化効率向上
│  ├→ penalty/positive ratio 0.193→0.153
│  └→ tracking_ang_vel +57.5%
└→ 胴体Roll直接抑制不要 → Roll std -16.0%
```

### 根本原因: 制約の非対称性

V17の失敗の根本原因は**制約の非対称性**:

| 方向 | 制約 | 位相 | 状態 |
|------|------|------|------|
| 内向き | PD clamp -0.05 rad | 接地相のみ | **あり** |
| 外向き | なし | なし | **なし** |
| 横方向速度 | なし（V17で削除） | なし | **なし** |

内向きPDクランプは存在するが、外向き方向には一切制約がない。foot_lateral_velocityは暗黙的に外向きhip_rollの天井として機能していたが、バランシング機構との衝突を引き起こしていた。V17での削除により、外向き方向が完全に無制約となり、方策が関節リミットまでの外向きswingを学習した。

### 次バージョンへの提案

**推奨案A（推奨）: 双方向PDクランプ — 遊脚相の外向き制限追加**

PDクランプ方式はV13-V14で有効性が確認済み。同じパラダイムを外向き方向に拡張する。

- **新設パラメータ**: `hip_roll_outward_limit = 0.20 rad`（遊脚相のみ適用）
- **実装**: `droid_env_unitree.py`の`step()`内、遊脚条件で外向きhip_rollターゲットをクランプ
  - L脚遊脚時: `target <= +0.20 rad`（外向き制限）
  - R脚遊脚時: `target >= -0.20 rad`（外向き制限）
- **期待されるhip_roll corridor**: 接地相は内向き-0.05 rad制限、遊脚相は外向き+0.20 rad制限
- **期待される実効範囲**: オーバーシュート~9°前提で、外向き最大 0.20+0.16=0.36 rad、内向き最大 0.05+0.16=0.21 rad → 合計~0.57 rad（V4の0.54-0.69 rad範囲に近い）
- **報酬項目数**: 17（変更なし）。コード変更のみ
- **1変更1検証**: 単一の概念的変更（外向きswing制限の追加）

**案B: foot_lateral_velocity再有効化 (-0.2、V16の40%)**

- foot_lateral_velocity: 0 → -0.2
- V16(-0.5)ではhip_roll range 0.44-0.52 rad（許容範囲だがチャタリング）、V17(0)では0.83-0.88 rad（過大）
- -0.2なら中間の~0.55-0.70 rad を期待
- ただしV7で「counterproductive」と確認済みの構造的衝突が残存（規模は軽減）
- コード変更不要の最もシンプルな変更

**案C: stance_foot_lateral_position削除 (-0.5→0) + foot_lateral_velocity再有効化 (-0.2)**

- 2変更同時のため1変更1検証原則違反
- stance_foot_lateral_positionはPDクランプと冗長（V14以降確認済み）

## まとめ

V17はfoot_lateral_velocity削除（-0.5→0）により、penalty/positive ratio 0.193→0.153（-20.7%、健全域回復）、X速度+11.3%、Roll std -16.0%、Yaw drift -22.6%の改善を達成した。

しかし、foot_lateral_velocityが暗黙的に担っていた外向きhip_rollの天井が消失し、遊脚hip_rollが±26-27°（関節リミット近辺）まで到達する「カタパルトパターン」を形成。balance strokeはV4の2.6倍に達し、ユーザー目視で「脚を外側に変に広げている」と判断された。横方向位置stdは+20.2%悪化、Roll-lateral R²は0.038に崩壊、チャタリングは13回に増加。

根本原因は制約の非対称性（内向きPDクランプあり、外向き制約なし）。次バージョンではPDクランプの双方向化（遊脚相に外向き制限0.20 rad追加）を推奨する（案A）。

## 備考

- V16→V17で唯一の変更はfoot_lateral_velocity削除のみ。結果の差異はすべてこの変更に起因
- foot_lateral_velocityの役割は「内股防止」（V7設計意図）だけでなく、「外向きswing制限」の暗黙的天井でもあった。この二重機能がV17で判明
- penalty/positive ratio 0.153は数値的には健全だが、制約不足（外向き方向の自由度過剰）を反映しており、ratioの低さ＝良好とは限らない
- 「カタパルトパターン」は遊脚を最大外向きに振り→急速に引き戻す動作。前進推力に変換されX速度+11.3%だが、視覚的に不自然
- 双方向PDクランプ（案A）はV13-V14で確認されたPDクランプパラダイムの自然な拡張であり、報酬整形よりも信頼性が高い
