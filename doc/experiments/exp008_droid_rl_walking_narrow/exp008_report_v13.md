# V13: hip_roll PDターゲットクランプ（ハード制約）

## 概要

V4-V12の9バージョンで、報酬ペナルティ（ソフト制約）による内股解消はPareto frontierに到達し、これ以上の改善は不可能と判断された。V13ではソフト制約からハード制約へパラダイムシフトし、env-levelでhip_rollのPDターゲットを-0.20 rad (-11.5°)にクランプすることで、RLの内股方向指令を物理的に制限する。

## 前バージョンからの改善事項

V12レポートの「次バージョンへの提案」に基づき、推奨案であるenv-levelでのhip_roll PDターゲットクランプを実施する。

| パラメータ | V12値 | V13値 | 変更理由 |
|-----------|-------|-------|---------|
| hip_roll_inward_limit | なし（新設） | -0.20 rad (-11.5°) | PDターゲットの内向き制限 |

報酬パラメータはV12から一切変更なし（1変更1検証原則）。

## 設計詳細

### PDターゲットクランプの動作原理

現在のロボットはPD制御で動作する。RLのニューラルネットが出力する目標角度をPD制御器に渡す前に、内向き方向のみをクランプする:

```
【現状（V12以前）】
RLの指令: 「左hip_rollを-25°にしたい」
    ↓ そのまま渡す
PD制御器 → モーターが-25°に向かって力を出す → 内股

【V13】
RLの指令: 「左hip_rollを-25°にしたい」
    ↓ -11.5°に書き換え
PD制御器が受け取る目標: 「-11.5°」
    ↓
モーターが-11.5°に向かって穏やかな力を出す → 内股が軽減される可能性
```

### V9-V11（URDF関節リミット変更）との違い

- **V9-V11**: 関節の物理的可動範囲を制限 → PD制御器は元の目標(-25°)に向かって全力で力を出す → 壁にぶつかって反発トルク → 歩行崩壊
- **V13**: PDターゲット自体を制限 → PD制御器は-11.5°に向かって穏やかな力を出す → 壁にぶつからない → 安定した制御が維持された

### クランプ値の設計根拠

- 現状のhip_roll接地相値: **-25°**（リミットいっぱい）
- FK解析でhip_roll = -12°のとき足が胴体中心の真下に来る
- 初回クランプ値: **-11.5°（-0.20 rad）**
  - -25°（現状）と-12°（胴体直下）の間の保守的な値
  - PD制御の応答遅れとGenesisの動力学により、実際のhip_rollは-11.5°より内側になると予測された

### 実装箇所

`droid_env_unitree.py`の`step()`関数内、`target_dof_pos`計算後・`control_dofs_position()`前:

```python
if self.hip_roll_inward_limit is not None:
    target_dof_pos[:, self.left_hip_roll_idx] = torch.clamp(
        target_dof_pos[:, self.left_hip_roll_idx], min=self.hip_roll_inward_limit
    )
    target_dof_pos[:, self.right_hip_roll_idx] = torch.clamp(
        target_dof_pos[:, self.right_hip_roll_idx], max=-self.hip_roll_inward_limit
    )
```

## 考察と改善案

### PDターゲットクランプの効果チェーン

```
PDターゲット制限(-0.20 rad)
├── [直接効果] hip_roll range 57%縮小 (0.89→0.38 rad)
│   ├── hip_roll vel RMS 75-80%減 (ほぼ固定化)
│   ├── stance_foot_lateral_position penalty -54.7% (足がやや外側に)
│   └── foot_lateral_velocity penalty -76.2% (横方向動きが減少)
├── [副次効果1] hip_roll動的バランス能力喪失
│   ├── Roll std +13.8% (能動的修正不可→受動的揺れ増大)
│   └── hip_pitch corr -13.1% (横方向重心移動制限→交互パターン不明確化)
├── [副次効果2] 姿勢変化
│   ├── base height +7.5% (0.240→0.258m, 直立化)
│   └── base_height penalty +81.6% (目標0.20mから乖離)
└── [副次効果3] 歩行効率改善
    ├── X velocity +8.1% (エネルギーが横方向→前方に再配分)
    ├── tracking_ang_vel +33.4% (hip_roll振動減→角速度追従改善)
    └── penalty/positive ratio 0.070 (best ever)
```

### 核心的発見: PDターゲット≠実際の関節位置

PDターゲットを-0.20 rad (-11.5°)にクランプしたが、実際のhip_rollは-0.385 rad (-22.1°)に到達。オーバーシュート10.6°の原因:
- 単脚支持相で重力が脚を内側に引く（重力トルク推定: ~2.35 Nm）
- PD制御器の復元力: kp × (target - actual) = 35 × 0.185 = 6.475 Nm
- PD復元力は重力に勝っているが、動力学的な慣性・接触力で定常偏差が残る
- V12の±25°飽和 → V13の±22° = **3°改善のみ**

### サーベイ知見との照合

- **「joint range ≠ body oscillation」法則が成立**: V37/V38で確立された法則がV13でも確認。hip_roll range -57%だがRoll std +13.8%（サーベイ Section 2.2.2/2.2.3）
- **PDターゲットクランプは「セミハード制約」**: URDFリミット変更（V9-V11、完全崩壊）よりは穏やかだが、物理エンジンの動力学でターゲット値の2倍まで逸脱。純粋なハード制約ではない
- **penalty/positive ratio改善はGoodhart's Law的**: 0.070（best ever）はペナルティ対象の行動自体を不可能にした結果であり、V7（0.102）の「自然な報酬バランス最適化」とは質的に異なる

### 成功点

- PDターゲットクランプによりV9-V11のような歩行崩壊なく、hip_roll可動域を57%制限できた
- X velocity +8.1%、penalty/positive ratio 0.070（全実験best）
- 歩行パターン（周波数0.88Hz、スイング0.49-0.53s、接地パターン）が完全に維持された
- hip_roll stance phase: ±25° → ±22°に部分的改善（3°）

### 目視確認

目視では多少見た目がマシになった程度で、大きな変化は感じられなかった。定量データ（±25°→±22°、3°改善のみ）と一致する。

### 課題

- **内股の改善が限定的**: PDターゲット-11.5°に対し実際値-22°。ターゲットの2倍までオーバーシュート
- **hip_pitch corr -13.1%悪化**: hip_roll固定化で横方向バランスが制約され、交互歩行パターンが不明確化
- **Roll std +13.8%悪化**: hip_rollによる能動的バランス修正能力が喪失
- **base_height penalty +81.6%**: 姿勢の直立化による目標値（0.20m）からの乖離

### 次バージョンへの提案

#### 推奨案（V14）: PDターゲットクランプ値の強化

| パラメータ | V13値 | V14提案値 | 変更理由 |
|-----------|-------|----------|---------|
| hip_roll_inward_limit | -0.20 rad (-11.5°) | -0.05 rad (-2.9°) | オーバーシュート10°を考慮し、実効値-13°程度を狙う |

**設計根拠**:
- V13でPDターゲットクランプの安全性を確認（歩行崩壊なし、gait frequency/接地パターン維持）
- オーバーシュート約10°を前提とすると、target=-0.05 radで実際値≈-13°（FK解析で足が胴体直下の-12°付近）
- V13の-22°から9°の追加改善が期待される
- fallback: 歩行品質悪化なら-0.10 rad (-5.7°)に緩和

#### 代替案: stance_foot_lateral_position除去

- V13でstance_foot_lateral_positionのペナルティが-54.7%に低下 → PDクランプと機能重複
- ただし2変更同時は1変更1検証原則に違反するため、V14ではクランプ値変更のみを実施し、V15で除去を検討

## まとめ

V13はPDターゲットクランプの初回実験として、**歩行安定性を維持しながらhip_roll可動域を57%制限できた**ことが最大の成果である。V9-V11のURDF関節リミット変更（歩行崩壊）とは対照的に、PDターゲット制限は穏やかに作用し、歩行パターン（周波数、スイング時間、接地パターン）を完全に保持した。

一方、内股改善は限定的で、PDターゲット-11.5°に対し実際のhip_rollは-22°（オーバーシュート10.6°）に留まった。これはPD制御の構造的限界（kp=35では重力+動力学に完全には抗えない）であり、次バージョンではクランプ値をさらに強化（-0.05 rad）してオーバーシュートを見込んだ設計とする。

## 備考

- PDターゲットクランプはreward_cfgで設定（`hip_roll_inward_limit`）。Noneの場合はクランプなし（既存バージョンとの互換性維持）
- 段階的調整計画:
  - V13: -0.20 rad (-11.5°) → 安全性確認済み、実効値-22°（3°改善のみ）
  - V14: -0.05 rad (-2.9°) → 実効値-13°を狙う
  - fallback: 歩行品質悪化なら-0.10 rad (-5.7°)に緩和
