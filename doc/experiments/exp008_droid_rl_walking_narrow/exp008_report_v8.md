# V8: stance_hip_roll_targetターゲット角度設定（±5°外向き）

## 概要

V7では内股着地（hip_roll inward offset L:-14.84°, R:+12.93°）が依然未解決であった。現在のstance_hip_roll_targetはhip_roll²=0をターゲットとしており、ペナルティの最小点が0°にあるため内股方向への矯正勾配が不十分であった。V8ではペナルティ関数をhip_roll²から(hip_roll - target)²に変更し、明示的に±5°（0.087 rad）の外向きターゲット角度を設定することで、内股着地の直接的な改善を狙う。

## 前バージョンからの改善事項

V7レポートの「次バージョンへの提案」代替案B（ターゲット角度設定）に基づく改善。

| パラメータ | V7値 | V8値 | 変更理由 |
|-----------|------|------|---------|
| stance_hip_roll_target_angle | 0.0 rad（未設定、hip_roll²=0） | 0.087 rad（±5°外向き） | ペナルティ最小点を0°→±5°に移動し、内股方向への矯正勾配を増大 |

報酬スケール（stance_hip_roll_target=-0.5）は変更なし。報酬項目数17維持。

## 設計詳細

### (hip_roll - target)²ペナルティの仕組み

V4-V7の`stance_hip_roll_target`は`hip_roll²`（ターゲット0°）をペナルティ化していた。V8では`(hip_roll - target)²`に変更:

- **L脚**: `(left_roll - 0.087)²` → hip_roll = +5°（外向き）で最小化
- **R脚**: `(right_roll + 0.087)²` → hip_roll = -5°（外向き）で最小化

### 符号規則

URDF: 両脚とも `axis xyz="1 0 0"`
- L hip_roll: 負 = 内股（内側）、正 = 外向き
- R hip_roll: 正 = 内股（内側）、負 = 外向き

### ターゲット角度±5°の選定根拠

1. V7の内股offset平均~14°に対し、5°外向きターゲットは19°の矯正距離 → 十分な勾配
2. V5で±4°到達実績あり（-2.0スケールで達成、ただしgait degradation発生）
3. スケール-0.5維持 + ターゲット移動のみ → V5のような実効ペナルティ過大を回避
4. 10°では過矯正（ガニ股）リスクが高い

### 後方互換性

`stance_hip_roll_target_angle`のデフォルト値は0.0 rad。V1-V7は`target_angle`未設定のため、`hip_roll²`と同等の動作を維持。

### リスク評価

- ガニ股（過矯正）リスク: 低（5°は保守的、V5で4°到達実績）
- gait degradation: 低（スケール維持、ターゲット移動のみ）
- Yaw悪化: 低-中（hip_roll offset改善はYaw安定に寄与する可能性もある）

## 考察と改善案

### 主目的の部分的達成と全体的悪化

V8は「部分的成功・全体的悪化」の結果となった:

- **R脚の内股offset -23.3%改善**: +12.93°→+9.92°。ターゲット角度が機能した直接的証拠
- **L脚の微悪化**: -14.84°→-15.00°（+1.1%）。RL確率性によるL/R非対称最適化
- **Yaw drift 8.3x悪化**: +1.79°→-14.81°。hip_roll range非対称化が主因
- **hip_pitch相関-19.2%低下**: -0.836→-0.667。Yaw driftによる歩行パターン非対称化の二次効果

### 因果分析

#### ターゲット角度がR脚のみ改善をもたらした機構

1. **ペナルティ energy landscape の変形**: target=0.087 rad設定により、ペナルティの最小点が0°→±5°に移動。L/R両脚に同等の外向き勾配を生成
2. **RL確率性によるL/R非対称応答**: MLPポリシーはL/Rを独立に最適化するため、同一の対称報酬でも片脚のみ応答する結果が生じうる（exp007 V34-V35で確認済み、Survey Section 7.2.3）
3. **R脚のみ応答**: R hip_roll offset +12.93°→+9.92°(-23.3%)、L hip_roll offset -14.84°→-15.00°(+1.1%)。L脚はペナルティ勾配がRより大きい（距離0.346 rad vs 0.313 rad）にもかかわらず改善しなかった

#### Yaw drift 8.3x悪化の機構

1. **hip_roll range非対称化**: ターゲット角度設定により、R hip_roll range 0.887 rad >> L hip_roll range 0.664 rad（ratio 1.336）に拡大
2. **angular momentum imbalance**: R脚スイング時の角運動量がL脚より大きくなり、net angular momentumが-Yaw方向にシフト（Survey Section 7.4.2）
3. **V7の回復型パターンの消失**: V7は4-10sでYaw回復したが、V8では回復なし。hip_roll非対称によりYaw自己修正メカニズムが破壊された
4. **Yaw方向の符号反転**: V7 +1.79° → V8 -14.81°。R脚の外向き改善がR>Lの角運動量不均衡を生成

#### hip_pitch相関低下の機構

1. **Yaw drift→歩行パターン非対称化**: -14.81°のYaw driftに「適応」するため、RLがL/Rで異なるhip_pitch戦略を採用
2. **penalty budget圧迫**: stance_hip_roll_target +23.6%増がhip_pitch最適化の余力を削減
3. **DOF range拡大の副作用**: hip_roll自由度の実質的拡大（ターゲット非ゼロ化）がhip_pitchの運動予算を分散

### ターゲット角度アプローチの構造的限界

1. **報酬shapingの限界**: ターゲット角度設定は報酬関数の形状変更であり、ソフト制約の改良に過ぎない。MLPはL/R独立に最適化するため、片脚のみ応答する構造的限界がある（Survey Section 7.2.3）
2. **hip_roll制御とYaw安定性の結合**: hip_rollの制約を変更するとangular momentum balanceが崩れてYaw driftが発生する。これはhip_pos compound penaltyとは異なる経路だが、「関節レベルの制約変更がbody-levelの安定性に波及する」という同種の現象
3. **orientation + ang_vel_xyはYaw軸を制御しない**: これらの補償メカニズムはRoll/Pitch軸に限定されており、hip_roll非対称化によるYaw悪化を補償できない

### V8補遺: 内股問題の根本原因分析（8バージョン総括）

V8完了後、目視で内股が全く改善していないことを受け、接地相のCSVデータ・URDF構造・環境コードを総合的に再調査した。その結果、**これまでの分析で見落としていた決定的な事実**が判明した。

#### 発見1: 接地相hip_rollは関節リミット（±25°）に飽和している

これまでのレポートでは「hip_roll offset ±15°」と報告していたが、これは**歩行全体（接地相＋遊脚相）の時間平均値**であった。接地相のみを抽出した結果:

| 接地相 hip_roll | V7 | 関節リミット | リミット使用率 |
|---|---|---|---|
| L脚 | **-24.96°** | -25.0° | **99.8%** |
| R脚 | **+24.96°** | +25.0° | **99.8%** |
| 接地相std | 0.65° / 1.41° | - | - |

**RLポリシーは毎回の接地時にhip_rollを物理的限界まで内向きに飽和させている。** stdが0.65°であり、事実上ロックされた状態である。遊脚相で大きく外向きに振り出すため、全体平均では±15°に見えていた。

#### 発見2: 足の実位置は77mm間隔（210mmの37%）

URDFの前方運動学に基づく足の横方向位置:

| 指標 | 現状（接地相） | ニュートラル（hip_roll=0°） |
|---|---|---|
| L足 Y位置 | +37.7mm | +105.0mm |
| R足 Y位置 | -39.1mm | -105.0mm |
| **足間距離** | **76.7mm** | **210.0mm** |
| 狭窄率 | - | **63.4%** |

各足は股関節位置（±75mm）から更に約37mm内側に入り込んでいる。

URDF幾何学的参照点:
- hip_roll = **-12°（L）/ +12°（R）**: 足がちょうど股関節直下（±75mm）
- hip_roll = **0°**: 足は股関節より30mm外側（±105mm）。hip_roll→hip_pitchの30mm lateral offsetによる
- hip_roll = **-25°（L）/ +25°（R）**: 足は股関節より37mm内側（±37.7mm）← **現在の状態**

#### 発見3: RLが内股を選択する物理的メカニズム

5つの構造的要因の重畳:

1. **CoMの重力トルクが内向きを助勢**: 片脚立ちの際、CoMが支持脚より内側にある → 重力が内向きhip_rollの方向に作用。内股ほど**エネルギー効率が高い**
2. **足間距離の縮小がCoM移動コストを削減**: 77mm間隔では左右の重心移動距離が最小化 → 歩行全体の効率向上。**RL的には最適解**
3. **ankle roll関節の不在**: hip_rollが横方向の唯一の自由度。胴体横傾斜と足底接地角度を独立制御できない
4. **半円筒形足底（kamaboko）**: 内側への傾斜に対して復元モーメントが小さい。フラット足底なら即座に抵抗するが、R=30mmの円筒では滑らかに転がる
5. **関節リミット飽和でペナルティ勾配がゼロ**: ±25°リミットに張り付いているため、quadraticペナルティは「これ以上内向きに行けない方向」への力しか提供しない。外向き方向の勾配は-0.5×2×0.436=0.436だが、RLの効率メリットがこれを上回る

#### 発見4: 報酬チューニングで解決できない構造的理由

V4-V8で試行した全アプローチの共通限界:

| バージョン | アプローチ | 結果 | 限界の理由 |
|---|---|---|---|
| V4 | stance_hip_roll_target -0.5 | offset ±25° | ペナルティ不足。効率メリット > ペナルティコスト |
| V5 | stance_hip_roll_target -2.0 | offset ±4° | 達成したがgait degradation。歩行品質と内股はトレードオフ |
| V6 | orientation -2.0 | R offset -29% | body-level制約は間接的。Yaw悪化 |
| V7 | foot_lateral_velocity -0.5 | offset微悪化 | 逆効果。内側に脚を寄せるインセンティブ |
| V8 | target angle ±5° | R offset -23% | MLP非対称応答。Yaw 8.3x悪化 |

**共通パターン**: ペナルティを控えめに設定すると効果なし、強めに設定するとgait degradation。「内股の効率メリット」と「ペナルティコスト」の競合であり、報酬チューニングでは**パレート最適のトレードオフ**を超えることができない。

この問題の本質は**関節リミット±25°が内股を許容する物理的制約空間の広さ**にある。RLは物理的に許容される範囲内で最も効率的な解を見つけており、報酬で「効率的だが望ましくない解」を排除しようとしても、効率との競合が不可避。

#### 結論: 物理構造変更が必要

exp008 V1で胴体幅75%縮小が40バージョンの報酬チューニングを超える効果を発揮したのと同じ原理（Survey Section 8.1: 物理構造 > 報酬設計）に基づき、**関節リミットの変更により内股が発生しない制約空間を設定する**のが正しいアプローチである。

| -10° | ~+85mm | ~10mm外側 | 軽度外側 |
| **-12°** | **~+75mm** | **股関節直下** | **中立** |
| -15° | ~+65mm | ~10mm内側 | 軽度内股 |
| -25° | +37.7mm | 37mm内側 | **現状（重度内股）** |

- **±12°で足がちょうど股関節直下**: これより内向きには行けないため、足が股関節より内側に入り込むことが**物理的に不可能**になる
- **バランスの自由度を確保**: 0°〜-12°の12°の可動域で外側〜直下をカバー。通常歩行の股関節内外転は5-10°であり十分
- **遊脚相の足運びにも十分**: 現在の遊脚相hip_rollは-25°〜+11°（36°幅）だが、これは接地相の-25°からの復帰が大部分。±12°制約下では遊脚相も±12°内に収まり、24°幅で足運びが可能

## まとめ

V8は`stance_hip_roll_target`のペナルティ関数を`hip_roll²`から`(hip_roll - target)²`に変更し、±5°の外向きターゲット角度を設定する1変更を実施した。**R脚の内股offsetは-23.3%改善（+12.93°→+9.92°）されたが、L脚は改善されず（-14.84°→-15.00°）、Yaw driftが8.3x悪化（+1.79°→-14.81°）**するという「部分的成功・全体的悪化」の結果となった。

V8完了後の根本原因分析により、**接地相hip_rollが関節リミット±25°に99.8%で飽和している**ことが判明。足間距離は210mm→77mmに63.4%狭窄しており、RLが関節リミットまで内向きに飽和させることで歩行効率を最大化していた。V4-V8の5バージョンにわたる報酬チューニングが全て失敗したのは、「内股の効率メリット vs ペナルティコスト」のパレート最適トレードオフを超えることができなかったためである。

次バージョン（V9）ではV7ベースに戻し、**URDF hip_roll関節リミットを±25°→±12°に縮小**することで、足が股関節直下より内側に入ることを物理的に不可能にする。exp008 V1の「胴体幅75%縮小」と同じ「物理構造 > 報酬設計」の原則に基づく抜本的解決策である。

## 備考

- ベースバージョン: V7
- 変更数: 1（stance_hip_roll_target_angle: 0.0 → 0.087 rad）
- 環境ファイル変更: あり（_reward_stance_hip_roll_targetをターゲット角度対応に拡張）
- 報酬項目数: 17維持
- 訓練時間: ~844秒（500イテレーション）
- サーベイ参照: Section 7.2.3（equivariant policy、L/R対称性の構造的限界）、Section 7.4.2（angular momentum imbalance）、Section 2.2.2/2.2.3（orientation/ang_vel_xyの相補性はYaw軸に適用外）
- **新知見**: ターゲット角度設定は片脚のみ応答する構造的限界がある。MLPのL/R独立最適化により、同一対称報酬でもL/R非対称な結果が生じる
- **新知見**: hip_roll range非対称化はYaw angular momentum imbalanceを通じてYaw driftを大幅悪化させる。hip_roll制御とYaw安定性は間接的に結合している
- **新知見**: ペナルティ最小点の移動（0°→±5°）はraw penalty +23.6%を生成し、実効的にはスケール増と類似の効果を持つ。「形状変更のみ」という設計意図とは異なる結果
- **教訓**: 報酬shapingの形状変更はスケール変更と独立ではない。ターゲット角度とスケールを同時に調整（スケール減でraw penalty増を相殺）する必要があった可能性
- **【根本原因分析】接地相hip_rollは関節リミット±25°に99.8%飽和**: これまでの「offset ±15°」は歩行全体の時間平均であり、実態は毎接地時に関節リミットまで内向きにロックしていた
- **【根本原因分析】足間距離77mm（ニュートラル210mmの37%）**: 各足が股関節から37mm内側に入り込み、RLは効率的な狭い足間距離歩行を選択している
- **【根本原因分析】報酬チューニングの構造的限界**: V4-V8の5バージョンで同一の「効率メリット vs ペナルティコスト」トレードオフに衝突。パレート最適を超えるには物理制約空間自体の変更が必要
- **【根本原因分析】hip_roll=-12°で足が股関節直下**: 前方運動学による幾何学的解析。±12°リミットで内股を構造的に解消可能
