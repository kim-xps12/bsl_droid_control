# V1: exp007 V41案の継承 + 胴体幅75%縮小URDF

## 概要

exp008の初回バージョン。exp007 V40レポートで提案されたV41案（V39ベース + ankle_roll=-0.5）の報酬設計を、胴体幅75%縮小URDF（droid_simplified_v2）上で検証する。

## 前バージョンからの改善事項

### exp007からの継承

exp007で40バージョンにわたり確立された報酬設計を継承する。詳細は [exp007_summary_for_exp008.md](../exp007_droid_rl_walking_ref_unitree/exp007_summary_for_exp008.md) を参照。

| パラメータ | exp007 V39値 | exp008 V1値 | 変更理由 |

### 期待される効果

1. 胴体幅縮小（0.18→0.135m）+ 脚間隔縮小（0.10→0.075m）による横方向制御性向上

## 考察と改善案

### 成功点

1. **X速度が目標を大幅超過**: 0.190 m/s（目標0.15 m/s、V39比+55.7%）。exp007で40バージョン試行しても到達できなかった水準を、URDF変更で一気に達成
2. **swing品質の大幅改善**: swing_contact_penalty +71.8%改善、swing_duration +84.6%改善、両足接地率0.8%。明確な片脚支持（98.5%）による安定した交互歩行パターン
3. **hip_roll stance offset改善**: L: -18.80°→-15.46°（+17.8%）、R: +17.84°→+15.04°（+15.7%）。ankle_roll=-0.5の効果とURDF変更の複合効果
4. **hip_yaw stance twist大幅改善**: L: -8.05°→-2.03°（+74.8%）、R: +5.90°→+3.28°（+44.4%）。内股姿勢が大幅に改善
5. **脱トレンド横方向振動の半減**: 49.1mm→23.5mm（-52.1%）。Roll vs 横方向変位の相関が-0.958と非常に強く、Roll以外の横方向変動（CoM並進）が大幅に減少
6. **base_vel_y std半減**: 0.263→0.138 m/s（-47.5%）。横方向速度変動の大幅抑制
7. **hip_pitch相関の改善**: -0.588→-0.681（+15.8%）。交互歩行パターンがより明確
8. **swing持続時間の延長と一貫性**: L: 0.531s、R: 0.571s。個別swingのばらつきが極めて小さく安定
9. **報酬バランスの健全化**: ペナルティ/正報酬比率0.142（V39: 0.199）。静止ポリシーリスクは極めて低い

### ユーザー目視観察

exp007 V41案からの改善傾向が確認された。ただし以下の課題が指摘された:
- **遊脚の持ち上げが大きすぎて不自然**: swing中の足高さが過度に高く、歩行の自然さを損なっている
- **横揺れの低減が引き続き必要**: 定量的には改善しているが、目視ではまだ揺れが目立つ

### 課題

1. **Yawドリフト**: +16.46°（方向反転、ドリフト速度1.801°/s）。V39（-19.17°、-0.810°/s）と比べ速度は2.2倍悪化
2. **横方向ドリフト**: +70.5 mm/s（V39: -33.0 mm/s）。Yawドリフト+X速度増加の複合効果
3. **遊脚高さが過大（目視指摘）**: feet_swing_heightペナルティ-0.0544（V39比-47.4%悪化）。swing_duration延長に伴い遊脚が高く上がりすぎている
4. **Roll std微増**: 6.72°（V39: 6.06°、+10.9%）。支持基底面縮小の影響。目視でも横揺れが確認
5. **Pitch std微増**: 1.56°（V39: 1.34°、+16.4%）。同上
6. **hip_pitch非対称度**: -11.9%（R range > L range）。V39（+1.9%）から悪化

### 因果メカニズム分析

#### 胴体幅縮小が歩行品質を構造的に改善した理由

1. **支持基底面の25%縮小**（hip_offset_y: 0.10→0.075m）により、両脚支持状態の安定性が低下
2. 「立ち止まる」局所最適が物理的に不利化 → 動的歩行の必然化
3. 明確な片脚支持-遊脚パターンの発現 → swing_contact_penalty +71.8%、both_feet 0.8%
4. 効率的な前進推進力 → X速度+55.7%

これはサーベイSection 8.1（静止ポリシー問題）の逆パターン：ロボットの物理構造が報酬シェーピングよりも強力に歩行パターンを規定した好例。

#### Yaw方向反転のメカニズム

1. RL学習の確率性により、hip_yaw L/R非対称パターンが反転（V39: L>R → V1: R>L）
2. R hip_yaw range（0.469）> L（0.353）→ R脚スイング時の角運動量が大きい → 正方向Yawドリフト
3. izz低下（0.04→0.029、-27.5%）により同じ非対称に対するYaw角加速度が増加 → ドリフト速度の相対的増加
4. 一方、izz低下は制御応答性も向上 → Yaw絶対値は14.1%改善

#### Roll/Pitch微悪化の原因

1. 支持基底面幅の25%縮小 → 横方向の転倒余裕が直接的に減少 → Roll std +10.9%
2. iyy低下（0.05→0.038、-24%）→ Pitch角加速度感度の増加 → Pitch std +16.4%
3. ang_vel_xy=-0.1の維持が大幅悪化を防止（悪化幅は支持基底面縮小率25%より小さい10-11%に抑制）

#### 横方向ドリフト増加の分解

1. Yaw由来の横成分: 0.190 × sin(16.46°) ≈ 54 mm/s
2. 実測ドリフト: 70.5 mm/s → 差分16.5 mm/sはL/R脚非対称による本質的横方向ドリフト
3. V39: 0.122 × sin(19.17°) ≈ 40 mm/s（実測33.0 mm/s）
4. V1はX速度増加（+55.7%）とYawドリフトの複合効果でドリフトが増大

### 報酬設計のシステム分析

#### 報酬寄与度ランキング（ペナルティ上位）

1. ang_vel_xy: 23.2% — Roll/Pitch角速度制御の主力
2. hip_pos: 19.2% — hip_yaw+hip_roll制約（V39比-29.6%で機能が低下傾向）
3. feet_swing_height: 12.2% — swing_duration延長に伴い増加
4. action_rate: 9.6% — 動作平滑化
5. ankle_roll: 8.7% — V1新規、hip_roll独立制御

#### 削減候補の検討

- velocity_deficit（0.5%）: 寄与極小だがX速度0.190（目標超過）のため適正に機能中
- contact_no_vel（0.9%）: 同様に寄与極小
- いずれも現時点で削除の必要性は低い

### サーベイ知見との照合

- **支持基底面縮小→swing改善**: 強く整合（Survey Section 8.1の逆パターン）。物理構造が動的歩行を強制
- **izz低下→Yaw制御**: 部分的に整合。絶対値改善はizz低下と整合するが、方向反転はRL確率性の寄与が大きい
- **支持基底面縮小→Roll/Pitch微増**: 強く整合（Survey Section 6.1）。ang_vel_xyが大幅悪化を防止
- **hip_yaw L/R非対称→Yaw方向**: 整合（Survey Section 7.4.2、exp007 V34パターン再現）。ただし確率的
- **ankle_roll→hip_roll独立制御**: 設計意図と整合するがURDF変更との分離困難

### 次バージョンへの提案

#### 推奨案: hip_pos緩和（-0.8 → -0.5）

| パラメータ | V1値 | V2値 | 変更理由 |
|-----------|------|------|---------|
| hip_pos | -0.8 | -0.5 | hip_yaw自由度をさらに回復しYawドリフト改善 |

**根拠**:

1. V1のhip_yaw stance twistは大幅改善済み（L: -2.03°、R: +3.28°）。narrow URDFの物理構造がhip_yaw twistを抑制しており、hip_pos制約の必要性が低下
2. ankle_roll=-0.5がhip_roll制御を担っているため、hip_posの役割がhip_yaw制約に偏っている。緩和してhip_yaw自由度を回復しYaw修正能力を向上
3. X速度0.190は目標0.15を十分に超えているため、多少のX速度低下は許容可能
4. exp007 V40でhip_pos=0（完全除去）は失敗したが、V1のnarrow URDFではhip_yaw twistが自然に抑制されているため、-0.5の段階的緩和は安全と予想
5. 1変更1検証の原則を遵守

**期待される効果**:
- hip_yaw自由度の拡大 → Yaw修正能力の向上 → Yawドリフト改善
- 横方向ドリフトの改善（Yaw改善に伴い）

**リスク**:
- X速度低下の可能性（exp007でhip_pos緩和→X速度低下の前例あり）
- hip_yaw twistが増加する可能性（ただしnarrow URDFの構造的抑制で軽減を期待）

#### V3以降の改善候補（ユーザー目視フィードバックに基づく）

以下はV2の結果を踏まえて段階的に対処する:

1. **遊脚高さの低減**: feet_swing_heightスケール緩和（-8.0 → -4.0等）またはswing_height_target（環境パラメータ）の調整。V1のfeet_swing_heightペナルティは-0.0544（ペナルティ3位、12.2%）と大きく、遊脚を高く上げすぎる原因。スケール緩和で自然な足運びを目指す
2. **横揺れのさらなる低減**: ang_vel_xy強化（-0.1 → -0.15）またはankle_roll強化（-0.5 → -0.8）。Roll std 6.72°の改善を目指す

#### 代替案A: ang_vel_xy強化（-0.1 → -0.15）

Roll/Pitch微増と横揺れの抑制を優先する場合。exp007 V38で効果実証済み。ただしYawドリフトの直接的解消にはならない。

#### 代替案B: 訓練延長（500 → 800 iterations）

報酬がまだ上昇トレンド（std 1.63）のため、追加学習で全指標の改善を目指す。報酬設計変更なしで効果を確認できるが、Yawドリフトの根本解決にはならない。

## まとめ

exp008 V1は、exp007 V41案（V39ベース + ankle_roll=-0.5）の報酬設計を胴体幅75%縮小URDF上で初めて検証した。

**達成した改善**:
- X速度 0.190 m/s: 目標0.15 m/sを大幅超過（V39比+55.7%）
- swing品質: both_feet 0.8%、swing_duration 0.360（V39比+84.6%）
- hip_roll stance offset: 3°改善（-18.80→-15.46、+17.84→+15.04）
- hip_yaw stance twist: 4-6°改善（-8.05→-2.03、+5.90→+3.28）
- 脱トレンド横方向振動: 23.5mm（V39比-52.1%）
- hip_pitch相関: -0.681（V39比+15.8%）

**残存する課題**:
- Yawドリフト +16.46°（方向反転、ドリフト速度1.801°/s）
- Roll/Pitch微増（+10.9% / +16.4%、支持基底面縮小の影響）
